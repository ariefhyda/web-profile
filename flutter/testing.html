Modul 13: Testing Aplikasi Flutter

Estimasi: 90 Menit
Teori & Praktikum

Sebelum aplikasi di-publish, Anda harus yakin bahwa fitur-fitur penting tidak rusak setiap kali ada perubahan kode. Testing otomatis membantu Anda menjaga kualitas aplikasi tanpa harus klik-klik manual setiap kali ada update. Modul ini membahas tiga jenis testing di Flutter: Unit Test, Widget Test, dan Integration Test.

Buat Project Baru
Terminal
$ flutter create flutter_testing
$ cd flutter_testing

Project Flutter baru sudah otomatis memiliki folder test/ dan file test/widget_test.dart. Package flutter_test juga sudah terdaftar di pubspec.yaml bagian dev_dependencies.

Piramida Testing

Konsep dasar yang harus dipahami: ada tiga level testing dengan trade-off berbeda antara kecepatan dan cakupan.

  1. Unit Test — Menguji satu function atau class secara terisolasi. Sangat cepat (milidetik), mudah ditulis, tapi cakupannya sempit.
  2. Widget Test — Menguji satu widget beserta interaksinya. Lebih lambat tapi memvalidasi UI. Tidak butuh emulator.
  3. Integration Test — Menguji seluruh aplikasi berjalan di device/emulator nyata. Paling realistis tapi paling lambat.
Aturan Praktis

Pakai komposisi 70% unit test, 20% widget test, 10% integration test. Unit test melindungi logic bisnis, widget test melindungi UI, integration test melindungi happy path end-to-end.

Unit Test — Menguji Fungsi & Class

Bayangkan Anda menulis class Kalkulator dengan beberapa method. Unit test memastikan setiap method bekerja sesuai harapan. Setup-nya minim: tidak perlu UI, tidak perlu emulator.

Mari buat class yang akan diuji di lib/kalkulator.dart:

lib/kalkulator.dart
class Kalkulator {
  int tambah(int a, int b) => a + b;
  int kurang(int a, int b) => a - b;
  double bagi(int a, int b) {
    if (b == 0) throw ArgumentError('Pembagi tidak boleh nol');
    return a / b;
  }
}

Lalu buat file test di test/kalkulator_test.dart:

test/kalkulator_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:flutter_testing/kalkulator.dart';

void main() {
  // Group untuk mengelompokkan test yang berkaitan
  group('Kalkulator', () {
    late Kalkulator kalkulator;

    setUp(() {
      // Dijalankan sebelum setiap test
      kalkulator = Kalkulator();
    });

    test('tambah dua bilangan positif', () {
      expect(kalkulator.tambah(2, 3), 5);
    });

    test('tambah dengan nol', () {
      expect(kalkulator.tambah(5, 0), 5);
    });

    test('kurang menghasilkan negatif', () {
      expect(kalkulator.kurang(3, 10), -7);
    });

    test('bagi normal', () {
      expect(kalkulator.bagi(10, 2), 5.0);
    });

    test('bagi nol harus throw ArgumentError', () {
      expect(() => kalkulator.bagi(10, 0), throwsArgumentError);
    });
  });
}
Anatomi Test (Pola AAA)

Arrange — siapkan data/object yang dibutuhkan (di setUp() atau di awal test).

Act — panggil method yang ingin diuji.

Assert — verifikasi hasilnya dengan expect().

Menjalankan Test

Terminal
# Jalankan semua test
$ flutter test

# Jalankan file tertentu
$ flutter test test/kalkulator_test.dart

# Jalankan dengan coverage report
$ flutter test --coverage

# Lihat coverage di browser (perlu lcov)
$ genhtml coverage/lcov.info -o coverage/html

Macam-Macam Matcher

expect() menerima berbagai matcher untuk verifikasi. Berikut yang paling sering dipakai:

matchers.dart
expect(5, 5);                                    // sama persis
expect('arif', equals('arif'));                  // equals (ekuivalen)
expect(10, greaterThan(5));                      // lebih besar
expect([1, 2, 3], hasLength(3));                // panjang list
expect([1, 2, 3], contains(2));                 // element ada
expect('hello world', contains('world'));          // substring
expect(null, isNull);                            // null check
expect('arif', isNotNull);                         // not null
expect(true, isTrue);                             // boolean true
expect(3.14, closeTo(3.0, 0.2));                   // approx (untuk double)
expect(() => Kalkulator().bagi(1, 0), throwsArgumentError);

Widget Test — Menguji UI

Widget test memvalidasi bahwa widget Anda merender dan merespon dengan benar. Anda tidak perlu emulator — Flutter punya engine render headless khusus untuk testing.

Misalkan Anda punya counter widget di lib/counter_widget.dart:

lib/counter_widget.dart
import 'package:flutter/material.dart';

class CounterWidget extends StatefulWidget {
  const CounterWidget({super.key});
  @override
  State<CounterWidget> createState() => _CounterWidgetState();
}

class _CounterWidgetState extends State<CounterWidget> {
  int _angka = 0;
  @override
  Widget build(BuildContext context) {
    return Column(children: [
      Text('$_angka', key: const Key('counter'), style: const TextStyle(fontSize: 32)),
      ElevatedButton(
        key: const Key('plus'),
        onPressed: () => setState(() => _angka++),
        child: const Text('+'),
      ),
    ]);
  }
}

Test-nya di test/counter_widget_test.dart:

test/counter_widget_test.dart
import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:flutter_testing/counter_widget.dart';

void main() {
  testWidgets('Counter mulai dari 0 dan bertambah saat tombol ditekan',
      (WidgetTester tester) async {

    // 1. Render widget
    await tester.pumpWidget(const MaterialApp(home: Scaffold(body: CounterWidget())));

    // 2. Verifikasi tampilan awal
    expect(find.text('0'), findsOneWidget);
    expect(find.text('1'), findsNothing);

    // 3. Simulasi user tap tombol
    await tester.tap(find.byKey(const Key('plus')));
    await tester.pump(); // trigger rebuild

    // 4. Verifikasi state berubah
    expect(find.text('0'), findsNothing);
    expect(find.text('1'), findsOneWidget);

    // 5. Tap lagi
    await tester.tap(find.byKey(const Key('plus')));
    await tester.pump();
    expect(find.text('2'), findsOneWidget);
  });
}
Finder & Action Penting

find.text('Halo') — cari widget berdasarkan teks.

find.byType(ElevatedButton) — cari berdasarkan tipe widget.

find.byKey(Key('plus')) — cari berdasarkan Key.

find.byIcon(Icons.add) — cari berdasarkan ikon.

tester.tap() — simulasi klik.

tester.enterText() — ketik di TextField.

tester.drag() — simulasi drag/swipe.

tester.pump() — render ulang satu frame.

tester.pumpAndSettle() — render terus hingga semua animasi selesai.

Menguji Form & TextField

Skenario umum lainnya: validasi form input. Berikut contoh test untuk form login sederhana yang menampilkan pesan error jika email kosong.

test/login_form_test.dart
testWidgets('Login menolak email kosong', (tester) async {
  await tester.pumpWidget(const MaterialApp(home: LoginForm()));

  // Ketik password tapi biarkan email kosong
  await tester.enterText(find.byKey(const Key('pass')), 'rahasia123');

  // Tekan tombol submit
  await tester.tap(find.text('Masuk'));
  await tester.pump();

  expect(find.text('Email wajib diisi'), findsOneWidget);
});

testWidgets('Login berhasil dengan data lengkap', (tester) async {
  await tester.pumpWidget(const MaterialApp(home: LoginForm()));

  await tester.enterText(find.byKey(const Key('email')), 'arif@kampus.ac.id');
  await tester.enterText(find.byKey(const Key('pass')), 'rahasia123');
  await tester.tap(find.text('Masuk'));
  await tester.pumpAndSettle();

  expect(find.text('Selamat datang'), findsOneWidget);
});

Mocking untuk Async / HTTP

Saat menguji function yang memanggil API, JANGAN benar-benar hit server di test. Gunakan mock dengan package mockito agar test cepat dan tidak bergantung jaringan.

Terminal
$ flutter pub add --dev mockito build_runner
test/api_service_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:mockito/annotations.dart';
import 'package:mockito/mockito.dart';
import 'package:http/http.dart' as http;

// Generate mock dengan build_runner: dart run build_runner build
@GenerateMocks([http.Client])
import 'api_service_test.mocks.dart';

void main() {
  test('getPosts mengembalikan list saat sukses', () async {
    final client = MockClient();

    when(client.get(any)).thenAnswer((_) async =>
      http.Response('[{"id":1,"title":"X","body":"Y"}]', 200));

    final service = PostService(client);
    final posts = await service.getPosts();

    expect(posts, hasLength(1));
    expect(posts.first.title, 'X');
  });
}

Integration Test — End-to-End

Integration test menjalankan aplikasi nyata di emulator/device, mensimulasikan user dari awal sampai akhir. Cocok untuk memvalidasi alur penting (login → tambah todo → logout).

Terminal
$ flutter pub add --dev integration_test
$ mkdir integration_test
integration_test/app_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:flutter_testing/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  testWidgets('Flow tambah todo end-to-end', (tester) async {
    app.main();
    await tester.pumpAndSettle();

    // Tekan FAB
    await tester.tap(find.byIcon(Icons.add));
    await tester.pumpAndSettle();

    // Isi form
    await tester.enterText(find.byType(TextField).first, 'Belajar Flutter');
    await tester.tap(find.text('Simpan'));
    await tester.pumpAndSettle();

    // Verifikasi todo muncul di list
    expect(find.text('Belajar Flutter'), findsOneWidget);
  });
}
Terminal
# Jalankan integration test (perlu emulator/device aktif)
$ flutter test integration_test/app_test.dart
Best Practice Testing

1. Beri Key di widget penting — supaya mudah ditemukan di test, tidak bergantung pada teks yang bisa berubah.

2. Pisahkan logic dari UI — class service/repository yang murni Dart lebih mudah di-unit-test daripada code yang nempel di widget.

3. Test edge case — input kosong, input invalid, jaringan error, list kosong, dll.

4. Jangan test framework — tidak perlu test bahwa setState bekerja, fokus pada logic Anda sendiri.

Tantangan Mahasiswa

1. Tambahkan 5 test baru untuk class Kalkulator: kali, modulo, akar pangkat 2, pangkat, dan persentase. Pastikan ada test untuk edge case (mis. modulo dengan nol).

2. Buat widget ProfileCard yang menerima parameter nama dan email. Tulis test yang memverifikasi kedua data muncul di layar saat widget di-render.

3. Bonus: Kembali ke project flutter_state_management di Modul 9. Tulis widget test untuk fitur tambah-todo, toggle, dan hapus. Hubungi rebuild dengan tester.pump() setelah setiap aksi.
