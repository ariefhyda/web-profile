Modul 4: Function & OOP di Dart

Estimasi: 60 Menit
Teori & Praktikum

Modul ini membahas cara menulis function di Dart serta pilar-pilar Object-Oriented Programming (OOP) yang menjadi fondasi utama pengembangan aplikasi Flutter.

Mengapa Modul Ini Penting?

Di Flutter, semuanya adalah widget, dan setiap widget adalah sebuah class. Anda akan menulis puluhan class dan function setiap kali membangun aplikasi. Tanpa pemahaman kuat di modul ini, Anda akan kesulitan memahami cara kerja Flutter di modul-modul selanjutnya.

Function Dasar

Function (fungsi) adalah blok kode yang dapat dipanggil berulang kali. Di Dart, setiap function memiliki return type (tipe kembalian). Jika function tidak mengembalikan nilai, gunakan void. Jika body function hanya berisi satu ekspresi, Anda bisa menggunakan arrow syntax (=>) untuk mempersingkat penulisan.

function_dasar.dart
// Function dengan return type
int tambah(int a, int b) {
  return a + b;
}

// Arrow function (satu ekspresi)
int kali(int a, int b) => a * b;

// Function tanpa return (void)
void spiout(String pesan) {
  print('[LOG] $pesan');
}

void main() {
  print(tambah(3, 5));  // 8
  print(kali(4, 6));    // 24
  spiout('Halo!');       // [LOG] Halo!
}

Kapan menggunakan arrow function? Gunakan arrow (=>) ketika body function hanya berisi satu baris ekspresi. Untuk logika yang lebih kompleks (if-else, loop, multi-statement), gunakan kurung kurawal biasa {}.

Named & Optional Parameters

Dart mendukung tiga jenis parameter yang membuat function sangat fleksibel:

  1. Positional (wajib) — parameter biasa yang harus diisi sesuai urutan. Contoh: tambah(3, 5)
  2. Named (dengan {}) — parameter yang dipanggil dengan nama, sehingga urutan tidak penting dan kode lebih terbaca. Tambahkan required jika wajib diisi.
  3. Optional positional (dengan []) — parameter yang boleh tidak diisi, biasanya diberi nilai default.
parameters.dart
// Named parameters (dengan {})
void profil({required String nama, int umur = 20}) {
  print('$nama, $umur tahun');
}

// Optional positional (dengan [])
String sapa(String nama, [String gelar = '']) {
  return gelar.isEmpty ? 'Halo, $nama!' : 'Halo, $gelar $nama!';
}

void main() {
  profil(nama: 'Arif', umur: 28);
  profil(nama: 'Budi');  // umur pakai default 20

  print(sapa('Arif'));           // Halo, Arif!
  print(sapa('Arif', 'Pak'));    // Halo, Pak Arif!
}
Kenapa Penting?

Di Flutter, hampir semua Widget constructor menggunakan named parameters. Contoh: Text('Hello', style: TextStyle(fontSize: 20)). Memahami konsep ini krusial sebelum masuk ke Flutter UI.

Class & Object

Dart adalah bahasa 100% object-oriented. Semua nilai di Dart adalah object, termasuk angka dan fungsi. Sebuah class adalah cetak biru (blueprint), sedangkan object adalah hasil nyata (instance) dari cetak biru tersebut.

Komponen utama sebuah class:

  1. Properties — variabel yang dimiliki oleh class (menyimpan data/state).
  2. Constructor — function khusus yang dipanggil saat object dibuat. Di Dart, bisa menggunakan shorthand this.nama untuk langsung meng-assign parameter ke property.
  3. Method — function di dalam class yang bisa mengakses properties.
  4. Getter/Setter — cara elegan untuk mengakses atau mengubah property dengan logika tambahan.
class_object.dart
class Mahasiswa {
  // Properties
  String nama;
  String nim;
  double ipk;

  // Constructor
  Mahasiswa({
    required this.nama,
    required this.nim,
    this.ipk = 0.0,
  });

  // Method
  String info() => '$nama ($nim) — IPK: $ipk';

  // Getter
  bool get cumlaude => ipk >= 3.5;
}

void main() {
  var mhs = Mahasiswa(nama: 'Budi', nim: '2021001', ipk: 3.75);
  print(mhs.info());      // Budi (2021001) — IPK: 3.75
  print(mhs.cumlaude);    // true
}
Encapsulation: Private di Dart

Dart tidak punya keyword private seperti Java. Sebagai gantinya, tambahkan underscore _ di depan nama property atau method untuk menjadikannya private di level library/file. Contoh: String _password; artinya property ini hanya bisa diakses dari file yang sama.

Inheritance (Pewarisan)

Inheritance memungkinkan kita membuat class baru yang mewarisi seluruh property dan method dari class lain (parent). Gunakan keyword extends. Child class bisa meng-override (menimpa) method parent dengan anotasi @override. Ini juga mendemonstrasikan Polymorphism — objek child bisa diperlakukan sebagai objek parent.

inheritance.dart
// Parent class
class Hewan {
  String nama;
  Hewan(this.nama);

  void suara() => print('$nama bersuara...');
}

// Child class (extends)
class Kucing extends Hewan {
  Kucing(String nama) : super(nama);

  @override
  void suara() => print('$nama: Meow! 🐱');
}

class Anjing extends Hewan {
  Anjing(String nama) : super(nama);

  @override
  void suara() => print('$nama: Woof! 🐶');
}

void main() {
  var hewan = [Kucing('Milo'), Anjing('Buddy')];
  for (var h in hewan) {
    h.suara(); // Polymorphism!
  }
}

Perhatikan pada main(), variabel hewan bertipe List<Hewan> namun berisi object Kucing dan Anjing. Ketika h.suara() dipanggil, Dart secara otomatis menjalankan versi method yang sesuai dengan tipe object sebenarnya — inilah polymorphism dalam aksi.

Abstract Class & Interface

Abstract class adalah class yang tidak bisa di-instantiate (tidak bisa dibuat object-nya secara langsung). Fungsinya sebagai kontrak/blueprint yang memaksa child class untuk mengimplementasikan method-method tertentu. Ini sangat berguna untuk mendefinisikan "apa yang harus bisa dilakukan" tanpa menentukan "bagaimana caranya".

abstract.dart
// Abstract class — tidak bisa di-instantiate langsung
abstract class Bentuk {
  double luas();      // abstract method
  double keliling();  // abstract method

  void info() {
    print('Luas: ${luas()}, Keliling: ${keliling()}');
  }
}

class Lingkaran extends Bentuk {
  double radius;
  Lingkaran(this.radius);

  @override
  double luas() => 3.14 * radius * radius;

  @override
  double keliling() => 2 * 3.14 * radius;
}

class Persegi extends Bentuk {
  double sisi;
  Persegi(this.sisi);

  @override
  double luas() => sisi * sisi;

  @override
  double keliling() => 4 * sisi;
}

void main() {
  List<Bentuk> shapes = [Lingkaran(7), Persegi(5)];
  for (var s in shapes) { s.info(); }
}

Perhatikan bahwa Bentuk memiliki method info() yang sudah ada implementasinya — ini membuktikan abstract class boleh memiliki method konkret. Hanya method tanpa body (luas() dan keliling()) yang wajib diimplementasikan oleh child class.

Interface di Dart

Dart tidak memiliki keyword interface secara eksplisit. Setiap class di Dart secara implisit sudah menjadi interface. Gunakan implements (bukan extends) jika Anda ingin memaksa class untuk mengimplementasikan seluruh method dan property dari class lain — tanpa mewarisi implementasinya.

Mixin

Dart tidak mendukung multiple inheritance (satu class extends banyak class). Sebagai solusinya, Dart menyediakan Mixin — sebuah mekanisme untuk "menempelkan" kemampuan tambahan ke class menggunakan keyword with. Satu class bisa menggunakan banyak mixin sekaligus.

mixin.dart
mixin BisaTerbang {
  void terbang() => print('Terbang! ✈️');
}

mixin BisaBerenang {
  void berenang() => print('Berenang! 🏊');
}

class Bebek with BisaTerbang, BisaBerenang {
  String nama;
  Bebek(this.nama);
}

void main() {
  var donald = Bebek('Donald');
  donald.terbang();    // Terbang! ✈️
  donald.berenang();   // Berenang! 🏊
}
Rangkuman 4 Pilar OOP di Dart

Encapsulation — Menggunakan underscore _ untuk membuat property/method private.

Inheritance — extends untuk mewarisi dari parent class.

Polymorphism — @override untuk mengubah perilaku method parent.

Abstraction — abstract class untuk blueprint tanpa implementasi langsung.
