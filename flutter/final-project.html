Modul 16: Proyek Akhir (Capstone)

Estimasi: 3–4 Minggu
Proyek Mandiri

Selamat! Anda sudah menempuh perjalanan panjang dari Dart dasar sampai publish ke Play Store. Saatnya merangkai semua skill itu menjadi satu aplikasi nyata yang akan menjadi portfolio Anda. Modul ini berisi panduan ide, rubrik penilaian, dan deliverable yang harus dikumpulkan.

Tujuan Proyek Akhir

  1. Membuktikan Anda mampu mengintegrasikan minimal 8 dari 15 modul yang sudah dipelajari.
  2. Menghasilkan aplikasi yang bisa dijalankan di HP fisik, bukan sekadar simulasi.
  3. Mendokumentasikan proses pengembangan dengan baik (GitHub, README, demo video).
  4. Berlatih bekerja dalam tim kecil (opsional 2–3 orang).

Pilihan Tema Aplikasi

Pilih SATU dari tema di bawah ini, atau ajukan tema sendiri ke dosen pengampu. Setiap tema sudah dirancang untuk mencakup minimal 8 modul materi.

Tema 1: Aplikasi Catatan Resep Pribadi

Deskripsi: Pengguna bisa menyimpan resep masakan favorit lengkap dengan foto, bahan, dan langkah-langkah memasak. Resep tersimpan offline di SQLite.

Modul yang dipakai: Widget Dasar, Layout, Navigation, State Management, SQLite, Media (foto), Build, Publish.

Fitur wajib: CRUD resep, kamera/galeri untuk foto, kategori (sarapan/makan siang/dessert), search berdasarkan nama, share resep ke WhatsApp.

Tema 2: Aplikasi Cuaca Multikota

Deskripsi: Tampilkan cuaca real-time dari API OpenWeather untuk daftar kota favorit pengguna. Kota tersimpan lokal di SQLite/SharedPreferences.

Modul yang dipakai: Widget, Layout Responsif, Navigation, State Management, REST API, Local Storage, Testing, Build, Publish.

Fitur wajib: Daftar kota, detail cuaca harian + 5 hari ke depan, tambah/hapus kota, dark mode, pull-to-refresh.

Tema 3: Mini E-Commerce Mahasiswa

Deskripsi: Aplikasi belanja sederhana yang menampilkan produk dari Fake Store API dengan fitur keranjang, riwayat transaksi, dan checkout simulasi.

Modul yang dipakai: Widget, Layout Responsif, Navigation, State Management (Provider), REST API, SQLite, Build, Publish.

Fitur wajib: Grid produk, detail produk, keranjang, kuantitas +/-, total harga otomatis, "checkout" yang simpan riwayat ke SQLite.

Tema 4: Aplikasi Catatan Keuangan Pribadi

Deskripsi: Track pemasukan & pengeluaran harian dengan grafik bulanan. Data tersimpan offline di SQLite.

Modul yang dipakai: Widget, Layout, Navigation, State, SQLite, Media (lampiran nota), Build, Publish, Testing.

Fitur wajib: CRUD transaksi, kategori (makanan/transport/dll), grafik bar/pie (pakai package fl_chart), filter bulanan, export ke CSV.

Tema 5: Aplikasi Habit Tracker

Deskripsi: Catat kebiasaan harian (olahraga, baca buku, ngoding) dengan streak counter dan visualisasi kalender.

Modul yang dipakai: Widget, Layout, Navigation, State, SQLite, Testing, Build, Publish.

Fitur wajib: CRUD habit, checklist harian, perhitungan streak, visualisasi kalender (mark warna hijau di hari tercentang), reminder dengan local notification.

Tema 6 — Bebas (Ajukan ke Dosen)

Punya ide sendiri? Boleh banget! Syaratnya: ajukan proposal singkat (1 halaman) yang berisi nama aplikasi, masalah yang dipecahkan, fitur utama, dan peta ke modul materi. Dosen akan review dalam 3 hari kerja.

Persyaratan Teknis Minimum

Apapun tema yang dipilih, aplikasi Anda wajib memiliki:

  1. Minimal 4 halaman dengan navigasi yang baik (push/pop, named routes, atau bottom navigation).
  2. State management menggunakan Provider (atau setState untuk skala kecil).
  3. Penyimpanan data persisten menggunakan SQLite, SharedPreferences, atau keduanya.
  4. Minimal 1 integrasi eksternal: REST API, kamera/galeri, atau pemilihan file.
  5. Minimal 5 unit/widget test yang berhasil pass.
  6. Ikon dan splash screen kustom (bukan default Flutter).
  7. Berfungsi di HP fisik Android, bukan hanya emulator.
  8. Sudah di-build sebagai signed APK dengan keystore Anda.

Deliverable yang Dikumpulkan

Kumpulkan 5 item berikut dalam folder Drive yang di-share ke dosen pengampu:

  1. Source code (GitHub) — repository publik yang berisi seluruh kode, dengan README yang menjelaskan project. Format URL: github.com/username/nama-project.
  2. File APK signed — file .apk yang sudah ditandatangani dengan keystore Anda, siap di-install.
  3. Demo video — rekam layar HP yang menjalankan aplikasi Anda, max 5 menit, format MP4. Tunjukkan SEMUA fitur utama secara berurutan.
  4. Dokumen laporan (PDF) — maksimal 10 halaman. Isi: latar belakang masalah, screenshot tiap halaman, arsitektur (folder structure & state flow), package yang dipakai + alasannya, dan refleksi (kesulitan + cara mengatasi).
  5. (Opsional) Link Play Store — jika sudah berhasil publish ke produksi/closed testing.

Timeline Pengerjaan (4 Minggu)

  1. Minggu 1 — Perencanaan
    Pilih tema, riset API/package yang dibutuhkan, buat wireframe sederhana (boleh kertas atau Figma), setup project Flutter baru & repo GitHub. Output: proposal singkat + wireframe.
  2. Minggu 2 — Backend Lokal & API
    Implementasi model data, database helper SQLite, integrasi REST API, service class. Output: data layer berfungsi (bisa dites lewat unit test).
  3. Minggu 3 — UI & State
    Bangun semua halaman, sambungkan dengan state management, polish tampilan dengan dark mode/responsif. Output: aplikasi end-to-end berjalan di emulator.
  4. Minggu 4 — Polish & Deploy
    Tulis test, ikon & splash, build APK signed, demo video, dokumentasi README & laporan. Output: seluruh deliverable siap dikumpulkan.
Tips Sukses dari Dosen

1. Mulai dari yang minimal — bangun versi paling sederhana yang berjalan dulu, baru tambah fitur. Hindari godaan "ngoding sambil mikir desain final".

2. Commit GitHub sering — minimal sekali per hari. Pesan commit yang deskriptif ("Tambah fitur edit catatan", bukan "update").

3. Test di HP fisik sejak awal — emulator sering bohong, terutama untuk kamera/permission.

4. Tanya saat stuck — jangan habiskan 5 jam di satu bug. Tanya teman, dosen, atau Stack Overflow. Sertakan error message lengkap.

5. Pakai ulang kode dari modul — kode di Modul 1–15 sudah teruji jalan. Jangan menulis ulang semuanya dari nol.
