Modul 5: Collection & Async di Dart

Estimasi: 60 Menit
Teori & Praktikum

Modul ini membahas tiga Collection utama di Dart — List, Set, dan Map — serta pemrograman Asynchronous menggunakan Future, async/await, dan Stream.

Mengapa Modul Ini Penting?

Di Flutter, Anda akan sering mengolah list data dari API untuk ditampilkan di UI (ListView, GridView). Operasi pengambilan data dari server bersifat asynchronous — tanpa pemahaman async/await, aplikasi Anda akan terasa freeze saat menunggu respons jaringan.

List (Array)

List adalah collection paling umum di Dart. List menyimpan data secara berurutan dan bisa diakses menggunakan index (dimulai dari 0). List bisa berisi elemen dengan tipe yang sama (List<String>) atau campuran (List<dynamic>).

list_dart.dart
// List — koleksi data berurutan (index dimulai dari 0)
List<String> buah = ['Apel', 'Mangga', 'Jeruk'];

print(buah[0]);       // Apel
print(buah.length);  // 3

buah.add('Anggur');     // Tambah di akhir
buah.removeAt(1);       // Hapus index 1 (Mangga)
print(buah);           // [Apel, Jeruk, Anggur]

// Iterasi dengan for-in
for (var item in buah) {
  print('Buah: $item');
}

Set

Set mirip List, namun tidak mengizinkan duplikasi. Set sangat berguna ketika Anda perlu memastikan semua elemen unik, misalnya daftar tag atau kategori. Set juga mendukung operasi himpunan matematika seperti union, intersection, dan difference.

set_dart.dart
// Set — koleksi unik, tidak ada duplikat
Set<int> angka = {1, 2, 3, 2, 1};
print(angka);  // {1, 2, 3} — duplikat otomatis hilang

angka.add(4);
angka.add(2);    // Tidak berpengaruh, 2 sudah ada
print(angka);  // {1, 2, 3, 4}

// Operasi himpunan
var a = {1, 2, 3};
var b = {2, 3, 4};
print(a.union(b));         // {1, 2, 3, 4}
print(a.intersection(b));  // {2, 3}
print(a.difference(b));    // {1}

Map

Map menyimpan data dalam pasangan key-value. Setiap key bersifat unik. Map sangat sering digunakan untuk merepresentasikan data JSON dari API. Tipe umumnya adalah Map<String, dynamic>.

map_dart.dart
// Map — pasangan key-value
Map<String, dynamic> mhs = {
  'nama': 'Budi Santoso',
  'nim': '2021001',
  'ipk': 3.75,
};

print(mhs['nama']);   // Budi Santoso

mhs['semester'] = 5;   // Tambah entry baru
mhs.remove('ipk');     // Hapus entry

// Iterasi Map
mhs.forEach((key, value) {
  print('$key: $value');
});
Kapan Pakai List, Set, atau Map?

List — saat urutan penting dan boleh ada duplikat (misal: daftar pesan chat).

Set — saat elemen harus unik (misal: daftar ID yang sudah diproses).

Map — saat data berupa pasangan key-value (misal: profil user, konfigurasi).

Spread & Collection Operators

Dart menyediakan operator modern untuk memanipulasi collection secara deklaratif. Spread operator (...) menggabungkan collection, sedangkan collection if dan collection for memungkinkan logika kondisional langsung di dalam literal collection.

spread_collection.dart
// Spread operator (...)
var awal = [1, 2, 3];
var semua = [0, ...awal, 4, 5];
print(semua);  // [0, 1, 2, 3, 4, 5]

// Collection if
bool isAdmin = true;
var menu = [
  'Home',
  'Profile',
  if (isAdmin) 'Admin Panel',
];

// Collection for
var angka = [1, 2, 3];
var kuadrat = [
  for (var n in angka) n * n,
];
print(kuadrat);  // [1, 4, 9]

Method Penting pada Collection

Dart menyediakan method higher-order yang menerima function sebagai parameter. Method ini sangat powerful untuk mengolah data tanpa menulis loop manual — pola yang akan sering Anda temui di Flutter.

higher_order.dart
// map — transformasi setiap elemen
var angka = [1, 2, 3, 4, 5];
var kuadrat = angka.map((n) => n * n).toList();
print(kuadrat);  // [1, 4, 9, 16, 25]

// where — filter elemen
var genap = angka.where((n) => n % 2 == 0).toList();
print(genap);  // [2, 4]

// reduce — akumulasi nilai
var total = angka.reduce((a, b) => a + b);
print(total);  // 15

// any & every — cek kondisi
print(angka.any((n) => n > 4));    // true
print(angka.every((n) => n > 0));  // true

Future & Async/Await

Future merepresentasikan nilai yang belum tersedia saat ini tapi akan tersedia di masa depan. Ini adalah fondasi pemrograman asynchronous di Dart. Gunakan async untuk menandai function sebagai asynchronous, dan await untuk menunggu hasilnya.

future_async.dart
// Simulasi ambil data dari server
Future<String> ambilData() async {
  print('Mengambil data...');
  await Future.delayed(Duration(seconds: 2));
  return 'Data berhasil diambil!';
}

void main() async {
  print('Mulai');
  String hasil = await ambilData();
  print(hasil);
  print('Selesai');
}
// Output:
// Mulai
// Mengambil data...
// (tunggu 2 detik)
// Data berhasil diambil!
// Selesai

Penting: await hanya bisa digunakan di dalam function yang ditandai async. Tanpa await, kode akan melanjutkan eksekusi tanpa menunggu Future selesai.

Error Handling dengan Try-Catch

Operasi async seperti pemanggilan API bisa gagal kapan saja (jaringan putus, server error, dll). Gunakan try-catch-finally untuk menangani error secara elegan.

error_handling.dart
Future<String> ambilUser(int id) async {
  await Future.delayed(Duration(seconds: 1));
  if (id < 0) throw Exception('ID tidak valid');
  return 'User #$id';
}

void main() async {
  try {
    var user = await ambilUser(-1);
    print(user);
  } catch (e) {
    print('Error: $e');
  } finally {
    print('Proses selesai');
  }
}

Stream

Jika Future mengembalikan satu nilai di masa depan, maka Stream mengembalikan serangkaian nilai secara bertahap. Stream sangat berguna untuk real-time data seperti WebSocket, sensor, atau countdown timer. Gunakan async* dan yield untuk membuat Stream.

stream_dart.dart
// Stream — data yang datang bertahap
Stream<int> hitungMundur(int dari) async* {
  for (int i = dari; i >= 0; i--) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

void main() async {
  await for (var n in hitungMundur(3)) {
    print(n);  // 3, 2, 1, 0 (setiap detik)
  }
}
Rangkuman Async di Dart

Future — satu nilai async. Gunakan async/await.

Stream — banyak nilai async secara bertahap. Gunakan async*/yield dan await for.

try-catch — wajib digunakan untuk menangani error pada operasi async.

Di Flutter, semua ini digunakan intensif: FutureBuilder dan StreamBuilder adalah widget yang langsung memanfaatkan konsep ini.
