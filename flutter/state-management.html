Modul 9: State Management

Estimasi: 90 Menit
Teori & Praktikum

State adalah segala data yang bisa berubah seiring waktu di aplikasi Anda: angka counter, daftar todo, data user yang login, isi keranjang, dll. Modul ini membahas tiga teknik state management yang wajib dikuasai: setState, ValueNotifier, dan Provider.

Buat Project Baru
Terminal
$ flutter create flutter_state_management
$ cd flutter_state_management

# Untuk bagian Provider di akhir modul:
$ flutter pub add provider

Stateless vs Stateful Widget

Sebelum bicara teknik canggih, pahami dulu perbedaan dua jenis widget yang sudah Anda kenal sejak modul pertama:

  1. StatelessWidget — tampilannya tetap sepanjang umur widget. Cocok untuk tampilan statis (Text, Icon, kartu profil).
  2. StatefulWidget — punya data internal yang bisa berubah. Setiap kali datanya diperbarui melalui setState, Flutter akan me-build ulang tampilan secara otomatis.

setState — State Management Termudah

setState adalah cara paling fundamental untuk mengubah state. Konsepnya sederhana: ubah variabel di dalam setState(() { ... }), lalu Flutter otomatis me-render ulang widget.

lib/main.dart
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: CounterPage()));

class CounterPage extends StatefulWidget {
  const CounterPage({super.key});
  @override
  State<CounterPage> createState() => _CounterPageState();
}

class _CounterPageState extends State<CounterPage> {
  int _angka = 0;

  void _tambah() {
    setState(() {       // ← memicu rebuild
      _angka++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Counter Sederhana')),
      body: Center(
        child: Text('$_angka', style: const TextStyle(fontSize: 72)),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _tambah,
        child: const Icon(Icons.add),
      ),
    );
  }
}
Kenapa Harus Dibungkus setState?

Jika Anda hanya menulis _angka++ tanpa setState, nilainya memang berubah di memori — tapi Flutter tidak tahu bahwa harus me-render ulang. Hasilnya tampilan tetap menampilkan angka lama. setState adalah sinyal eksplisit: "data berubah, tolong build ulang!"

Lifting State Up — Berbagi State Antar Widget

Ketika dua widget bersaudara perlu berbagi state, pindahkan state-nya ke parent yang sama, lalu turunkan via constructor. Konsep ini disebut "lifting state up".

lifting_state.dart
// State disimpan di parent
class _ParentState extends State<Parent> {
  int _counter = 0;

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Tampilan(angka: _counter),                                 // terima state
        Tombol(onTambah: () => setState(() => _counter++)),         // ubah state
      ],
    );
  }
}

class Tampilan extends StatelessWidget {
  final int angka;
  const Tampilan({super.key, required this.angka});
  @override
  Widget build(BuildContext context) => Text('$angka');
}

class Tombol extends StatelessWidget {
  final VoidCallback onTambah;
  const Tombol({super.key, required this.onTambah});
  @override
  Widget build(BuildContext context) => ElevatedButton(onPressed: onTambah, child: const Text('+'));
}
Masalah Saat Aplikasi Membesar

Lifting state up bagus untuk aplikasi kecil. Tapi saat state perlu diakses dari widget yang jauh dalam tree (mis. dari halaman keempat ke halaman pertama), passing constructor terus-menerus menjadi merepotkan. Inilah masalah "prop drilling" — solusinya adalah Provider.

ValueNotifier — Reactive State Bawaan

ValueNotifier adalah teknik reactive bawaan Flutter — tidak perlu package tambahan. Cocok untuk state sederhana yang ingin di-listen tanpa harus StatefulWidget.

value_notifier.dart
final counter = ValueNotifier<int>(0);

// Listen perubahan secara reactive
ValueListenableBuilder<int>(
  valueListenable: counter,
  builder: (context, value, _) => Text('$value'),
)

// Ubah nilainya
ElevatedButton(
  onPressed: () => counter.value++,
  child: const Text('+'),
)

Provider — State Global yang Scalable

Untuk aplikasi nyata, Provider adalah pilihan paling banyak dipakai. Provider memungkinkan state diakses dari widget mana pun di bawah Provider, tanpa passing constructor.

Tiga komponen utama Provider:

  1. ChangeNotifier — class yang menyimpan state dan memberitahu listener saat berubah, dengan memanggil notifyListeners().
  2. ChangeNotifierProvider — widget yang men-inject ChangeNotifier ke widget tree.
  3. Consumer / context.watch — cara membaca state dan otomatis rebuild saat state berubah.
lib/main.dart (Provider)
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

// 1. Definisikan model state
class CounterModel extends ChangeNotifier {
  int _angka = 0;
  int get angka => _angka;

  void tambah() {
    _angka++;
    notifyListeners(); // beritahu semua listener: state berubah!
  }

  void reset() {
    _angka = 0;
    notifyListeners();
  }
}

void main() {
  runApp(
    // 2. Inject model ke tree
    ChangeNotifierProvider(
      create: (_) => CounterModel(),
      child: const MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.orange, useMaterial3: true),
      home: const CounterPage(),
    );
  }
}

class CounterPage extends StatelessWidget {
  const CounterPage({super.key});
  @override
  Widget build(BuildContext context) {
    // 3. Baca state (akan rebuild saat berubah)
    final counter = context.watch<CounterModel>();

    return Scaffold(
      appBar: AppBar(title: const Text('Counter + Provider')),
      body: Center(
        child: Text('${counter.angka}', style: const TextStyle(fontSize: 72)),
      ),
      floatingActionButton: Row(
        mainAxisAlignment: MainAxisAlignment.end,
        children: [
          FloatingActionButton(
            heroTag: 'reset',
            onPressed: () => context.read<CounterModel>().reset(),
            child: const Icon(Icons.refresh),
          ),
          const SizedBox(width: 12),
          FloatingActionButton(
            heroTag: 'add',
            onPressed: () => context.read<CounterModel>().tambah(),
            child: const Icon(Icons.add),
          ),
        ],
      ),
    );
  }
}
context.watch vs context.read

context.watch<T>() — membaca data dan rebuild widget saat data berubah. Pakai di build().

context.read<T>() — hanya membaca tanpa subscribe. Pakai di event handler (onPressed, dll) — tidak akan trigger rebuild.

Mini Project: TodoList dengan Provider

Aplikasi todo sederhana yang menyimpan daftar pekerjaan menggunakan Provider. Tampilan dan logic dipisah dengan rapi.

lib/main.dart
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

class Todo {
  final String judul;
  bool selesai;
  Todo(this.judul, {this.selesai = false});
}

class TodoModel extends ChangeNotifier {
  final List<Todo> _items = [];
  List<Todo> get items => List.unmodifiable(_items);

  void tambah(String judul) {
    if (judul.trim().isEmpty) return;
    _items.add(Todo(judul));
    notifyListeners();
  }

  void toggle(int index) {
    _items[index].selesai = !_items[index].selesai;
    notifyListeners();
  }

  void hapus(int index) {
    _items.removeAt(index);
    notifyListeners();
  }
}

void main() {
  runApp(ChangeNotifierProvider(
    create: (_) => TodoModel(),
    child: const MyApp(),
  ));
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});
  @override
  Widget build(BuildContext context) => MaterialApp(
    theme: ThemeData(colorSchemeSeed: Colors.green, useMaterial3: true),
    home: const TodoPage(),
  );
}

class TodoPage extends StatelessWidget {
  const TodoPage({super.key});
  @override
  Widget build(BuildContext context) {
    final model = context.watch<TodoModel>();
    final ctrl = TextEditingController();

    return Scaffold(
      appBar: AppBar(title: const Text('Todo List')),
      body: Column(
        children: [
          Padding(
            padding: const EdgeInsets.all(12),
            child: Row(children: [
              Expanded(child: TextField(controller: ctrl, decoration: const InputDecoration(hintText: 'Pekerjaan baru...'))),
              const SizedBox(width: 8),
              ElevatedButton(
                onPressed: () { context.read<TodoModel>().tambah(ctrl.text); ctrl.clear(); },
                child: const Text('+'),
              ),
            ]),
          ),
          Expanded(
            child: ListView.builder(
              itemCount: model.items.length,
              itemBuilder: (_, i) {
                final todo = model.items[i];
                return CheckboxListTile(
                  value: todo.selesai,
                  onChanged: (_) => context.read<TodoModel>().toggle(i),
                  title: Text(todo.judul, style: TextStyle(decoration: todo.selesai ? TextDecoration.lineThrough : null)),
                  secondary: IconButton(
                    icon: const Icon(Icons.delete),
                    onPressed: () => context.read<TodoModel>().hapus(i),
                  ),
                );
              },
            ),
          ),
        ],
      ),
    );
  }
}
Kapan Pakai Apa?

setState — state lokal di satu widget. Contoh: toggle password show/hide.

ValueNotifier — state sederhana yang ingin di-listen reactive tanpa setup library.

Provider — state global atau yang dipakai di banyak halaman. Standar industri.

Riverpod / Bloc — alternatif populer untuk skala enterprise. Akan dibahas di kuliah lanjutan.

Tantangan Mahasiswa

Modifikasi TodoList di atas:

1. Tambahkan counter di AppBar yang menampilkan jumlah todo yang belum selesai. Gunakan getter di TodoModel untuk menghitungnya.

2. Tambahkan tombol "Hapus Semua Selesai" di bottom bar.

3. Buat halaman kedua StatistikPage yang menampilkan total todo, jumlah selesai, dan persentase. State tetap diakses dari Provider yang sama.
