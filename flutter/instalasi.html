Modul 2: Instalasi & Konfigurasi Flutter

Estimasi: 90 Menit
Praktikum & Setup

Pada modul ini, kita akan menyiapkan seluruh lingkungan pengembangan (development environment) yang diperlukan untuk memulai coding Flutter. Ikuti langkah-langkah berikut secara berurutan agar proses berjalan lancar.

Persyaratan Sistem (Windows)

Minimum Requirement

OS: Windows 10 64-bit atau lebih baru

Disk: Minimal 2.5 GB ruang kosong (belum termasuk IDE & Android SDK)

RAM: Minimal 8 GB (disarankan 16 GB untuk emulator)

Tools: Git for Windows, Windows PowerShell 5.0+

Langkah 1: Download & Install Flutter SDK

  1. Download Flutter SDK
    Kunjungi docs.flutter.dev/get-started/install dan download file ZIP Flutter SDK versi terbaru (stable).
  2. Ekstrak ke Lokasi Permanen
    Ekstrak ZIP ke folder yang mudah diakses, misalnya C:\src\flutter. Jangan taruh di folder yang memerlukan hak akses admin (seperti C:\Program Files).
  3. Tambahkan ke PATH
    Buka Environment Variables → Edit variabel Path → Tambahkan C:\src\flutter\bin.
PowerShell 
# Verifikasi instalasi Flutter
flutter --version

# Output yang diharapkan (contoh):
# Flutter 3.24.x • channel stable
# Framework • revision xxxxxxx
# Engine • revision xxxxxxx
# Tools • Dart 3.5.x • DevTools 2.37.x

Langkah 2: Install Android Studio

Android Studio diperlukan untuk menyediakan Android SDK, Android Emulator, dan build tools. Meskipun kita akan menggunakan VS Code sebagai code editor, Android Studio tetap wajib di-install.

  1. Download Android Studio
    Kunjungi developer.android.com/studio dan install versi terbaru.
  2. Jalankan Setup Wizard
    Pilih Standard Installation. Wizard akan otomatis mengunduh Android SDK, SDK Command-line Tools, dan Build Tools.
  3. Install SDK Platform
    Buka SDK Manager → Tab SDK Platforms → Centang Android 14 (API 34) atau versi terbaru.
  4. Install Command-line Tools
    Tab SDK Tools → Centang Android SDK Command-line Tools (latest) dan Android SDK Build-Tools.
  5. Accept Licenses
    Jalankan perintah di bawah untuk menerima seluruh lisensi Android:
PowerShell 
# Accept semua lisensi Android SDK
flutter doctor --android-licenses

# Tekan 'y' untuk setiap prompt yang muncul

Langkah 3: Setup Android Emulator

  1. Buka AVD Manager
    Di Android Studio → Tools → Device Manager → Klik Create Device.
  2. Pilih Device
    Pilih Pixel 7 atau device apapun → Klik Next.
  3. Pilih System Image
    Download dan pilih image API 34 (Android 14) dengan ABI x86_64 → Klik Next → Finish.
  4. Jalankan Emulator
    Klik tombol ▶ (Play) di samping virtual device yang baru dibuat.
Alternatif: Gunakan HP Fisik

Jika RAM laptop terbatas, gunakan HP Android fisik sebagai target device:
1. Aktifkan Developer Options (tap 7x di "Build Number" pada Settings → About Phone).
2. Aktifkan USB Debugging di Developer Options.
3. Sambungkan HP ke laptop via kabel USB → Pilih "Transfer Files/MTP".
4. Jalankan flutter devices untuk memastikan HP terdeteksi.

Langkah 4: Setup Visual Studio Code

VS Code adalah editor ringan yang sangat direkomendasikan untuk Flutter development sehari-hari.

  1. Download VS Code
    Kunjungi code.visualstudio.com dan install.
  2. Install Extension Flutter
    Buka VS Code → Extensions (Ctrl+Shift+X) → Cari "Flutter" (by Dart Code) → Install. Extension Dart akan otomatis terinstall.
  3. Install Extension Tambahan (Opsional)
    • Awesome Flutter Snippets — snippet kode siap pakai
    • Pubspec Assist — manajemen package pubspec.yaml
    • Error Lens — highlight error langsung di baris kode

Langkah 5: Verifikasi dengan Flutter Doctor

Perintah flutter doctor adalah cara terbaik untuk memastikan seluruh komponen telah terpasang dengan benar. Targetkan semua item mendapat centang hijau (✓).

PowerShell 
flutter doctor

# ✓ Flutter (Channel stable, 3.24.x)
# ✓ Windows Version (Windows 10/11)
# ✓ Android toolchain (API 34)
# ✓ Android Studio (2024.x)
# ✓ VS Code (version 1.9x)
# ✓ Connected device (1 available)
# • No issues found!

Langkah 6: Membuat Project Flutter Pertama

Setelah semua siap, saatnya membuat project Flutter pertama Anda!

PowerShell 
# Buat project baru
flutter create hello_flutter

# Masuk ke folder project
cd hello_flutter

# Jalankan di emulator/device
flutter run

# Atau jalankan di Chrome (web)
flutter run -d chrome

Setelah perintah flutter run selesai, Anda akan melihat aplikasi Flutter Demo default berjalan di emulator atau device fisik. Tekan r di terminal untuk Hot Reload dan R untuk Hot Restart.

Struktur Project Flutter

Berikut adalah struktur folder utama yang perlu Anda pahami:

Struktur Folder 
hello_flutter/
├── android/          # Konfigurasi native Android
├── ios/              # Konfigurasi native iOS
├── lib/              # ⭐ KODE UTAMA ANDA DI SINI
│   └── main.dart     # Entry point aplikasi
├── test/             # Unit test & widget test
├── web/              # Konfigurasi Flutter Web
├── pubspec.yaml      # ⭐ Dependency & package manager
├── pubspec.lock      # Lock file (jangan edit manual)
└── README.md         # Dokumentasi project
Catatan Penting

Semua kode Dart Anda ditulis di dalam folder lib/. File pubspec.yaml digunakan untuk mendaftarkan package/dependency pihak ketiga. Jangan pernah mengedit file di folder android/ atau ios/ kecuali Anda tahu pasti apa yang dilakukan.

Troubleshooting Umum

  1. "flutter" is not recognized
    PATH belum diset dengan benar. Pastikan C:\src\flutter\bin sudah ditambahkan ke System Environment Variables, lalu restart terminal.
  2. Android licenses not accepted
    Jalankan flutter doctor --android-licenses dan tekan y untuk setiap prompt.
  3. Emulator sangat lambat
    Aktifkan Hardware Acceleration (HAXM/Hyper-V) di BIOS. Atau gunakan HP fisik sebagai alternatif.
  4. No connected devices
    Pastikan emulator sedang berjalan atau HP terhubung via USB dengan USB Debugging aktif. Jalankan flutter devices untuk verifikasi.
