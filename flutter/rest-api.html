Modul 10: REST API & HTTP

Estimasi: 90 Menit
Teori & Praktikum

Hampir setiap aplikasi mobile mengambil data dari server. Modul ini mengajarkan Anda cara melakukan request HTTP ke REST API, parsing JSON menjadi object Dart, dan menampilkannya di UI menggunakan FutureBuilder.

Buat Project Baru
Terminal

$ cd flutter_rest_api

# Tambahkan package http
$ flutter pub add http

AndroidManifest.xml
<uses-permission android:name="android.permission.INTERNET" />

Mengenal REST API

REST API adalah cara komunikasi antara aplikasi (client) dengan server menggunakan protokol HTTP. Setiap operasi punya HTTP method tersendiri:

  1. GET — Mengambil data (mis. daftar produk, detail user).
  2. POST — Menambahkan data baru (mis. menambahkan komentar).
  3. PUT / PATCH — Mengubah data yang sudah ada.
  4. DELETE — Menghapus data.
API untuk Latihan

Di modul ini kita akan menggunakan JSONPlaceholder — REST API publik gratis untuk latihan. API ini menyediakan endpoint dummy seperti /posts, /users, /comments, dll.

GET Request — Mengambil Data

Mari mulai dengan request paling sederhana: GET. Kita akan ambil daftar posts dari JSONPlaceholder.

contoh_get.dart
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<void> ambilData() async {
  final url = Uri.parse('https://jsonplaceholder.typicode.com/posts/1');
  final response = await http.get(url);

  if (response.statusCode == 200) {
    final data = jsonDecode(response.body);
    print('Judul: ${data['title']}');
    print('Isi: ${data['body']}');
  } else {
    print('Error: ${response.statusCode}');
  }
}

Parsing JSON ke Object Dart

Bekerja dengan Map<String, dynamic> langsung itu rawan typo dan tidak type-safe. Praktik terbaik: buat model class dengan factory constructor fromJson yang mengubah JSON menjadi object Dart yang nyaman.

post_model.dart
class Post {
  final int id;
  final String title;
  final String body;

  Post({required this.id, required this.title, required this.body});

  // Factory untuk parse dari JSON
  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      id: json['id'] as int,
      title: json['title'] as String,
      body: json['body'] as String,
    );
  }
}

Service Class — Memisahkan Logic API

Daripada menulis kode HTTP langsung di widget, lebih rapi memisahkannya ke service class. Pola ini membuat kode mudah dimaintain dan di-test.

post_service.dart
import 'dart:convert';
import 'package:http/http.dart' as http;

class PostService {
  static const _baseUrl = 'https://jsonplaceholder.typicode.com';

  // GET daftar posts
  Future<List<Post>> getPosts() async {
    final response = await http.get(Uri.parse('$_baseUrl/posts'));

    if (response.statusCode == 200) {
      final List<dynamic> list = jsonDecode(response.body);
      return list.map((j) => Post.fromJson(j)).toList();
    } else {
      throw Exception('Gagal memuat posts');
    }
  }

  // POST data baru
  Future<Post> createPost(String title, String body) async {
    final response = await http.post(
      Uri.parse('$_baseUrl/posts'),
      headers: {'Content-Type': 'application/json'},
      body: jsonEncode({'title': title, 'body': body, 'userId': 1}),
    );

    if (response.statusCode == 201) {
      return Post.fromJson(jsonDecode(response.body));
    } else {
      throw Exception('Gagal membuat post');
    }
  }
}

FutureBuilder — Menampilkan Hasil API

FutureBuilder adalah widget magis yang otomatis menangani 3 keadaan: loading, sukses, dan error. Anda tinggal memberi tahu builder bagaimana setiap keadaan ditampilkan.

future_builder.dart
FutureBuilder<List<Post>>(
  future: PostService().getPosts(),
  builder: (context, snapshot) {
    // 1. Loading
    if (snapshot.connectionState == ConnectionState.waiting) {
      return const Center(child: CircularProgressIndicator());
    }
    // 2. Error
    if (snapshot.hasError) {
      return Center(child: Text('Error: ${snapshot.error}'));
    }
    // 3. Sukses
    final posts = snapshot.data!;
    return ListView.builder(
      itemCount: posts.length,
      itemBuilder: (_, i) => ListTile(
        leading: CircleAvatar(child: Text('${posts[i].id}')),
        title: Text(posts[i].title),
        subtitle: Text(posts[i].body, maxLines: 2, overflow: TextOverflow.ellipsis),
      ),
    );
  },
)

Aplikasi Lengkap: Pembaca Postingan

Gabungkan semuanya menjadi satu aplikasi nyata. Ganti seluruh isi lib/main.dart di project flutter_rest_api dengan kode berikut:

lib/main.dart
import 'dart:convert';
import 'package:flutter/material.dart';
import 'package:http/http.dart' as http;

void main() => runApp(const MyApp());

class Post {
  final int id;
  final String title, body;
  Post({required this.id, required this.title, required this.body});
  factory Post.fromJson(Map<String, dynamic> j) =>
    Post(id: j['id'], title: j['title'], body: j['body']);
}

Future<List<Post>> fetchPosts() async {
  final r = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts'));
  if (r.statusCode != 200) throw Exception('Gagal: ${r.statusCode}');
  final list = jsonDecode(r.body) as List;
  return list.map((j) => Post.fromJson(j)).toList();
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});
  @override
  Widget build(BuildContext context) => MaterialApp(
    theme: ThemeData(colorSchemeSeed: Colors.blue, useMaterial3: true),
    home: const PostListPage(),
  );
}

class PostListPage extends StatefulWidget {
  const PostListPage({super.key});
  @override
  State<PostListPage> createState() => _PostListPageState();
}

class _PostListPageState extends State<PostListPage> {
  late Future<List<Post>> _future;

  @override
  void initState() {
    super.initState();
    _future = fetchPosts();
  }

  Future<void> _refresh() async {
    setState(() => _future = fetchPosts());
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('Pembaca Postingan'),
        actions: [IconButton(icon: const Icon(Icons.refresh), onPressed: _refresh)],
      ),
      body: RefreshIndicator(
        onRefresh: _refresh,
        child: FutureBuilder<List<Post>>(
          future: _future,
          builder: (context, snap) {
            if (snap.connectionState == ConnectionState.waiting) {
              return const Center(child: CircularProgressIndicator());
            }
            if (snap.hasError) {
              return Center(
                child: Column(
                  mainAxisAlignment: MainAxisAlignment.center,
                  children: [
                    const Icon(Icons.error_outline, size: 64, color: Colors.red),
                    const SizedBox(height: 8),
                    Text('${snap.error}'),
                    const SizedBox(height: 12),
                    ElevatedButton(onPressed: _refresh, child: const Text('Coba lagi')),
                  ],
                ),
              );
            }
            final posts = snap.data!;
            return ListView.separated(
              itemCount: posts.length,
              separatorBuilder: (_, __) => const Divider(height: 1),
              itemBuilder: (_, i) {
                final p = posts[i];
                return ListTile(
                  leading: CircleAvatar(child: Text('${p.id}')),
                  title: Text(p.title, maxLines: 1, overflow: TextOverflow.ellipsis),
                  subtitle: Text(p.body, maxLines: 2, overflow: TextOverflow.ellipsis),
                  onTap: () => showDialog(
                    context: context,
                    builder: (_) => AlertDialog(
                      title: Text(p.title),
                      content: Text(p.body),
                      actions: [TextButton(onPressed: () => Navigator.pop(context), child: const Text('OK'))],
                    ),
                  ),
                );
              },
            );
          },
        ),
      ),
    );
  }
}
RefreshIndicator

Widget RefreshIndicator menambahkan gesture pull-to-refresh ala Instagram/Twitter. User tinggal tarik konten ke bawah dan API akan dipanggil ulang. Sangat dianjurkan untuk semua list yang berasal dari API.

Hindari Memanggil API di build()

JANGAN tulis future: fetchPosts() langsung di build(). Setiap rebuild, request HTTP akan diulang. Pola yang benar adalah simpan Future di state (initState) seperti di contoh di atas, atau panggil melalui Provider.

Tantangan Mahasiswa

1. Tambahkan halaman DetailPostPage. Saat user mengklik item di list, navigasi ke halaman tersebut dan tampilkan judul + body lengkap (bukan dialog).

2. Buat halaman "Tambah Post Baru" dengan dua TextField (judul & body) dan tombol Submit yang memanggil POST ke https://jsonplaceholder.typicode.com/posts.

3. Tampilkan SnackBar hijau jika berhasil dan merah jika gagal.

4. Bonus: Gunakan endpoint /users dari JSONPlaceholder untuk membuat aplikasi "Direktori User" di project lain. Tampilkan nama, email, kota, dan website tiap user.
