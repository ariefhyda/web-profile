Modul 14: Build Aplikasi untuk Release (APK & AAB)

Estimasi: 90 Menit
Praktikum

Setelah aplikasi Anda diuji dan berjalan baik di emulator, langkah berikutnya adalah build versi rilis yang bisa di-install di perangkat siapa saja. Modul ini membahas cara mengganti nama aplikasi, ikon, splash screen, menandatangani aplikasi dengan keystore, dan menghasilkan dua format distribusi utama: APK (file langsung install) dan AAB (Android App Bundle, wajib untuk Play Store).

Project untuk Latihan

Berbeda dari modul sebelumnya, di sini Anda tidak perlu buat project baru. Pakai salah satu project dari modul sebelumnya — direkomendasikan flutter_widget_dasar (Modul 6) atau flutter_rest_api (Modul 10) karena UI-nya sudah cukup matang untuk diberi identitas rilis.

Terminal
$ cd flutter_widget_dasar
$ flutter clean && flutter pub get

APK vs AAB — Apa Bedanya?

  1. APK (Android Package Kit) — file .apk berisi seluruh aplikasi untuk semua arsitektur CPU dan resolusi layar. Ukurannya besar tapi bisa langsung di-install lewat file manager. Cocok untuk distribusi langsung (testing internal, side-loading).
  2. AAB (Android App Bundle) — file .aab berisi resources aplikasi dalam bentuk modular. Google Play akan otomatis menghasilkan APK ramping yang dioptimalkan untuk tiap device. Wajib untuk publish ke Play Store sejak Agustus 2021.

1. Ubah Nama Aplikasi

Nama aplikasi yang tampil di home screen Android diatur di android/app/src/main/AndroidManifest.xml.

AndroidManifest.xml
<application
    android:label="Galeri Saya"     <!-- Ubah jadi nama aplikasi Anda -->
    android:icon="@mipmap/ic_launcher"
    ...>

2. Ubah Application ID

Setiap aplikasi di Play Store harus punya Application ID unik (format reverse-domain: com.namaperusahaan.namaapp). ID ini tidak bisa diubah lagi setelah aplikasi pertama kali dipublish.

Buka android/app/build.gradle atau android/app/build.gradle.kts:

android/app/build.gradle
android {
    namespace "id.ac.kampus.galerisaya"
    compileSdk 34

    defaultConfig {
        applicationId "id.ac.kampus.galerisaya"   // ← UBAH INI
        minSdk 21
        targetSdk 34
        versionCode 1
        versionName "1.0.0"
    }
}
Konvensi Application ID

Gunakan domain yang Anda miliki/kontrol, dibalik. Contoh: jika kampus Anda kampus.ac.id, ID-nya menjadi id.ac.kampus.namaapp. Hindari pakai com.example.* — Play Store akan menolak.

3. Ganti Ikon Aplikasi

Ikon default Flutter berbentuk logo biru standar. Cara termudah menggantinya adalah dengan package flutter_launcher_icons.

Terminal
$ flutter pub add --dev flutter_launcher_icons

Buat folder assets/icon/ dan simpan ikon Anda dengan nama icon.png berukuran 1024×1024 px. Lalu tambahkan konfigurasi di pubspec.yaml:

pubspec.yaml
flutter_launcher_icons:
  android: "launcher_icon"
  ios: true
  image_path: "assets/icon/icon.png"
  min_sdk_android: 21
  adaptive_icon_background: "#FFFFFF"
  adaptive_icon_foreground: "assets/icon/icon_foreground.png"

Generate ikon dengan perintah:

Terminal
$ dart run flutter_launcher_icons

Tool ini otomatis akan menghasilkan ikon dalam berbagai resolusi (mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi) dan menempatkannya di folder yang tepat. Jalankan ulang flutter run untuk melihat hasilnya.

4. Splash Screen — Tampilan Saat Loading

Splash screen adalah layar yang muncul saat aplikasi pertama kali dibuka, sebelum widget pertama selesai di-render. Tanpa konfigurasi, layar ini hanya menampilkan background putih. Gunakan package flutter_native_splash untuk membuatnya cantik.

Terminal
$ flutter pub add --dev flutter_native_splash
pubspec.yaml
flutter_native_splash:
  color: "#1a1a2e"                          # warna background
  image: "assets/icon/splash_logo.png"     # gambar tengah (512×512)
  android: true
  ios: true
  android_12:
    color: "#1a1a2e"
    image: "assets/icon/splash_logo.png"
Terminal
$ dart run flutter_native_splash:create

5. Versioning — versionCode & versionName

Setiap update aplikasi yang Anda upload ke Play Store wajib memiliki versionCode lebih tinggi dari versi sebelumnya. Atur di pubspec.yaml (Flutter akan sinkronkan otomatis ke build.gradle):

pubspec.yaml
version: 1.0.0+1
#       └──┬──┘ └┬┘
#           │     └── versionCode (angka, naik tiap rilis)
#           └──────── versionName (yang dilihat user)

6. Buat Keystore — Tanda Tangan Aplikasi

Aplikasi Android wajib ditandatangani sebelum bisa di-install di luar mode debug. Keystore adalah file kunci kriptografi yang membuktikan aplikasi benar-benar dari Anda. JANGAN PERNAH KEHILANGAN FILE INI — tanpa keystore yang sama, Anda tidak bisa mengupdate aplikasi yang sudah ada di Play Store.

Terminal
# Buat folder untuk menyimpan keystore (di luar project lebih baik)
$ keytool -genkey -v -keystore ~/upload-keystore.jks \
    -keyalg RSA -keysize 2048 -validity 10000 \
    -alias upload

# Anda akan diminta:
#  - password keystore (minimal 6 karakter)
#  - nama lengkap Anda
#  - organisasi, kota, provinsi, kode negara (ID untuk Indonesia)

Setelah keystore dibuat, daftarkan kredensialnya di file android/key.properties:

android/key.properties
storePassword=password_keystore_anda
keyPassword=password_key_anda
keyAlias=upload
storeFile=/Users/nama_anda/upload-keystore.jks
JANGAN Commit key.properties & .jks ke Git!

Tambahkan ke .gitignore:

.gitignore
**/android/key.properties
**/*.jks
**/*.keystore

Simpan file .jks di tempat aman (Google Drive privat, password manager) + catat password-nya. Kehilangan keystore = harus publish aplikasi baru dari nol.

7. Konfigurasi Signing di build.gradle

Buka android/app/build.gradle dan tambahkan blok signingConfigs sebelum buildTypes. Sesuaikan juga release di buildTypes agar memakai signing config tersebut.

android/app/build.gradle
// Tambahkan di paling atas, sebelum 'android { ... }'
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    ...

    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release      // ← UBAH dari .debug
            minifyEnabled true
            shrinkResources true
        }
    }
}

8. Build APK

Sekarang saatnya menghasilkan file rilis. Untuk APK (cocok untuk testing langsung di HP):

Terminal
# APK universal (1 file, semua arsitektur — ukurannya besar)
$ flutter build apk --release

# APK terpisah per arsitektur — ukuran lebih kecil per file
$ flutter build apk --release --split-per-abi

# Output:
#   build/app/outputs/flutter-apk/app-release.apk
#   build/app/outputs/flutter-apk/app-armeabi-v7a-release.apk
#   build/app/outputs/flutter-apk/app-arm64-v8a-release.apk
#   build/app/outputs/flutter-apk/app-x86_64-release.apk
Install APK ke HP

Pindahkan file app-release.apk ke HP via USB/Drive/WhatsApp, lalu tap untuk install. HP Anda mungkin akan minta izin "Install from unknown sources" — pastikan ini diaktifkan di Settings.

9. Build AAB untuk Play Store

Untuk Play Store, format yang wajib adalah App Bundle (.aab):

Terminal
$ flutter build appbundle --release

# Output:
#   build/app/outputs/bundle/release/app-release.aab

File .aab ini yang akan Anda upload ke Google Play Console (dibahas di modul berikutnya).

10. Verifikasi Hasil Build

Sebelum mendistribusikan, lakukan verifikasi cepat:

  1. Cek tanda tangan — pastikan APK ditandatangani dengan keystore Anda, bukan debug keystore.
  2. Install di device fisik — emulator tidak selalu mencerminkan device nyata.
  3. Coba semua fitur utama — release build menghapus log dan optimasi berbeda dari debug.
  4. Cek ukuran — AAB biasanya 30–40% lebih kecil dari APK universal.
Terminal
# Cek signature APK
$ keytool -printcert -jarfile build/app/outputs/flutter-apk/app-release.apk

# Install APK ke device yang terhubung via USB
$ flutter install --release

# Cek ukuran file
$ ls -lh build/app/outputs/flutter-apk/
$ ls -lh build/app/outputs/bundle/release/

Singgah Sebentar: Build untuk iOS

Build untuk iOS perlu komputer Mac dan akun Apple Developer (USD 99/tahun). Perintah dasarnya:

Terminal (Mac saja)
$ flutter build ipa --release

Setelah itu, archive aplikasi di Xcode, lalu upload ke App Store Connect. Karena tidak semua mahasiswa punya Mac, materi iOS di-skip di matakuliah ini.

Troubleshooting Umum

Error "Keystore file not found" — pastikan path di key.properties menggunakan path absolut, bukan relatif. Di Windows gunakan slash / bukan \.

Aplikasi crash di release tapi normal di debug — biasanya karena minifyEnabled. Tambahkan rule ProGuard di android/app/proguard-rules.pro, atau matikan sementara minify.

APK terlalu besar (>50 MB) — pakai --split-per-abi, hapus asset tidak terpakai, atau gunakan AAB.

Tantangan Mahasiswa

1. Pilih salah satu project Flutter Anda. Ubah nama aplikasi, applicationId, ikon, dan splash screen sesuai identitas Anda sendiri.

2. Buat keystore upload (simpan password-nya!). Konfigurasi signing di build.gradle.

3. Hasilkan 3 file: APK universal, APK split-per-abi (arm64-v8a saja), dan AAB. Catat ukuran masing-masing dan bandingkan.

4. Install APK arm64-v8a di HP Anda. Pastikan ikon, splash, dan nama aplikasi sudah sesuai.

5. Bonus: Coba aktifkan --obfuscate --split-debug-info=build/symbols saat build untuk menambah keamanan dari reverse-engineering.
