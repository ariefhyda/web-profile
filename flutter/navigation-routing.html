Modul 8: Navigation & Routing

Estimasi: 75 Menit
Teori & Praktikum

Aplikasi nyata selalu terdiri dari banyak halaman: login, beranda, detail produk, profil, dan seterusnya. Modul ini mengajarkan cara berpindah halaman di Flutter — dari yang paling sederhana hingga teknik named routes yang scalable.

Buat Project Baru
Terminal
$ flutter create flutter_navigation
$ cd flutter_navigation
$ flutter run

Konsep Stack Navigation

Flutter mengelola halaman menggunakan struktur stack (tumpukan). Saat Anda membuka halaman baru, halaman tersebut didorong (push) ke atas tumpukan. Saat Anda menekan tombol back, halaman teratas dilepas (pop) dari tumpukan.

stack_concept.txt
[ DetailPage ]  ← Halaman teratas (terlihat)
[ ListPage   ]
[ HomePage   ]  ← Halaman pertama (di paling bawah)

Navigator.push & Navigator.pop

Cara paling sederhana untuk berpindah halaman adalah dengan Navigator.push. Untuk kembali, gunakan Navigator.pop — atau biarkan tombol back perangkat melakukannya secara otomatis.

push_pop.dart
// Pindah ke halaman baru
ElevatedButton(
  onPressed: () {
    Navigator.push(
      context,
      MaterialPageRoute(builder: (context) => const HalamanKedua()),
    );
  },
  child: const Text('Buka Halaman Kedua'),
)

// Kembali ke halaman sebelumnya
ElevatedButton(
  onPressed: () => Navigator.pop(context),
  child: const Text('Kembali'),
)

Contoh Lengkap: Aplikasi 2 Halaman

Mari kita buat aplikasi sederhana dengan dua halaman: HomePage dan AboutPage. Ganti seluruh isi lib/main.dart dengan kode berikut:

lib/main.dart
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.deepPurple, useMaterial3: true),
      home: const HomePage(),
    );
  }
}

class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Beranda')),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (_) => const AboutPage()),
            );
          },
          child: const Text('Tentang Aplikasi →'),
        ),
      ),
    );
  }
}

class AboutPage extends StatelessWidget {
  const AboutPage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Tentang')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            const Text('Aplikasi ini dibuat untuk belajar Flutter 🚀'),
            const SizedBox(height: 16),
            ElevatedButton(
              onPressed: () => Navigator.pop(context),
              child: const Text('← Kembali'),
            ),
          ],
        ),
      ),
    );
  }
}

Mengirim Data ke Halaman Baru

Sangat umum kita ingin mengirim data dari satu halaman ke halaman lain (misalnya ID produk, nama user). Caranya: kirim sebagai parameter constructor halaman tujuan.

passing_data.dart
// Halaman tujuan menerima parameter
class DetailPage extends StatelessWidget {
  final String namaProduk;
  final int harga;

  const DetailPage({super.key, required this.namaProduk, required this.harga});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(namaProduk)),
      body: Center(child: Text('Harga: Rp $harga')),
    );
  }
}

// Halaman pemanggil mengirim data
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (_) => const DetailPage(namaProduk: 'Sepatu Lari', harga: 450000),
  ),
);

Mengembalikan Data ke Halaman Sebelumnya

Pola sebaliknya juga sering dipakai. Misalnya halaman "Pilih Kota" mengembalikan kota yang dipilih user. Navigator.push mengembalikan Future yang resolve saat halaman tujuan di-pop dengan data.

return_data.dart
// Halaman pemanggil — tunggu hasilnya
final hasil = await Navigator.push<String>(
  context,
  MaterialPageRoute(builder: (_) => const PilihKotaPage()),
);

if (hasil != null) {
  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(content: Text('Kamu memilih: $hasil')),
  );
}

// Halaman PilihKotaPage — kembalikan data lewat pop
ListTile(
  title: const Text('Bandung'),
  onTap: () => Navigator.pop(context, 'Bandung'),
)
async / await pada Navigator.push

Karena Navigator.push mengembalikan Future, function yang memanggilnya harus ditandai async dan menggunakan await. Konsep ini sudah Anda pelajari di modul Collection & Async.

Named Routes — Navigasi yang Lebih Rapi

Jika aplikasi Anda punya banyak halaman, named routes jauh lebih rapi. Daftarkan seluruh rute di MaterialApp.routes, lalu panggil dengan Navigator.pushNamed.

named_routes.dart
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (_) => const HomePage(),
    '/about': (_) => const AboutPage(),
    '/settings': (_) => const SettingsPage(),
  },
)

// Cara memanggilnya
Navigator.pushNamed(context, '/about');

// Replace (tidak bisa di-back, cocok untuk login → home)
Navigator.pushReplacementNamed(context, '/home');

// Pop sampai ke route tertentu
Navigator.popUntil(context, ModalRoute.withName('/'));

BottomNavigationBar — Navigasi Tab

Banyak aplikasi (Instagram, WhatsApp, Tokopedia) menggunakan bottom navigation untuk berpindah antar halaman utama. Implementasinya cukup sederhana — gunakan StatefulWidget untuk menyimpan index tab aktif.

bottom_nav.dart
class MainScaffold extends StatefulWidget {
  const MainScaffold({super.key});
  @override
  State<MainScaffold> createState() => _MainScaffoldState();
}

class _MainScaffoldState extends State<MainScaffold> {
  int _index = 0;

  final _pages = const [
    Center(child: Text('🏠 Beranda', style: TextStyle(fontSize: 24))),
    Center(child: Text('🔍 Cari', style: TextStyle(fontSize: 24))),
    Center(child: Text('👤 Profil', style: TextStyle(fontSize: 24))),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: _pages[_index],
      bottomNavigationBar: NavigationBar(
        selectedIndex: _index,
        onDestinationSelected: (i) => setState(() => _index = i),
        destinations: const [
          NavigationDestination(icon: Icon(Icons.home), label: 'Beranda'),
          NavigationDestination(icon: Icon(Icons.search), label: 'Cari'),
          NavigationDestination(icon: Icon(Icons.person), label: 'Profil'),
        ],
      ),
    );
  }
}

Drawer — Menu Samping

Drawer adalah menu samping yang muncul dengan swipe atau klik tombol menu di AppBar. Cocok untuk pengaturan, profil, daftar shortcut, dll.

drawer.dart
Scaffold(
  appBar: AppBar(title: const Text('Beranda')),
  drawer: Drawer(
    child: ListView(
      children: [
        const DrawerHeader(
          decoration: BoxDecoration(color: Colors.deepPurple),
          child: Text('Halo, Arif!', style: TextStyle(color: Colors.white, fontSize: 20)),
        ),
        ListTile(
          leading: const Icon(Icons.settings),
          title: const Text('Pengaturan'),
          onTap: () => Navigator.pushNamed(context, '/settings'),
        ),
        ListTile(
          leading: const Icon(Icons.logout),
          title: const Text('Keluar'),
          onTap: () {},
        ),
      ],
    ),
  ),
  body: const Center(child: Text('Konten halaman')),
)
Tantangan Mahasiswa

Buat aplikasi Katalog Produk di project ini:

1. HomePage menampilkan ListView berisi 5 produk (nama + harga).

2. Klik salah satu produk → buka DetailPage dengan passing data nama dan harga.

3. Di DetailPage, ada tombol "Pilih Produk Ini" yang melakukan Navigator.pop mengembalikan nama produk.

4. HomePage menampilkan SnackBar berisi nama produk yang dipilih.
