Modul 11: Local Storage — SharedPreferences & SQLite

Estimasi: 90 Menit
Teori & Praktikum

Aplikasi profesional harus bisa menyimpan data secara lokal agar tetap tersedia saat offline atau setelah aplikasi ditutup. Modul ini membahas dua teknik penyimpanan lokal yang paling sering dipakai: SharedPreferences untuk data kecil (key-value), dan SQLite untuk data terstruktur.

Buat Project Baru
Terminal
$ flutter create flutter_local_storage
$ cd flutter_local_storage

# Untuk SharedPreferences
$ flutter pub add shared_preferences

# Untuk SQLite
$ flutter pub add sqflite path

Memilih Storage yang Tepat

Sebelum coding, pahami kapan harus pakai apa:

  1. SharedPreferences — Untuk data kecil berbentuk key-value: pengaturan tema, token login, flag "sudah lihat onboarding", last opened page. Cepat & mudah, tapi tidak cocok untuk data kompleks.
  2. SQLite (sqflite) — Database relasional. Cocok untuk data terstruktur dengan banyak field dan relasi: catatan, daftar transaksi, kontak, todo dengan kategori, dll. CRUD penuh dengan query SQL.
  3. Hive / Isar / ObjectBox — Database NoSQL berbasis object. Lebih cepat dari SQLite untuk skenario tertentu. (Tidak dibahas di modul ini.)

SharedPreferences — Key-Value Sederhana

Bayangkan SharedPreferences sebagai Map persisten yang otomatis disimpan ke disk. Mendukung tipe primitif: int, double, bool, String, dan List<String>.

shared_prefs_demo.dart
import 'package:shared_preferences/shared_preferences.dart';

// SIMPAN
Future<void> simpanNama(String nama) async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.setString('user_nama', nama);
  await prefs.setBool('sudah_login', true);
  await prefs.setInt('umur', 21);
}

// BACA (kembalian bisa null jika belum pernah disimpan)
Future<void> muatNama() async {
  final prefs = await SharedPreferences.getInstance();
  final nama = prefs.getString('user_nama') ?? 'Belum diisi';
  final sudahLogin = prefs.getBool('sudah_login') ?? false;
  print('Nama: $nama, Login: $sudahLogin');
}

// HAPUS
Future<void> logout() async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.remove('user_nama');
  // atau hapus semua:
  await prefs.clear();
}
Use-case Klasik SharedPreferences

Theme switcher: simpan bool isDarkMode agar pilihan user tidak hilang setelah aplikasi ditutup.

Token login: simpan JWT token, baca saat aplikasi dibuka, otomatis arahkan ke Home jika ada.

Onboarding flag: simpan bool sudahLihatTutorial agar tutorial hanya muncul sekali di first launch.

SQLite dengan sqflite

SQLite adalah database relasional ringan yang sudah ada di setiap perangkat Android & iOS. Kita mengaksesnya melalui package sqflite. Konsep dasarnya sama dengan database biasa: ada tabel, kolom, dan kita menjalankan query SQL.

Membuat Model — Catatan

Sebagai contoh, kita akan membangun aplikasi "Buku Catatan". Setiap catatan punya id, judul, isi, dan createdAt.

note_model.dart
class Note {
  final int? id;        // null saat catatan baru, auto-fill setelah insert
  final String judul;
  final String isi;
  final String createdAt;

  Note({this.id, required this.judul, required this.isi, required this.createdAt});

  // Konversi ke Map untuk insert/update
  Map<String, dynamic> toMap() => {
    'id': id,
    'judul': judul,
    'isi': isi,
    'createdAt': createdAt,
  };

  // Konversi dari Map (hasil query)
  factory Note.fromMap(Map<String, dynamic> m) => Note(
    id: m['id'] as int?,
    judul: m['judul'] as String,
    isi: m['isi'] as String,
    createdAt: m['createdAt'] as String,
  );
}

DatabaseHelper — Singleton untuk Akses DB

Daripada membuka koneksi database di setiap halaman, kita buat satu class DatabaseHelper yang bertanggung jawab untuk inisialisasi dan operasi CRUD.

database_helper.dart
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

class DatabaseHelper {
  static final DatabaseHelper instance = DatabaseHelper._();
  DatabaseHelper._();

  static Database? _db;

  Future<Database> get database async {
    if (_db != null) return _db!;
    _db = await _initDb();
    return _db!;
  }

  Future<Database> _initDb() async {
    final path = join(await getDatabasesPath(), 'notes.db');
    return openDatabase(
      path,
      version: 1,
      onCreate: (db, version) {
        return db.execute('''
          CREATE TABLE notes (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            judul TEXT NOT NULL,
            isi TEXT NOT NULL,
            createdAt TEXT NOT NULL
          )
        ''');
      },
    );
  }

  // CREATE
  Future<int> insert(Note note) async {
    final db = await database;
    return db.insert('notes', note.toMap());
  }

  // READ semua
  Future<List<Note>> getAll() async {
    final db = await database;
    final maps = await db.query('notes', orderBy: 'id DESC');
    return maps.map(Note.fromMap).toList();
  }

  // UPDATE
  Future<int> update(Note note) async {
    final db = await database;
    return db.update(
      'notes',
      note.toMap(),
      where: 'id = ?',
      whereArgs: [note.id],
    );
  }

  // DELETE
  Future<int> delete(int id) async {
    final db = await database;
    return db.delete('notes', where: 'id = ?', whereArgs: [id]);
  }
}
Kenapa Singleton?

Membuka koneksi database itu mahal. Pola singleton memastikan hanya ada satu instance Database selama aplikasi berjalan. Trik DatabaseHelper._() (constructor private) + static final instance adalah idiom Dart standar untuk singleton.

Aplikasi Lengkap: Buku Catatan

Sekarang gabungkan semuanya. Untuk kemudahan, kita akan menulis semua kode dalam satu file lib/main.dart. Di aplikasi nyata, sebaiknya dipecah menjadi beberapa file: note_model.dart, database_helper.dart, dan halaman-halaman terpisah.

lib/main.dart
import 'package:flutter/material.dart';
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

void main() => runApp(const MyApp());

// === MODEL ===
class Note {
  final int? id;
  final String judul, isi, createdAt;
  Note({this.id, required this.judul, required this.isi, required this.createdAt});
  Map<String, dynamic> toMap() => {'id': id, 'judul': judul, 'isi': isi, 'createdAt': createdAt};
  factory Note.fromMap(Map<String, dynamic> m) =>
    Note(id: m['id'], judul: m['judul'], isi: m['isi'], createdAt: m['createdAt']);
}

// === DATABASE HELPER ===
class DB {
  static final DB instance = DB._();
  DB._();
  static Database? _db;

  Future<Database> get db async {
    if (_db != null) return _db!;
    final path = join(await getDatabasesPath(), 'notes.db');
    _db = await openDatabase(path, version: 1, onCreate: (d, _) {
      return d.execute('CREATE TABLE notes(id INTEGER PRIMARY KEY AUTOINCREMENT, judul TEXT, isi TEXT, createdAt TEXT)');
    });
    return _db!;
  }

  Future<int> insert(Note n) async => (await db).insert('notes', n.toMap());
  Future<List<Note>> all() async =>
    (await (await db).query('notes', orderBy: 'id DESC')).map(Note.fromMap).toList();
  Future<int> delete(int id) async =>
    (await db).delete('notes', where: 'id=?', whereArgs: [id]);
}

// === UI ===
class MyApp extends StatelessWidget {
  const MyApp({super.key});
  @override
  Widget build(BuildContext context) => MaterialApp(
    theme: ThemeData(colorSchemeSeed: Colors.amber, useMaterial3: true),
    home: const NotePage(),
  );
}

class NotePage extends StatefulWidget {
  const NotePage({super.key});
  @override
  State<NotePage> createState() => _NotePageState();
}

class _NotePageState extends State<NotePage> {
  List<Note> _notes = [];

  @override
  void initState() { super.initState(); _load(); }

  Future<void> _load() async {
    final data = await DB.instance.all();
    setState(() => _notes = data);
  }

  Future<void> _showForm() async {
    final judulCtrl = TextEditingController();
    final isiCtrl = TextEditingController();

    final simpan = await showDialog<bool>(
      context: context,
      builder: (_) => AlertDialog(
        title: const Text('Catatan Baru'),
        content: Column(mainAxisSize: MainAxisSize.min, children: [
          TextField(controller: judulCtrl, decoration: const InputDecoration(labelText: 'Judul')),
          TextField(controller: isiCtrl, decoration: const InputDecoration(labelText: 'Isi'), maxLines: 3),
        ]),
        actions: [
          TextButton(onPressed: () => Navigator.pop(context, false), child: const Text('Batal')),
          FilledButton(onPressed: () => Navigator.pop(context, true), child: const Text('Simpan')),
        ],
      ),
    );

    if (simpan == true && judulCtrl.text.isNotEmpty) {
      await DB.instance.insert(Note(
        judul: judulCtrl.text,
        isi: isiCtrl.text,
        createdAt: DateTime.now().toIso8601String().substring(0, 16),
      ));
      _load();
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Buku Catatan')),
      body: _notes.isEmpty
        ? const Center(child: Text('Belum ada catatan. Tekan + untuk menambah'))
        : ListView.builder(
            itemCount: _notes.length,
            itemBuilder: (_, i) {
              final n = _notes[i];
              return Dismissible(
                key: ValueKey(n.id),
                background: Container(color: Colors.red, alignment: Alignment.centerRight,
                  padding: const EdgeInsets.only(right: 16),
                  child: const Icon(Icons.delete, color: Colors.white)),
                onDismissed: (_) async {
                  await DB.instance.delete(n.id!);
                  _load();
                },
                child: ListTile(
                  title: Text(n.judul, style: const TextStyle(fontWeight: FontWeight.bold)),
                  subtitle: Text('${n.isi}\n📅 ${n.createdAt}'),
                  isThreeLine: true,
                ),
              );
            },
          ),
      floatingActionButton: FloatingActionButton(
        onPressed: _showForm,
        child: const Icon(Icons.add),
      ),
    );
  }
}
Dismissible — Swipe to Delete

Widget Dismissible menambahkan gesture swipe-to-delete ala Gmail. Setiap item memerlukan key unik (gunakan id dari database). Saat user swipe, callback onDismissed akan dipanggil.

Saat Skema Tabel Berubah

Jika Anda mengubah CREATE TABLE (mis. menambah kolom kategori), aplikasi yang sudah pernah di-run tidak akan otomatis mengupdate skema. Cara cepat saat development: uninstall aplikasi dari emulator/device, lalu flutter run lagi. Cara profesional: gunakan onUpgrade di openDatabase dan naikkan parameter version.
