Modul 2: Instalasi & Konfigurasi Laravel

Estimasi: 60 Menit
Setup & Praktikum

Sebelum mulai membangun aplikasi, kita perlu menyiapkan environment development. Modul ini memandu instalasi PHP, Composer, Laravel, dan konfigurasi database MySQL dari nol sampai server berjalan.

🧰
Analogi: Menyiapkan Bengkel

Sebelum bisa membuat mebel, tukang kayu harus punya gergaji, palu, paku, dan meja kerja. Begitu juga programmer Laravel — sebelum coding, harus install PHP (kayunya), Composer (palunya), MySQL (lem & paku), dan editor (meja kerja). Tanpa salah satu, project tidak bisa jalan.

Prasyarat (Prerequisites)

Centang tools berikut saat Anda sudah menginstalnya. Progress bar akan menunjukkan kesiapan Anda untuk mulai coding.

Checklist Persiapan Environment

Klik setiap item ketika sudah terinstal. Progress bar akan menunjukkan seberapa siap workstation Anda.

  • PHP ≥ 8.1 — Laravel 10/11 butuh PHP 8.1+. Cek dengan php --version. Ekstensi wajib: mbstring, xml, curl, zip, pdo_mysql.
  • Composer — Dependency manager PHP. Download dari getcomposer.org. Cek dengan composer --version.
  • MySQL / MariaDB — Database. Bisa via Laragon, XAMPP, atau MySQL standalone. Cek service jalan di port 3306.
  • Code Editor — Direkomendasikan VS Code + ekstensi: PHP Intelephense, Laravel Blade Snippets, DotENV.
  • Node.js & NPM — Bundling asset frontend (Vite, Tailwind). Cek dengan node -v dan npm -v.

Rekomendasi: Laragon (Windows)

Untuk pengguna Windows, sangat direkomendasikan Laragon — sebuah portable development environment yang sudah menyertakan PHP, MySQL, Apache/Nginx, dan Composer. Satu klik install, langsung bisa coding Laravel tanpa konfigurasi manual. Alternatif lain: Herd (Mac & Windows), XAMPP, WAMP.

Langkah 1: Cek PHP & Install Composer

Sebelum install Laravel, pastikan PHP dan Composer sudah tersedia. Jika belum punya Composer, download dari getcomposer.org.

terminal 
# Cek versi PHP (minimal 8.1)
php --version

# Cek versi Composer
composer --version

# Lihat ekstensi PHP yang aktif
php -m

Langkah 2: Membuat Project Laravel Baru

Ada dua cara membuat project: via Composer (paling umum) atau via Laravel Installer (lebih singkat untuk multiple project).

terminal 
# Cara 1: Via Composer (rekomendasi pemula)
composer create-project laravel/laravel nama-project

# Cara 2: Via Laravel Installer (lebih cepat untuk multi-project)
composer global require laravel/installer
laravel new nama-project

# Masuk ke folder project
cd nama-project

# Jalankan development server
php artisan serve
# → Server berjalan di http://localhost:8000
Tanda Berhasil

Buka browser ke http://localhost:8000 — Anda akan melihat halaman welcome Laravel berwarna ungu. Selamat, project Anda sudah hidup!

Langkah 3: Konfigurasi Database

Buka file .env di root project. File ini berisi environment variables — pengaturan yang berbeda untuk lokal dan production (kunci API, password DB, dll). File ini tidak boleh di-commit ke Git.

.env 
APP_NAME=Laravel
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nama_database
DB_USERNAME=root
DB_PASSWORD=

# Pastikan database sudah dibuat di MySQL terlebih dahulu:
# CREATE DATABASE nama_database;
Jangan Push .env ke Git!

File .env berisi rahasia (password DB, API key). Laravel sudah meng-ignore-nya secara default lewat .gitignore. Yang dicommit hanya .env.example sebagai template — tanpa nilai sebenarnya.

Langkah 4: Generate Key & Jalankan Migration

terminal 
# Generate application key (untuk enkripsi session, cookie)
php artisan key:generate

# Jalankan migrasi bawaan Laravel (users, password_resets, dll)
php artisan migrate

# Cek status migrasi
php artisan migrate:status

Troubleshooting Umum

⚠ Page Expired (419)

Penyebab: Form POST/PUT/DELETE tidak menyertakan @csrf.
Solusi: Tambahkan @csrf di dalam tag <form>. Jika tetap error, jalankan php artisan cache:clear.

⚠ SQLSTATE[HY000] [2002] Connection refused

Penyebab: MySQL belum jalan, atau konfigurasi .env salah.
Solusi: Pastikan MySQL/MariaDB hidup. Cek DB_HOST (biasanya 127.0.0.1, bukan localhost di beberapa sistem). Jalankan php artisan config:clear setelah edit .env.

⚠ Class "App\Models\X" not found

Penyebab: Autoload Composer belum di-refresh.
Solusi: Jalankan composer dump-autoload. Pastikan namespace di file Model sesuai (App\Models).

⚠ Perubahan .env tidak berlaku

Penyebab: Laravel cache config di production.
Solusi: Jalankan php artisan config:clear dan restart server (php artisan serve Ctrl+C lalu jalankan lagi).
