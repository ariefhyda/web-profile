Modul 5: Middleware

Estimasi: 40 Menit
Teori & Praktikum

Middleware adalah lapisan filter yang harus dilalui setiap HTTP request sebelum sampai ke Controller. Bayangkan ia sebagai penjaga gerbang: setiap request diperiksa — jika lolos, request lanjut; jika tidak, ditolak (di-redirect atau di-block).

🛂
Analogi: Pemeriksaan Imigrasi Bandara

Sebelum masuk ke negara tujuan (Controller), setiap penumpang (Request) melewati beberapa pos: cek paspor, scan barang, pemeriksaan dokumen. Kalau ada yang gagal di salah satu pos, penumpang ditahan (redirect). Kalau semua lolos, baru bisa masuk. Setiap pos = satu middleware.

Kapan Middleware Digunakan?

  1. Authentication — Memastikan user sudah login sebelum mengakses dashboard.
  2. Authorization / Role — Hanya admin yang boleh akses halaman admin.
  3. CSRF Protection — Verifikasi token form (sudah aktif default).
  4. Logging — Catat semua request masuk untuk audit.
  5. Locale — Set bahasa berdasarkan IP atau pilihan user.
  6. Rate Limiting — Batasi jumlah request per IP per menit (anti-brute-force).
  7. Maintenance Mode — Saat aplikasi sedang di-update, semua user lihat halaman maintenance.

Membuat Middleware Sendiri

Untuk membuat middleware baru, gunakan Artisan make:middleware:

Terminal 
php artisan make:middleware CheckAge

Perintah di atas membuat class CheckAge.php di app/Http/Middleware/. Contoh middleware untuk membatasi akses berdasarkan usia:

app/Http/Middleware/CheckAge.php 
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Symfony\Component\HttpFoundation\Response;

class CheckAge
{
    public function handle(Request $request, Closure $next): Response
    {
        // Logika: kalau usia < 18 → tolak, redirect ke /home
        if ($request->age < 18) {
            return redirect('home');
        }

        // Lolos? Teruskan ke middleware/controller berikutnya
        return $next($request);
    }
}
Pola: Before vs After

Kode sebelum $next($request) dijalankan saat request masuk (before). Kode setelah dijalankan saat response keluar (after). Berguna untuk logging response time, modifikasi header, dll.

middleware before-after 
public function handle(Request $request, Closure $next)
{
    // === BEFORE: dijalankan saat request masuk ===
    $start = microtime(true);

    $response = $next($request); // → controller dijalankan di sini

    // === AFTER: dijalankan setelah controller selesai ===
    $duration = microtime(true) - $start;
    Log::info("Request took $duration sec");

    return $response;
}

Mendaftarkan Middleware (Laravel 11+)

Di Laravel 11, pendaftaran middleware dilakukan di bootstrap/app.php (sebelumnya di app/Http/Kernel.php). Beri alias agar mudah dipakai:

bootstrap/app.php 
use App\Http\Middleware\CheckAge;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(...)
    ->withMiddleware(function (Middleware $middleware) {
        // Alias — bisa dipakai dengan ->middleware('cek.umur')
        $middleware->alias([
            'cek.umur' => CheckAge::class
        ]);

        // Global — diterapkan ke SEMUA request
        $middleware->append(function ($request, $next) {
            // contoh: header tracking
            return $next($request);
        });
    })
    ->create();

Menerapkan Middleware di Route

routes/web.php 
// Cara 1: satu route
Route::get('/admin/profile', [AdminController::class, 'index'])
    ->middleware('cek.umur');

// Cara 2: multiple middleware (urutan = urutan eksekusi)
Route::get('/admin', ...)
    ->middleware(['auth', 'verified', 'cek.umur']);

// Cara 3: route group (paling sering dipakai)
Route::middleware(['auth', 'cek.umur'])->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index']);
    Route::get('/settings', [SettingsController::class, 'index']);
});

// Cara 4: middleware dengan parameter
Route::get('/admin/posts', ...)
    ->middleware('role:admin'); // parameter setelah titik dua

Middleware Bawaan yang Sering Dipakai

AliasFungsi
authMengharuskan user login. Belum login → redirect ke /login
guestKebalikan auth — hanya boleh diakses user yang belum login (mis. halaman login)
verifiedUser harus sudah verifikasi email
throttle:60,1Maksimal 60 request per menit per IP
signedMemvalidasi URL bertanda tangan (untuk link email)
password.confirmWajib konfirmasi password ulang (mis. sebelum delete akun)
Urutan Middleware itu Penting!

Jika Anda menulis ['verified', 'auth'], middleware verified akan jalan duluan — padahal ia butuh user yang sudah login. Hasilnya error. Urutan benar: ['auth', 'verified'] — login dulu, baru cek verifikasi.
