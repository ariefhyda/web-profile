Modul 10: Eloquent Relationship

Estimasi: 60 Menit
Teori & Praktikum

Tabel-tabel dalam database jarang berdiri sendiri — mereka saling terhubung. Post punya banyak comment, user punya banyak order, book ditulis oleh banyak author. Eloquent menyediakan API yang elegan untuk mendefinisikan dan menavigasi relasi ini tanpa harus menulis JOIN SQL manual.

👨‍👩‍👧
Analogi: Pohon Keluarga

Bayangkan database sebagai silsilah keluarga. Setiap orang (record) punya hubungan: anak dari, suami dari, paman dari. Tanpa relasi, Anda harus menjelaskan satu per satu siapa anak siapa. Dengan relasi, cukup tunjukkan pohon — semua hubungan terbaca otomatis.

Tipe-tipe Relasi Eloquent

Pilih tipe untuk melihat detailnya.

1. One To One (Satu ke Satu)

Contoh: Satu User memiliki satu Profile. Satu Negara punya satu Ibu Kota.

User.php & Profile.php 
// Migration: profiles punya kolom user_id (foreign key)
$table->foreignId('user_id')->constrained()->cascadeOnDelete();

// app/Models/User.php — sisi "punya" (hasOne)
class User extends Model
{
    public function profile()
    {
        return $this->hasOne(Profile::class);
    }
}

// app/Models/Profile.php — sisi "dimiliki" (belongsTo)
class Profile extends Model
{
    public function user()
    {
        return $this->belongsTo(User::class);
    }
}

// Pemakaian:
$user = User::find(1);
echo $user->profile->bio; // auto join, ambil bio

$profile = Profile::find(1);
echo $profile->user->name; // kebalikannya

2. One To Many (Satu ke Banyak)

Contoh: Satu Post punya banyak Comment. Satu User punya banyak Order.

Post.php & Comment.php 
// Post.php — sisi "punya banyak"
public function comments()
{
    return $this->hasMany(Comment::class);
}

// Comment.php — sisi "milik"
public function post()
{
    return $this->belongsTo(Post::class);
}

// Pemakaian:
$post = Post::find(1);
foreach ($post->comments as $comment) {  // Collection
    echo $comment->text;
}

// Filter pada relasi
$approved = $post->comments()->where('approved', true)->get();

// Hitung tanpa load semua
$total = $post->comments()->count();
$totalCached = $post->comments_count; // pakai withCount(), lebih cepat

3. Many To Many (Banyak ke Banyak)

Contoh: Satu Post punya banyak Tag, satu Tag dipasang di banyak Post. Butuh pivot table (tabel perantara) post_tag.

migration pivot + model 
// Migration pivot table: post_tag (sorted alfabetis, singular)
Schema::create('post_tag', function (Blueprint $table) {
    $table->foreignId('post_id')->constrained()->cascadeOnDelete();
    $table->foreignId('tag_id')->constrained()->cascadeOnDelete();
    $table->primary(['post_id', 'tag_id']); // composite key
});

// Post.php
public function tags()
{
    return $this->belongsToMany(Tag::class);
}

// Tag.php (mirror)
public function posts()
{
    return $this->belongsToMany(Post::class);
}

// Pemakaian: attach, detach, sync
$post->tags()->attach($tagId);              // tambah relasi
$post->tags()->detach($tagId);              // hapus relasi
$post->tags()->sync([1, 2, 5]);          // set ulang (semua yang lain dihapus)
$post->tags()->toggle([1, 3]);            // toggle on/off

// Pivot dengan kolom tambahan (mis. created_at, priority)
$post->tags()->attach($tagId, ['priority' => 1]);
foreach ($post->tags as $tag) {
    echo $tag->pivot->priority;
}

Eager Loading — Solusi N+1 Query Problem

Masalah N+1

Tanpa eager loading, kode ini bisa melakukan 101 query SQL untuk 100 post (1 ambil semua post + 100 ambil author setiap post). Akibatnya: aplikasi lambat dan database overload.

N+1 problem 
$posts = Post::all(); // Query 1: SELECT * FROM posts

foreach ($posts as $post) {
    echo $post->author->name;
    // Query 2: SELECT * FROM users WHERE id = 1
    // Query 3: SELECT * FROM users WHERE id = 2
    // ... 100 query lagi! Total: 101 query SQL 😱
}
eager loading 
$posts = Post::with('author')->get();
// Query 1: SELECT * FROM posts
// Query 2: SELECT * FROM users WHERE id IN (1, 2, 3, ...)

foreach ($posts as $post) {
    echo $post->author->name;
    // Tidak query — data sudah di-load!
}

// Total: 2 query saja, tidak peduli ada berapa post 🎉

// Multiple relasi sekaligus
$posts = Post::with(['author', 'comments', 'tags'])->get();

// Nested (relasi dari relasi)
$posts = Post::with('comments.user')->get();

// Eager loading dengan filter
$posts = Post::with(['comments' => function ($q) {
    $q->where('approved', true);
}])->get();

withCount() — Hitung Tanpa Load

withCount 
// Ambil post + jumlah komentar (tanpa load datanya)
$posts = Post::withCount('comments')->get();

foreach ($posts as $post) {
    echo $post->title;
    echo $post->comments_count; // otomatis tersedia
}

// 2. Hubungkan record yang sudah ada (belongsTo)
