Modul 7: Layout dan Component Blade

Estimasi: 50 Menit
Teori & Praktikum

Sebagian besar halaman website punya struktur sama: navbar di atas, footer di bawah, sidebar di samping. Menyalin HTML ini ke setiap file = mimpi buruk maintenance. Blade menyediakan dua mekanisme untuk berbagi layout: Template Inheritance (klasik) dan Components (modern).

🖼️
Analogi: Bingkai & Foto

Bayangkan bingkai foto. Bingkainya (kayu, kaca, lubang di tengah) = master layout. Fotonya = konten halaman. Bingkai sama untuk semua foto; yang berbeda hanya foto di tengahnya. Anda tidak membuat bingkai baru setiap kali ingin pajang foto baru — Anda tinggal ganti fotonya.

1. Template Inheritance (@extends)

Cara klasik dengan dua direktif kunci:

  1. @yield('name') — di master layout: "di sini akan diisi konten dari child."
  2. @section('name') — di child view: "ini isinya untuk slot 'name'."

Membuat Master Layout

resources/views/layouts/app.blade.php 
<!DOCTYPE html>
<html lang="id">
<head>
    <title>@yield('title', 'Aplikasi Saya')</title>
    <link rel="stylesheet" href="/css/app.css">
@stack('styles') <!-- CSS tambahan dari child -->
</head>
<body>
    <nav>Navbar tetap di sini</nav>

    <main>
        @yield('content')
    </main>

    <footer>Footer tetap di sini</footer>
@stack('scripts') <!-- JS tambahan dari child -->
</body>
</html>

Menggunakan Master Layout (Child View)

resources/views/posts/index.blade.php 
@extends('layouts.app')

@section('title', 'Daftar Post')

@section('content')
    <h1>Semua Post</h1>
    @foreach ($posts as $post)
        <article>{{ $post->title }}</article>
    @endforeach
@endsection

@push('scripts') <!-- Inject ke @stack('scripts') -->
    <script src="/js/posts.js"></script>
@endpush

2. Blade Components (Direkomendasikan)

Components adalah cara modern yang mirip dengan Vue/React. Sintaksnya pakai tag XML-like (<x-nama>) — lebih fleksibel, mendukung props, slot, dan dapat di-compose seperti puzzle.

terminal 
# Generate component dengan class (untuk logic kompleks)
php artisan make:component Alert

# Generate anonymous component (cuma view, lebih ringan)
php artisan make:component Alert --view

Component Sederhana

resources/views/components/alert.blade.php 
<!-- Deklarasi props yang diterima -->
@props(['type' => 'info', 'dismissible' => false])

<div class="alert alert-{{ $type }}">
    {{ $slot }} <!-- isi tag -->
    @if ($dismissible)
        <button onclick="this.parentElement.remove()">×</button>
    @endif
</div>

Memakai Component

pakai <x-alert> 
<!-- Pakai default (type=info, dismissible=false) -->
<x-alert>
    Selamat datang di aplikasi!
</x-alert>

<!-- Pakai dengan props -->
<x-alert type="success" :dismissible="true">
    Data berhasil disimpan!
</x-alert>

<!-- Bind variabel (titik dua di depan props) -->
<x-alert :type="$flashType">
    {{ session('message') }}
</x-alert>

Component Layout Modern

Daripada @extends, modern Laravel pakai layout component <x-layout>:

resources/views/components/layout.blade.php 
@props(['title' => 'Default'])

<!DOCTYPE html>
<html>
<head>
    <title>{{ $title }}</title>
</head>
<body>
    <nav>Navbar</nav>
    <main>{{ $slot }}</main>
    <footer>Footer</footer>
</body>
</html>
resources/views/welcome.blade.php 
<x-layout title="Halaman Beranda">
    <h1>Selamat Datang!</h1>
    <p>Halaman ini otomatis mendapatkan nav dan footer dari layout.</p>
</x-layout>

Named Slots (Banyak Slot)

Component bisa punya beberapa slot dengan nama berbeda — misal: header dan footer kartu:

components/card.blade.php 
<div class="card">
    <div class="card-header">{{ $header }}</div>
    <div class="card-body">{{ $slot }}</div>
    <div class="card-footer">{{ $footer }}</div>
</div>
pakai <x-card> 
<x-card>
    <x-slot:header>
        <h2>Judul Kartu</h2>
    </x-slot>

    <p>Konten utama kartu</p>

    <x-slot:footer>
        <button>Tombol Aksi</button>
    </x-slot>
</x-card>

@extends vs Component

Aspek@extends (klasik)<x-component> (modern)
SintaksDirektif: @section, @yieldTag XML-like: <x-foo>
ReusabilityHanya 1 master per childBisa nested, compose, props
Pass dataSection bisa terima paramProps (lebih natural)
LogicCuma di viewBisa pakai class PHP
Cocok untukProject legacy, single pageProject modern, UI library
Rekomendasi untuk Project Baru

Pakai Components untuk semua yang reusable (button, card, alert, badge, modal). Pakai layout component juga untuk master template. Laravel Breeze & Jetstream sudah memakai pola ini. @extends masih boleh, tapi sudah dianggap "gaya lama".
