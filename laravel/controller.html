Modul 4: Controller dan Request

Estimasi: 45 Menit
Teori & Praktikum

Controller adalah jantung dari arsitektur MVC. Alih-alih menulis semua logika aplikasi di dalam file route (web.php), kita mengelompokkan logika tersebut ke dalam class Controller. Hasilnya: kode lebih terstruktur, mudah dipelihara, dan dapat diuji.

👨‍🍳
Analogi: Koki di Dapur

Bayangkan Anda di restoran. Pelayan (Route) menerima pesanan Anda. Koki (Controller) yang sebenarnya memasak — ambil bahan dari kulkas (Model), olah, plating, lalu kasih ke pelayan untuk diantar (View). Tanpa koki, pelayan harus sambil masak — chaos!

Membuat Controller

Cara terbaik membuat controller di Laravel adalah lewat Artisan CLI. Laravel akan generate file dengan struktur class yang benar.

Terminal 
# Controller biasa (kosong)
php artisan make:controller UserController

# Resource Controller (7 method CRUD lengkap otomatis dibuat)
php artisan make:controller ProductController --resource

# Resource Controller + Model binding (paling sering dipakai)
php artisan make:controller ProductController --resource --model=Product

# API Resource Controller (tanpa create dan edit form view)
php artisan make:controller ProductController --api

File controller akan dibuat di app/Http/Controllers/. Berikut contoh isi UserController.php:

app/Http/Controllers/UserController.php 
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Models\User;

class UserController extends Controller
{
    // GET /users — Daftar semua user
    public function index()
    {
        $users = User::all();
        return view('users.index', compact('users'));
    }

    // GET /users/{id} — Detail satu user
    public function show($id)
    {
        $user = User::findOrFail($id); // 404 jika tidak ketemu
        return view('users.show', compact('user'));
    }
}

Request Object — Membaca Input User

Class Illuminate\Http\Request menyediakan metode berorientasi objek untuk berinteraksi dengan HTTP request: membaca input form, query string, header, cookie, dan file upload.

PostController.php 
public function store(Request $request)
{
    // Mengambil SEMUA input form (sebagai array)
    $input = $request->all();

    // Mengambil input spesifik berdasarkan name
    $title = $request->input('title');

    // Dengan nilai default jika kosong
    $author = $request->input('author', 'Anonim');

    // Cara singkat via dynamic property
    $email = $request->email;

    // Cek apakah field ada
    if ($request->has('newsletter')) { /* checkbox dicentang */ }

    // Cek apakah ada file upload
    if ($request->hasFile('photo')) {
        $path = $request->file('photo')->store('uploads');
    }

    // Query string: /posts?page=2 → $request->query('page') === 2
    $page = $request->query('page', 1);

    // Hanya ambil field tertentu (filter)
    $clean = $request->only(['title', 'body']);

    // Ambil semua kecuali field tertentu
    $safe = $request->except(['_token', 'password']);
}

Route Model Binding — Sihir Otomatis

Daripada menulis User::findOrFail($id) di setiap method, Laravel bisa otomatis mengubah parameter URL menjadi instance Model. Ini disebut Route Model Binding.

tanpa binding 
// routes/web.php
Route::get('/posts/{id}', [PostController::class, 'show']);

// PostController.php
public function show($id)
{
    $post = Post::findOrFail($id); // Manual query
    return view('posts.show', compact('post'));
}
dengan binding 
// routes/web.php — parameter pakai nama model (lowercase)
Route::get('/posts/{post}', [PostController::class, 'show']);

// PostController.php — type-hint Post, Laravel ambilkan otomatis
public function show(Post $post)
{
    return view('posts.show', compact('post'));
    // Tidak ditemukan? Auto 404. Selesai.
}
Kapan Pakai Route Model Binding?

Selalu — kecuali Anda butuh logic khusus saat fetch (misal: filter tambahan, query relasi). Binding menghemat 1 baris di setiap method dan otomatis handle 404. Untuk binding pakai kolom non-id (misal slug): Route::get('/posts/{post:slug}', ...).

Response: Mengirim Balik ke Browser

Controller bisa mengembalikan berbagai macam response:

contoh response 
// 1. Return view HTML
return view('posts.index', $data);

// 2. Return JSON (auto-serialize, cocok untuk API)
return response()->json(['status' => 'ok', 'data' => $post]);

// 3. Redirect ke halaman lain
return redirect('/dashboard');
return redirect()->route('posts.show', $post);
return back(); // Balik ke halaman sebelumnya

// 4. Redirect dengan flash message (tampil sekali, lalu hilang)
return redirect('/posts')->with('success', 'Post berhasil dibuat!');

// 5. Download file
return response()->download('path/to/file.pdf');

// 6. Custom status code
return response()->json(['error' => 'Not Found'], 404);
Routing Super Singkat

Daripada menulis 7 baris route manual di web.php, cukup satu baris:
Route::resource('photos', PhotoController::class);
Laravel otomatis pasangkan ke-7 method CRUD ke route yang sesuai.
