Selamat datang di kelas Web Framework (Laravel). Pada modul ini kita akan berkenalan dengan Laravel — framework PHP paling populer di dunia untuk membangun aplikasi web modern, serta memahami arsitektur Model-View-Controller (MVC) yang menjadi fondasinya.

Apa itu Framework?

Sebelum bicara Laravel, mari pahami dulu apa itu framework. Bayangkan Anda ingin membangun rumah. Anda bisa mulai dari nol — menebang pohon, membakar batu bata, membuat semen. Tapi itu lambat, mahal, dan kemungkinan besar hasilnya buruk. Pilihan kedua: beli rangka rumah yang sudah jadi (fondasi, atap, dinding standar), Anda tinggal menyesuaikan interior dan finishing sesuai kebutuhan.

Itulah framework — kerangka kode yang sudah menyediakan struktur dan fungsi-fungsi umum yang dibutuhkan hampir semua aplikasi web (routing, koneksi database, autentikasi, keamanan). Anda fokus ke fitur unik aplikasi Anda, bukan ke pekerjaan repetitif.

🏗️
Analogi sederhana

PHP murni = Anda memasak dari bahan mentah (beli ayam hidup, potong, bumbui, panaskan minyak). Laravel = restoran dengan dapur lengkap dan resep terstandar — Anda tinggal "racik" hidangan akhir. Hasilnya: konsisten, cepat, dan kesalahan minim.

Apa itu Laravel?

Laravel adalah framework PHP open-source yang diciptakan oleh Taylor Otwell pada tahun 2011. Laravel dirancang untuk membuat pengembangan web menjadi lebih ekspresif, elegan, dan menyenangkan dengan menyediakan tools dan struktur yang terorganisasi rapi.

Laravel mengikuti filosofi "developer happiness" — setiap fitur dirancang agar kode lebih bersih, lebih mudah dibaca, dan lebih cepat dikembangkan. Dengan komunitas besar dan ekosistem yang kaya (Forge, Vapor, Nova, Livewire), Laravel menjadi pilihan utama untuk proyek web dari skala kecil hingga enterprise.

Mengapa Laravel?

Laravel adalah framework PHP #1 di GitHub dengan 70k+ stars. Digunakan oleh perusahaan besar seperti BBC, Pfizer, 9GAG, dan ribuan startup. Fitur seperti authentication, routing, ORM, queue, testing sudah tersedia out-of-the-box — pada framework lain harus dikonfigurasi manual.

Arsitektur MVC (Model-View-Controller)

MVC adalah pola arsitektur yang memisahkan aplikasi menjadi tiga komponen dengan tanggung jawab yang jelas. Pemisahan ini menjaga kode tetap rapi: ketika ingin mengubah tampilan, Anda tidak perlu menyentuh logika database; ketika mengubah skema database, tampilan tidak ikut rusak.

🍽️
Analogi MVC: Restoran

Model = juru masak & gudang bahan (mengambil & mengolah data). View = piring saji yang sampai ke meja Anda (tampilan akhir). Controller = pelayan (menerima pesanan, ke dapur ambil makanan, antar ke meja).

Pengatur Alur
Model — Merepresentasikan tabel database. Di Laravel menggunakan Eloquent ORM: 1 Model = 1 tabel. Contoh: Model User mengelola tabel users. Kode: User::where('active', true)->get() → ambil semua user yang aktif.

Fitur Unggulan Laravel

  1. Eloquent ORM
    Object-Relational Mapping yang elegan. Query database memakai sintaks PHP yang ekspresif: User::where('active', true)->get() — tanpa menulis SQL manual.
  2. Blade Templating
    Template engine ringan namun powerful. Mendukung inheritance (@extends), section (@section), dan komponen reusable (<x-component>).
  3. Artisan CLI
    Command-line tool bawaan untuk membuat model, controller, migration, seeder, dan menjalankan berbagai tugas development. Contoh: php artisan make:model Post -mcr.
  4. Migration & Seeder
    Version control untuk database. Definisikan struktur tabel dengan kode PHP — semua anggota tim bisa sinkron tanpa share file .sql.
  5. Routing Ekspresif
    Sistem routing yang bersih dan fleksibel: route groups, middleware, named routes, resource routes, dan API versioning.
  6. Authentication & Authorization
    Sistem auth out-of-the-box: login, register, reset password, verifikasi email, dan policy-based authorization. Cukup install Laravel Breeze/Jetstream.

Perbandingan: PHP Murni vs Laravel

Aspek PHP Murni Laravel
Koneksi DB mysqli_connect() manual setiap file Otomatis dari .env
Query data Tulis SQL: SELECT * FROM users WHERE active=1 User::where('active', 1)->get()
Validasi form Cek satu per satu dengan if/else Aturan ringkas: 'email' => 'required|email'
Keamanan Manual: escape XSS, CSRF, SQL injection Otomatis: {{ }}, @csrf, Eloquent prepared
Routing File per URL atau switch ($_GET['page']) Deklaratif: Route::get('/x', ...)

Struktur Folder Laravel

Sebuah project Laravel baru memiliki struktur folder yang konsisten. Mengenal lokasi setiap "ruangan" ini akan menyelamatkan banyak waktu saat coding.

project-structure 
laravel-project/
├── app/                # Logika utama aplikasi
│   ├── Http/
│   │   ├── Controllers/  # Controller (logic handler)
│   │   └── Middleware/   # Middleware (filter request)
│   └── Models/          # Eloquent Models
├── config/             # File konfigurasi (database, mail, dll)
├── database/
│   ├── migrations/     # Definisi struktur tabel
│   ├── seeders/        # Data dummy untuk testing
│   └── factories/       # Factory untuk generate data
├── resources/
│   └── views/          # Blade templates (HTML)
├── routes/
│   ├── web.php         # Routes untuk web (session, CSRF)
│   └── api.php         # Routes untuk REST API
├── public/             # Asset publik (CSS, JS, images)
├── .env                 # Environment variables (DB, API keys)
└── composer.json        # Dependencies PHP

Apa yang Akan Dipelajari?

Kursus ini dirancang dengan pendekatan 70% praktikum dan 30% teori. Berikut roadmap pembelajaran:

  1. Instalasi & Konfigurasi — Setup PHP, Composer, Laravel, dan konfigurasi database (MySQL).
  2. Routing & Controller — Mendefinisikan URL, membuat controller, menangani request/response.
  3. Blade Templating — Membuat tampilan dinamis dengan template inheritance dan komponen.
  4. Database & Eloquent — Migration, seeder, CRUD operations, dan relasi antar model.
  5. Form & Validasi — Menangani input user, validasi data, dan menampilkan error.
  6. Authentication — Implementasi login, register, dan proteksi halaman.
  7. REST API — Membangun API endpoint untuk aplikasi mobile/SPA.
