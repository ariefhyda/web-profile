Modul 3: Routing Dasar Laravel

Estimasi: 60 Menit
Teori & Praktikum

Routing adalah mekanisme untuk menentukan bagaimana aplikasi merespons request dari client ke URL tertentu. Sederhananya, routing menjawab pertanyaan: "Ketika user buka URL X, kode mana yang harus dijalankan?"

📞
Analogi: Resepsionis Hotel

Bayangkan router sebagai resepsionis. Setiap tamu (request) datang menyebutkan kamar tujuan (URL). Resepsionis mencocokkan dengan daftar (route table) lalu mengarahkan ke petugas yang tepat (controller). Tanpa resepsionis, tamu akan tersesat — semua request akan menumpuk di pintu masuk.

Di Laravel, semua route didefinisikan dalam file routes/web.php (untuk web) dan routes/api.php (untuk API). Mulai Laravel 11, file api.php tidak ada secara default — aktifkan dengan php artisan install:api.

HTTP Methods — Mengapa Penting?

Setiap interaksi web memakai salah satu HTTP method. Pilihan method bukan sembarangan — ada konvensi yang harus diikuti agar API Anda RESTful dan masuk akal:

MethodTujuan (CRUD)Contoh URLIdempotent?
GET Mengambil data (Read) /posts, /users/1 Ya — boleh diulang
POST Membuat data baru (Create) /posts (kirim body) Tidak — buat duplikat
PUT Update seluruh data (Update) /posts/5 Ya — hasilnya sama
PATCH Update sebagian field /posts/5 Ya
DELETE Hapus data (Delete) /posts/5 Ya
Idempotent?

Operasi idempotent = mengulanginya menghasilkan efek yang sama. Refresh halaman GET = aman. Refresh halaman setelah POST checkout = berbahaya (bisa double-charge). Karena itu browser sering menampilkan peringatan "Confirm form resubmission".

Route Dasar

Route paling sederhana menerima URI dan sebuah closure (fungsi anonim):

routes/web.php 
use Illuminate\Support\Facades\Route;

// Route GET — menampilkan halaman
Route::get('/', function () {
    return view('welcome');
});

// Route dengan parameter wajib (id WAJIB ada di URL)
Route::get('/user/{id}', function ($id) {
    return "User ID: " . $id;
});

// Route dengan parameter opsional (tanda ? di belakang)
Route::get('/hello/{name?}', function ($name = 'Guest') {
    return "Hello, " . $name;
});

// Named route — bisa dipanggil via route('dashboard')
Route::get('/dashboard', function () {
    return view('dashboard');
})->name('dashboard');

// Route dengan constraint (id harus angka)
Route::get('/post/{id}', function ($id) {
    return "Post #" . $id;
})->where('id', '[0-9]+');

Route ke Controller

Untuk aplikasi yang lebih besar, logika sebaiknya dipindahkan dari closure di file route ke Controller — sehingga route hanya bertugas "menunjukkan jalan".

routes/web.php 
use App\Http\Controllers\PostController;

// Route ke method tertentu di controller
Route::get('/posts', [PostController::class, 'index']);
Route::get('/posts/{id}', [PostController::class, 'show']);
Route::post('/posts', [PostController::class, 'store']);

// Resource Route — 7 route CRUD sekaligus!
Route::resource('posts', PostController::class);
// Menghasilkan: index, create, store, show, edit, update, destroy

Resource Route — 7 Route dalam 1 Baris

Saat membangun CRUD, Anda akan butuh route yang persis sama berulang. Route::resource() membuatkan ke-7 route standar sekaligus:

MethodURIController@methodTujuan
GET/postsindexDaftar semua post
GET/posts/createcreateForm tambah
POST/postsstoreSimpan post baru
GET/posts/{id}showDetail satu post
GET/posts/{id}/editeditForm edit
PUT/PATCH/posts/{id}updateUpdate post
DELETE/posts/{id}destroyHapus post

Route Group — Hemat Pengetikan

Sering kali sekelompok route punya prefix atau middleware sama. Daripada menulis berulang, gunakan group():

routes/web.php 
// Semua route admin pakai prefix /admin DAN middleware auth
Route::prefix('admin')
    ->middleware('auth')
    ->name('admin.') // prefix nama
    ->group(function () {
        Route::get('/dashboard', [AdminController::class, 'index'])->name('dashboard');
        Route::get('/users', [AdminController::class, 'users'])->name('users');
        Route::get('/settings', [AdminController::class, 'settings'])->name('settings');
    });

// URL hasilnya: /admin/dashboard, /admin/users, /admin/settings
// Nama route: admin.dashboard, admin.users, admin.settings
// Semua diproteksi middleware 'auth'

Named Route — Jangan Hardcode URL!

Daripada menulis <a href="/posts/5"> di banyak tempat, beri nama route lalu rujuk by name. Jika URL berubah, cukup ubah di satu tempat.

blade.php — buruk 
<a href="/posts/5">Detail</a>
<a href="/posts/5/edit">Edit</a>
<form action="/posts/5" method="POST">
// Kalau URL berubah jadi /articles/5, semua harus diubah manual
blade.php — bagus 
<a href="{{ route('posts.show', 5) }}">Detail</a>
<a href="{{ route('posts.edit', 5) }}">Edit</a>
<form action="{{ route('posts.update', 5) }}" method="POST">
// URL berubah? Tinggal ubah definisi route — semua link otomatis update
Tips Debugging: php artisan route:list

Lupa route apa saja yang ada? Jalankan php artisan route:list untuk lihat semua route terdaftar (method, URI, name, controller, middleware). Filter dengan --name=posts untuk lihat hanya yang mengandung "posts".
