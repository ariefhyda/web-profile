Modul 8: Migration dan Seeder

Estimasi: 55 Menit
Teori & Praktikum

Migration adalah version control untuk database. Daripada membagikan file .sql ke rekan tim, Anda mendefinisikan struktur tabel pakai kode PHP. Rekan tim cukup jalankan satu perintah untuk menyamakan skema database.

📜
Analogi: Resep Masak Tertulis

Daripada teman Anda harus melihat masakan jadi dan menebak bumbunya, lebih mudah memberinya resep tertulis step-by-step. Tinggal ikuti, hasilnya pasti sama. Migration = resep database Anda. Cukup git pull + php artisan migrate, teman Anda punya struktur DB yang sama persis.

Mengapa Tidak Pakai .sql Saja?

AspekFile .sqlMigration Laravel
VersioningManual: backup tiap perubahanOtomatis tersimpan di Git
RollbackSulit, harus tulis ulang DROPphp artisan migrate:rollback
Lintas DBBerbeda MySQL vs PostgreSQLSintaks sama, Laravel translate
Sharing timKirim file via WhatsApp 😅Push ke Git → sync otomatis

Membuat Migration

Terminal 
# Migration baru untuk tabel baru
php artisan make:migration create_flights_table

# Migration untuk menambah kolom ke tabel yang sudah ada
php artisan make:migration add_status_to_flights_table --table=flights

# Otomatis dibuat di database/migrations/<timestamp>_*.php

Timestamp di nama file menentukan urutan eksekusi. Migration lama jalan duluan — penting karena tabel yang punya foreign key harus dibuat setelah tabel yang dirujuk.

Schema Builder Interaktif

Coba bangun skema tabel sendiri! Tambahkan kolom, pilih tipe data, lalu lihat kode migration yang dihasilkan.

Schema Builder

Mendefinisikan Struktur Tabel

            

                

                    

                    database/migrations/..._create_flights_table.php
                    
                

                
use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    // Dijalankan saat php artisan migrate
    public function up(): void
    {
        Schema::create('flights', function (Blueprint $table) {
            $table->id();                          // Primary key auto-increment
            $table->string('name');                // VARCHAR(255)
            $table->string('airline')->nullable(); // Boleh NULL
            $table->integer('price')->default(0);  // Default 0
            $table->boolean('is_active')->default(true);
            $table->text('description')->nullable();
            $table->timestamp('departure_at')->useCurrent();
            $table->timestamps();                  // created_at + updated_at
        });
    }

    // Dijalankan saat php artisan migrate:rollback
    public function down(): void
    {
        Schema::dropIfExists('flights');
    }
};

Foreign Key (Relasi Antar Tabel)

            

                

                    

                    contoh foreign key
                

                
// Cara lama (verbose)
$table->unsignedBigInteger('user_id');
$table->foreign('user_id')
      ->references('id')
      ->on('users')
      ->onDelete('cascade');

// Cara modern (singkat — Laravel 7+)
$table->foreignId('user_id')
      ->constrained()           // otomatis cari tabel "users"
      ->cascadeOnDelete();    // hapus post jika user dihapus

// Jika nama tabel tidak sesuai konvensi:
$table->foreignId('author_id')->constrained('users');

            


            
Menjalankan Migration

            

                

                    

                    Terminal
                    
                

                
# Jalankan semua migration yang belum dieksekusi
php artisan migrate

# Batalkan migration terakhir (1 batch)
php artisan migrate:rollback

# Reset SEMUA migration (rollback semua, lalu migrate lagi)
php artisan migrate:refresh

# DROP semua tabel, jalankan ulang dari awal
php artisan migrate:fresh

# Sekaligus jalankan seeder setelah fresh
php artisan migrate:fresh --seed

# Cek status migration (mana yang sudah, mana yang belum)
php artisan migrate:status

            


            

                

                    
                    JANGAN migrate:fresh di Production!
                

                
migrate:fresh dan migrate:refresh akan menghapus SEMUA DATA. Hanya gunakan di environment lokal/dev. Di production, selalu pakai migrate biasa atau buat migration baru untuk perubahan tabel.

            


            
Seeder — Mengisi Data Awal

            
Seeder mengisi database dengan data awal atau dummy. Berguna untuk testing dan demo. Biasanya dipadukan dengan Factory yang men-generate data acak.

            

                

                    

                    database/seeders/DatabaseSeeder.php
                    
                

                
use Illuminate\Database\Seeder;
use Illuminate\Support\Facades\Hash;
use App\Models\{User, Post};

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        // 1. Data spesifik (admin user)
        User::create([
            'name' => 'Admin',
            'email' => 'admin@example.com',
            'password' => Hash::make('password'),
        ]);

        // 2. Data dummy via Factory (10 user random)
        User::factory(10)->create();

        // 3. Setiap user buat 3 post
        User::all()->each(function ($user) {
            Post::factory(3)->create(['user_id' => $user->id]);
        });
    }
}

            


            

                

                    
                    Workflow Pengembangan Database
                

                
Workflow tipikal: 1) Buat migration → tulis schema → 2) Buat factory → definisikan data dummy → 3) Daftarkan di DatabaseSeeder → 4) Jalankan php artisan migrate:fresh --seed → DB siap untuk testing.
