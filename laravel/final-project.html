Modul 14: Final Project

Estimasi: 2 Minggu
Tugas Besar

Selamat! Anda telah mencapai akhir dari rangkaian modul dasar Laravel. Untuk memastikan pemahaman Anda terhadap arsitektur MVC, ORM, Routing, Blade, dan Middleware, Anda diwajibkan untuk mengerjakan sebuah Final Project (Tugas Besar) secara individu atau berkelompok.

Deskripsi Proyek: Aplikasi Blog Sederhana

Tugas Anda adalah membangun sebuah sistem CMS (Content Management System) atau Aplikasi Blog Sederhana (mirip seperti Medium atau Dev.to) yang mengimplementasikan semua konsep yang telah dipelajari dari Modul 1 hingga Modul 13.

Tech Stack Wajib

Backend: Laravel 10 / 11
Database: MySQL / PostgreSQL / SQLite
Frontend: Blade Templating + TailwindCSS / Bootstrap 5

Kriteria dan Fitur Wajib

1. Autentikasi & Otorisasi

  • Sistem Login dan Register (Boleh menggunakan Breeze/Jetstream).
  • Dua role utama: Admin (bisa mengelola semua artikel & user) dan User/Penulis (hanya bisa mengelola artikelnya sendiri).
  • Gunakan Middleware untuk memproteksi halaman Dashboard.

2. Manajemen Artikel (CRUD)

  • Fitur Create, Read, Update, Delete untuk tabel posts atau articles.
  • Input form wajib divalidasi (Judul wajib, konten minimal 50 karakter).
  • Artikel memiliki slug untuk URL yang ramah SEO (contoh: /post/cara-belajar-laravel).
  • Fitur upload gambar untuk thumbnail artikel.

3. Relasi Database (Eloquent)

  • Relasi One-to-Many antara User dan Post (Satu penulis bisa punya banyak artikel).
  • Relasi One-to-Many antara Category dan Post (Satu kategori memiliki banyak artikel).
  • (Opsional Nilai Plus): Relasi Many-to-Many antara Post dan Tag.

4. Tampilan (Blade Views)

  • Halaman Landing/Beranda: Menampilkan daftar semua artikel terbaru dari semua penulis (Pagination diterapkan).
  • Halaman Detail Artikel: Menampilkan isi lengkap beserta nama penulis, tanggal (format manusiawi pakai diffForHumans()), dan kategori.
  • Gunakan Blade Layout (@extends atau Component <x-layout>).

Sistem Penilaian (Rubrik)

Aspek Penilaian Bobot Keterangan
Fungsionalitas CRUD 30% Semua fungsi berjalan normal tanpa error 500.
Database & Relasi 25% Desain skema (Migration) yang tepat dan implementasi Eloquent.
UI / UX (Blade) 20% Tampilan rapi, responsif, menggunakan layout yang dapat digunakan kembali.
Keamanan (Auth & Validasi) 15% Form diproteksi CSRF, input divalidasi, dan route diamankan.
Kode Bersih (Clean Code) 10% Penamaan variabel yang jelas, controller tidak berisi terlalu banyak logika, dll.

Langkah Pengumpulan

  1. Push kode akhir Anda ke repositori GitHub. (Pastikan tidak mem-push folder vendor/ dan file .env asli).
  2. Ekspor database Anda (file .sql) atau pastikan Seeder berfungsi dengan sempurna.
  3. Submit URL Repositori ke platform e-learning kampus/kelas Anda.
