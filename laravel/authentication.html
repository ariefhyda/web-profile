Modul 12: Authentication

Estimasi: 40 Menit
Teori & Praktikum

Sistem Authentication (autentikasi / login) bisa memakan waktu berhari-hari untuk dibangun dari awal karena banyaknya detail keamanan (hashing password, token session, proteksi brute force). Untungnya, Laravel menyediakan semua ini secara out-of-the-box.

Starter Kits (Breeze / Jetstream)

Untuk aplikasi baru, sangat disarankan menggunakan Starter Kits resmi Laravel daripada menulis sistem login sendiri. Starter kit yang paling sederhana dan direkomendasikan untuk pemula adalah Laravel Breeze.

Terminal 
# Instalasi package Breeze via Composer
composer require laravel/breeze --dev

# Generate file view, route, dan controller untuk login/register
php artisan breeze:install

# Jalankan migrasi tabel users
php artisan migrate

# Install dan compile asset frontend (Tailwind/Vite)
npm install
npm run dev

Setelah perintah di atas selesai, Anda otomatis memiliki halaman /login, /register, /forgot-password, dan sistem proteksi route.

Facade Auth

Facade Auth digunakan untuk berinteraksi dengan user yang sedang login di Controller atau route Anda.

PHP 
use Illuminate\Support\Facades\Auth;

// Mengecek apakah user sudah login
if (Auth::check()) {
    // User sedang login...
}

// Mengambil objek user yang sedang login
$user = Auth::user();
echo $user->name;

// Mengambil ID user yang sedang login
$id = Auth::id();

Auth Helpers di Blade

Di dalam file view Blade, Anda tidak perlu menggunakan Facade. Cukup gunakan directive @auth dan metode auth().

resources/views/navbar.blade.php 
@auth
    <!-- Tampil jika sudah login -->
    <p>Selamat datang kembali, {{ auth()->user()->name }}!</p>
    <form method="POST" action="{{ route('logout') }}">
        @csrf
        <button type="submit">Logout</button>
    </form>
@else
    <!-- Tampil jika belum login -->
    <a href="{{ route('login') }}">Log in</a>
    <a href="{{ route('register') }}">Register</a>
@endauth

Melindungi Route (Proteksi)

Untuk membatasi akses ke suatu halaman hanya untuk user yang sudah login, cukup pasangkan middleware auth pada route tersebut.

routes/web.php 
Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware('auth');
