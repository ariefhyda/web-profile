Modul 8: Decision Tree (Pohon Keputusan) & C4.5

Estimasi: 60 Menit
Teori & Interaktif

Decision Tree (Pohon Keputusan) adalah salah satu metode klasifikasi paling populer karena mudah diinterpretasikan oleh manusia. Algoritma ini memecah dataset menjadi himpunan bagian yang lebih kecil dan homogen berdasarkan aturan "Jika-Maka" (If-Then rules), membentuk struktur seperti pohon terbalik.

Komponen Pohon

• Root Node (Akar): Atribut paling membedakan data.
• Internal Node (Cabang): Atribut pengujian yang membagi data lebih lanjut.
• Leaf Node (Daun): Hasil akhir / keputusan / kelas target.

Algoritma C4.5: Membangun Pohon

Bagaimana kita menentukan atribut mana yang menjadi akar atau cabang? Algoritma C4.5 (pengembangan dari ID3) menggunakan konsep Entropy dan Gain Ratio dari Teori Informasi.

1. Entropy

Entropy mengukur tingkat ketidakteraturan (impurity) atau keacakan dari sebuah dataset. Jika data semuanya memiliki kelas yang sama, Entropy = 0 (murni). Jika terbagi rata 50/50, Entropy = 1.

Entropy(S) = ∑ -pi * log2(pi)

2. Information Gain

Gain mengukur seberapa banyak "informasi" atau pengurangan Entropy yang kita dapatkan dengan membagi data berdasarkan atribut tertentu.

Gain(S, A) = Entropy(S) - ∑ (|Sv| / |S|) * Entropy(Sv)

Catatan Penting: C4.5 menyempurnakan Information Gain dengan menggunakan Gain Ratio untuk menghindari bias terhadap atribut yang memiliki banyak nilai unik.

Bedah Kasus: Menentukan Root Node

Mengapa Cuaca (Outlook) menjadi akar (Root), bukan Kelembaban atau Angin? Mari kita bedah perhitungannya berdasarkan dataset di atas.

Langkah 1: Hitung Entropy Total (Root)

Dari keseluruhan 14 data, terdapat: 9 YES dan 5 NO.
Kita masukkan ke dalam rumus Entropy:

Entropy(Total) = -(9/14 * log2(9/14)) - (5/14 * log2(5/14))
Entropy(Total) = -(0.642 * -0.637) - (0.357 * -1.485)
Entropy(Total) = 0.409 + 0.531 = 0.940
Langkah 2: Hitung Gain untuk Atribut "Outlook"

Atribut Outlook memiliki 3 nilai cabang: Sunny, Overcast, dan Rainy.

• Sunny (5 data: 2 Yes, 3 No)

Entropy(Sunny) = -(2/5 * log2(2/5)) - (3/5 * log2(3/5)) = 0.971

• Overcast (4 data: 4 Yes, 0 No)

Entropy(Overcast) = -(4/4 * log2(4/4)) - (0) = 0 (Murni!)

• Rainy (5 data: 3 Yes, 2 No)

Entropy(Rainy) = -(3/5 * log2(3/5)) - (2/5 * log2(2/5)) = 0.971

Rumus Gain: Gain(S, A) = Entropy(Total) - ∑ (|Sv|/|S| * Entropy(Sv))

Gain(Outlook) = 0.940 - [ (5/14 * 0.971) + (4/14 * 0) + (5/14 * 0.971) ]
Gain(Outlook) = 0.940 - [ 0.346 + 0 + 0.346 ]
Gain(Outlook) = 0.940 - 0.692 = 0.246
Langkah 3: Hitung Gain untuk Atribut Lain (Contoh: Humidity)

Sebagai perbandingan, mari kita hitung Gain untuk atribut Humidity yang memiliki 2 nilai cabang: High dan Normal.

• High (7 data: 3 Yes, 4 No)

Entropy(High) = -(3/7 * log2(3/7)) - (4/7 * log2(4/7)) = 0.985

• Normal (7 data: 6 Yes, 1 No)

Entropy(Normal) = -(6/7 * log2(6/7)) - (1/7 * log2(1/7)) = 0.592
Gain(Humidity) = 0.940 - [ (7/14 * 0.985) + (7/14 * 0.592) ]
Gain(Humidity) = 0.940 - [ 0.4925 + 0.296 ]
Gain(Humidity) = 0.940 - 0.7885 = 0.151

Jika kita menghitung atribut lainnya dengan cara yang sama, kita akan mendapatkan:

Atribut Information Gain Keterangan
Outlook 0.246 Paling Tinggi → Jadi Root
Humidity 0.151 -
Windy 0.048 -

Karena Outlook memiliki Gain tertinggi, ia dipilih sebagai pemisah utama (Root). Namun, proses belum selesai! Kita harus melanjutkan proses pembagian ini secara rekursif untuk cabang yang belum "murni" (Entropy > 0).

Langkah 4: Melanjutkan Cabang "Outlook"

Atribut Outlook membagi 14 data menjadi 3 cabang: Overcast, Sunny, dan Rainy. Mari kita evaluasi masing-masing cabang untuk melihat apakah kita perlu membaginya lagi.

Cabang 1: Overcast (Selesai!)

Dari 4 data pada cabang Overcast, semuanya menghasilkan target "YES". Ingat pada perhitungan sebelumnya, Entropy-nya adalah 0.
Karena datanya sudah murni (semua YES), cabang ini langsung menjadi Leaf Node (Daun) "YES". Tidak perlu perhitungan lebih lanjut.

Cabang 2: Sunny (Perlu Dibagi Lagi)

Pada cabang Sunny, terdapat 5 data yang masih bercampur (2 YES, 3 NO) dengan Entropy = 0.971.
Kita harus mencari atribut mana (antara Humidity, Windy, atau Temperature) yang memiliki Gain tertinggi untuk membagi ke-5 data ini.

A. Mari kita hitung Gain untuk "Humidity" (Khusus 5 data Sunny):

  • High: Ada 3 data (Semuanya NO) → Entropy = 0 (Murni)
  • Normal: Ada 2 data (Semuanya YES) → Entropy = 0 (Murni)
Gain(Humidity|Sunny) = 0.971 - [ (3/5 * 0) + (2/5 * 0) ] = 0.971

B. Jika kita hitung atribut lain pada cabang Sunny:

  • Gain(Temperature|Sunny) = 0.571
  • Gain(Windy|Sunny) = 0.020

Kesimpulan Cabang Sunny:
Humidity memiliki Gain tertinggi (0.971). Maka, di bawah cabang Sunny kita pasang Humidity sebagai node.
Karena percabangan Humidity (High dan Normal) keduanya menghasilkan Entropy = 0, pembagian selesai! (High → NO, Normal → YES).

Cabang 3: Rainy (Perlu Dibagi Lagi)

Pada cabang Rainy, terdapat 5 data yang masih bercampur (3 YES, 2 NO) dengan Entropy = 0.971.
Sama seperti sebelumnya, kita cari Gain tertinggi dari sisa atribut untuk membagi data ini.

A. Mari kita hitung Gain untuk "Windy" (Khusus 5 data Rainy):

  • False (Weak): Ada 3 data (Semuanya YES) → Entropy = 0 (Murni)
  • True (Strong): Ada 2 data (Semuanya NO) → Entropy = 0 (Murni)
Gain(Windy|Rainy) = 0.971 - [ (3/5 * 0) + (2/5 * 0) ] = 0.971

B. Jika kita hitung atribut lain pada cabang Rainy:

  • Gain(Temperature|Rainy) = 0.020
  • Gain(Humidity|Rainy) = 0.020

Kesimpulan Cabang Rainy:
Windy memiliki Gain tertinggi (0.971). Maka, di bawah cabang Rainy kita pasang Windy sebagai node.
Karena percabangannya menghasilkan Entropy = 0, pembagian selesai! (False → YES, True → NO).

🎉 Selesai! Karena semua cabang di ujung sudah mencapai titik murni (Entropy = 0), struktur pohon tidak perlu dibagi lagi. Gambar struktur pohon di atas adalah hasil final dari seluruh tahapan perhitungan rekursif ini.

Implementasi di Python

Pustaka scikit-learn menggunakan algoritma CART (Classification and Regression Trees), yang mirip dengan C4.5 tetapi menghasilkan pohon biner dan menggunakan Gini Impurity secara default (meskipun bisa diatur ke Entropy).

decision_tree_sklearn.py 
from sklearn.tree import DecisionTreeClassifier
from sklearn.preprocessing import LabelEncoder
from sklearn import tree
import pandas as pd
import matplotlib.pyplot as plt

# 1. Dataset dan Preprocessing (sama seperti algoritma lain)
# Pastikan fitur dikonversi menjadi numerik
df = pd.read_csv('play_tennis.csv')
X = df.iloc[:, :-1].apply(LabelEncoder().fit_transform)
y = df.iloc[:, -1]

# 2. Inisialisasi Decision Tree (Menggunakan Entropy)
clf = DecisionTreeClassifier(criterion='entropy', random_state=42)

# 3. Latih Model
clf.fit(X, y)

# 4. Visualisasi Pohon (Menghasilkan grafik plot)
plt.figure(figsize=(12,8))
tree.plot_tree(clf, feature_names=X.columns, class_names=clf.classes_, filled=True)
plt.show()

# 5. Prediksi Data Baru
prediksi = clf.predict([[2, 0, 1, 0]]) # Misal: Sunny, Hot, High, Weak
print("Prediksi:", prediksi[0])
