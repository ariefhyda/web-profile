Selamat datang di sesi pertama kelas Data Mining. Pada modul ini, kita akan membahas pengantar, sejarah kelahiran Data Mining, serta proses Knowledge Discovery in Databases (KDD).

Apa itu Data Mining?

Berdasarkan materi perkuliahan, Data Mining adalah studi tentang pengumpulan, pembersihan, pengolahan, analisis, dan memperoleh wawasan (insight) yang bermanfaat dari data. Tujuannya adalah ekstraksi pola menarik dari sejumlah besar data yang implisit, sebelumnya tidak diketahui, dan berpotensi berguna atau dapat ditindaklanjuti.

Latar Belakang Kelahiran

Data mining lahir karena transisi dari penyimpanan data fisik ke digital (Revolusi Industri 3.0). Perannya semakin berevolusi di era Industri 4.0 menjadi bagian integral dari Big Data Analytics dan AI & Machine Learning sebagai "bahan bakar" untuk melakukan prediksi otomatis secara real-time.

Data Mining diantara AI, ML dan DL

Dalam ekosistem ilmu pengetahuan komputer modern, konsep Data Mining sering kali bersinggungan dengan disiplin ilmu tingkat lanjut seperti Artificial Intelligence (AI), Machine Learning (ML), dan Deep Learning (DL). Meskipun memiliki irisan yang sangat tebal, Data Mining bukanlah sebuah subset (bagian murni) dari ML atau AI, melainkan disiplin multidisiplin yang berdiri sendiri.

Data Mining memanfaatkan algoritma dari Machine Learning sebagai tools atau alat utamanya dalam membangun model analitik, namun fokus akhir dari Data Mining adalah ekstraksi wawasan (insight) bagi manusia dan temuan bisnis, bukan membuat agen cerdas sepenuhnya seperti AI.

Tahapan Proses KDD (Knowledge Discovery in Databases)

Data Mining sejatinya adalah bagian dari proses yang lebih besar yang disebut KDD. Proses KDD meliputi tahapan-tahapan berikut:

  1. Pemilihan Data (Data Selection)
    Mengidentifikasi dan menyeleksi subset data yang relevan dari dataset untuk dianalisis. Hal ini meningkatkan efisiensi dan akurasi dengan memastikan hanya data berfokus pada target yang digunakan.
  2. Pembersihan Data (Data Cleaning)
    Memastikan dataset akurat dengan menangani nilai yang hilang (Missing Values), mengurangi data berisik (Noise), dan menghapus data duplikat.
  3. Transformasi dan Reduksi Data
    Melakukan normalisasi, diskretisasi, maupun agregasi agar format data sesuai untuk analisis. Juga dilakukan reduksi dimensi (contoh: PCA) untuk menyederhanakan data namun tetap mempertahankan informasi esensial.
  4. Penambangan Data (Data Mining)
    Eksplorasi data untuk mengekstrak pola (seperti klasifikasi, asosiasi, klastering). Pola ini harus valid, baru, berguna, dan mudah dipahami.
  5. Evaluasi dan Interpretasi Hasil
    Menilai relevansi serta "skor ketertarikan" pola yang ditemukan. Tahap ini sering melibatkan visualisasi agar pembuat keputusan dapat mengambil strategi berdasarkan insight tersebut.

CRISP-DM (Cross-Industry Standard Process for Data Mining)

Selain proses akademik KDD, di era 1990-an lahir juga standar industri bernama CRISP-DM yang sangat populer digunakan oleh komunitas bisnis. Siklus hidup penambangan data menurut kerangka kerja ini terdiri dari 6 tahapan terstruktur:

  1. Business Understanding (Pemahaman Bisnis)
    Tahap fundamental awal untuk mengidentifikasi tujuan proyek dengan jelas dari perspektif bisnis, mengubah rumusan target menjadi definisi masalah teknis data mining.
  2. Data Understanding (Pemahaman Data)
    Mengumpulkan dataset komprehensif, mengeksplorasi isinya, mengidentifikasi kualitas masalah, dan menarik wawasan awal (hipotesis pertama).
  3. Data Preparation (Persiapan Data)
    Tahap yang paling menyita waktu; melibatkan pembersihan, transformasi, serta penyusunan data dari kondisi raw hingga membentuk tabel rapi (siap olah).
  4. Modeling (Pemodelan)
    Memilih dan menerapkan berbagai teknik data mining (algoritma Machine Learning) sambil mengkalibrasi parameternya secara optimal.
  5. Evaluation (Evaluasi)
    Memastikan model berkualitas tinggi dan meninjau kembali setiap proses yang telah dibuat hingga titik ini mematuhi/menyelesaikan masalah bisnis di poin pertama.
  6. Deployment (Penyebaran)
    Memproduksi wawasan menjadi aksi. Baik itu berupa pelaporan analisis berkala bagi petinggi manajemen, atau *deploy* algoritma ke sistem software aplikasi produksi secara real-time.
