Modul 3: Data Preprocessing — Cleaning (Pembersihan Data)

Estimasi: 60 Menit
Teori & Analisis

Selamat datang di modul Data Cleaning. Tahap awal dalam proses data preprocessing yang paling esensial adalah membersihkan data kotor agar siap digunakan dan dipahami oleh algoritma Data Mining.

Pengenalan Mengenai Pentingnya Data Cleaning

Kualitas data sangat menentukan hasil dari analisis data. Terdapat ungkapan terkenal di bidang ilmu komputer: "Garbage in, garbage out". Jika data yang digunakan untuk melatih model tidak berkualitas, tidak akurat, atau banyak mengandung kekeliruan, maka prediksi atau wawasan yang dihasilkan juga akan jauh dari kebenaran.

Tujuan Utama

Data cleaning difokuskan untuk menangani nilai yang hilang, menghapus duplikat, menangani outlier, memperbaiki data yang tidak konsisten, serta mengonversi tipe data ke format yang tepat.

Anatomi "Data Kotor"

Sebelum memulai pembersihan, kita perlu mengenali jenis masalah dataset yang umumnya terjadi di dunia nyata beserta penanganannya:

1 Missing Values (Nilai Kosong)

Penjelasan: Terjadi saat kolom memiliki nilai kosong (seperti NaN, NULL, atau ?) karena hilang atau tidak terisi.
Solusi: Imputasi (mengisi kekosongan dengan nilai rata-rata/median, misal: 25, 30) atau Hapus baris sel tersebut secara keseluruhan (drop row).

missing_values.py 
import pandas as pd

# Solusi 1: Imputasi dengan rata-rata (average/mean)
df['UMUR'].fillna(df['UMUR'].mean(), inplace=True)

# Solusi 2: Hapus baris yang memiliki nilai kosong
df.dropna(inplace=True)

2 Duplicate Data

Penjelasan: Terdapat baris data kembar/identik masuk lebih dari satu kali (misal, staf bernama ANNA LEE dengan ID: 123 terekam dua kali).
Solusi: Drop duplicates (hapus baris yang terindikasi sebagai ganda agar hanya tersisa satu entitas asli).

duplicates.py 
# Menghapus duplikat dari keseluruhan dataset
df = df.drop_duplicates()

3 Outliers

Penjelasan: Terjadi pencilan nilai yang tak wajar atau sangat ekstrem dibanding data lain (misalnya, rerata gaji 5-20 Juta, tiba-tiba ada data Rp 1 Milyar).
Solusi: Trimming (memotong nilai ekstrem secara penuh) atau Winsorizing (capping angka maksimum, seperti diset maksimal 50 Juta).

outliers.py 
import numpy as np

# Winsorizing: capping batas gaji di 50
df['gaji'] = np.where(df['gaji'] > 50, 50, df['gaji'])

4 Inconsistent Data

Penjelasan: Data memuat entitas sama namun dengan format penamaan simpang siur, seperti Kota "Jakarta", "JKT", atau "jkt".
Solusi: Standardisasi Teks (menyeragamkan tulisan secara universal, misal menjadi uppercase: JAKARTA).

inconsistent.py 
# Standarisasi huruf menjadi kapital (Uppercase) dan mapping JKT -> JAKARTA
df['kota'] = df['kota'].str.upper()
df['kota'] = df['kota'].replace('JKT', 'JAKARTA')

5 Wrong Types

Penjelasan: Nilai numerik terbaca sebagai tipe teks (object) oleh komputer gara-gara imbuhan simbol (contoh harga `Rp 5.000` atau `"10000"`).
Solusi: Type casting (konversi) dari `object` menjadi `numeric/float`.

wrong_type.py 
# Bersihkan string 'Rp ' dan titik, lalu Type Casting ke float
df['harga'] = df['harga'].str.replace(r'\D', '', regex=True) # hapus non-digit
df['harga'] = df['harga'].astype(float)
