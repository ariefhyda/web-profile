Modul 6: AI-Powered Features

Estimasi: 90 Menit
Teori & Praktikum

Setelah memahami fondasi TFLite dan tools AI dari Google, saatnya membangun fitur-fitur AI yang siap produksi di aplikasi Flutter. Modul ini membahas implementasi praktis menggunakan Google ML Kit — suite SDK on-device dari Google yang menyediakan API siap pakai untuk berbagai task ML.

Mengapa Google ML Kit?

ML Kit menyediakan model pre-trained yang sudah dioptimasi untuk mobile. Anda tidak perlu melatih model sendiri — cukup panggil API-nya. Semua berjalan on-device (offline, cepat, privasi terjaga). Tersedia untuk Android dan iOS via plugin Flutter.

Setup Google ML Kit

Instal package sesuai fitur yang dibutuhkan. Setiap fitur ML Kit adalah package terpisah agar ukuran APK tetap kecil:

Terminal 
# Tambahkan package Google ML Kit
flutter pub add google_mlkit_text_recognition
flutter pub add google_mlkit_face_detection
flutter pub add google_mlkit_object_detection
flutter pub add google_mlkit_barcode_scanning
Catatan Ukuran APK

Setiap package ML Kit menambah ~2-5 MB ke ukuran APK karena menyertakan model on-device. Instal hanya package yang Anda butuhkan untuk menjaga ukuran aplikasi tetap optimal.

1. Text Recognition (OCR)

OCR (Optical Character Recognition) mengekstrak teks dari gambar. Use case: scan dokumen, baca kartu nama, extract teks dari foto, input data otomatis.

lib/services/ocr_service.dart 
import 'package:google_mlkit_text_recognition/google_mlkit_text_recognition.dart';

class OcrService {
  final _recognizer = TextRecognizer();

  Future<String> extractText(String imagePath) async {
    final inputImage = InputImage.fromFilePath(imagePath);
    final result = await _recognizer.processImage(inputImage);

    // Gabungkan semua teks yang terdeteksi
    final buffer = StringBuffer();
    for (final block in result.blocks) {
      for (final line in block.lines) {
        buffer.writeln(line.text);
      }
    }
    return buffer.toString();
  }

  void dispose() => _recognizer.close();
}
Bahasa yang Didukung

ML Kit Text Recognition mendukung Latin, Chinese, Devanagari, Japanese, dan Korean. Untuk bahasa Indonesia (Latin script), akurasi sangat tinggi — biasanya >95% pada teks cetak yang jelas.

2. Face Detection

Deteksi wajah dengan fitur lengkap: landmark (mata, hidung, mulut), klasifikasi ekspresi (senyum, mata terbuka), dan head pose estimation.

lib/services/face_service.dart 
import 'package:google_mlkit_face_detection/google_mlkit_face_detection.dart';

class FaceService {
  final _detector = FaceDetector(
    options: FaceDetectorOptions(
      enableClassification: true,  // smile & eye open probability
      enableLandmarks: true,       // posisi mata, hidung, mulut
      enableTracking: true,         // tracking ID per wajah
      performanceMode: FaceDetectorMode.accurate,
    ),
  );

  Future<List<Map<String, dynamic>>> detectFaces(String path) async {
    final input = InputImage.fromFilePath(path);
    final faces = await _detector.processImage(input);

    return faces.map((f) => {
      return {
        'smile': f.smilingProbability,
        'leftEye': f.leftEyeOpenProbability,
        'rightEye': f.rightEyeOpenProbability,
        'headAngleY': f.headEulerAngleY,
        'bounds': f.boundingBox,
      };
    }).toList();
  }

  void dispose() => _detector.close();
}

Use case populer untuk face detection:

  1. Verifikasi Selfie
    Pastikan user mengambil foto wajah yang valid (mata terbuka, menghadap depan) untuk KYC atau registrasi.
  2. Filter & AR Effect
    Gunakan landmark wajah untuk menempatkan filter (kacamata, topi, mask) secara akurat di posisi yang tepat.
  3. Mood Detection
    Analisis smilingProbability untuk mendeteksi mood pengguna dan menyesuaikan UI/konten.
  4. Attendance System
    Deteksi dan tracking wajah untuk sistem absensi berbasis face recognition.

3. Barcode & QR Code Scanning

Scan berbagai format barcode dan QR code secara real-time. Mendukung: QR Code, EAN-13, Code 128, UPC-A, dan banyak lagi.

lib/services/barcode_service.dart 
import 'package:google_mlkit_barcode_scanning/google_mlkit_barcode_scanning.dart';

class BarcodeService {
  final _scanner = BarcodeScanner(formats: [
    BarcodeFormat.qrCode,
    BarcodeFormat.ean13,
    BarcodeFormat.code128,
  ]);

  Future<List<String>> scan(String imagePath) async {
    final input = InputImage.fromFilePath(imagePath);
    final barcodes = await _scanner.processImage(input);
    return barcodes.map((b) => b.rawValue ?? '').toList();
  }

  void dispose() => _scanner.close();
}
Format yang Didukung

• 2D: QR Code, Aztec, Data Matrix, PDF417
• 1D: EAN-8, EAN-13, UPC-A, UPC-E, Code 39, Code 93, Code 128, ITF, Codabar
Scanner ML Kit jauh lebih reliable dibanding library barcode biasa karena menggunakan ML untuk mendeteksi barcode bahkan dalam kondisi blur atau miring.

4. Smart Recommendations

Kombinasikan hasil klasifikasi ML dengan business logic untuk memberikan rekomendasi cerdas kepada pengguna. Pattern ini menggabungkan model TFLite (Modul 3) dengan rule engine sederhana:

lib/services/smart_recommender.dart 
// Pattern sederhana: rekomendasi berdasarkan klasifikasi
class SmartRecommender {
  final Map<String, List<String>> _rules = {
    'kucing': ['Makanan kucing premium', 'Mainan bulu', 'Pasir wangi'],
    'anjing': ['Dog food grain-free', 'Tali kekang', 'Bola karet'],
    'burung': ['Biji-bijian mix', 'Sangkar minimalis', 'Vitamin burung'],
  };

  List<String> getRecommendations(
      Map<String, double> classificationResult) {
    final topLabel = classificationResult.entries.first.key;
    return _rules[topLabel.toLowerCase()] ?? ['Tidak ada rekomendasi'];
  }
}
Evolusi Rekomendasi

Pattern di atas menggunakan rule-based approach (cocok untuk MVP). Untuk produksi, Anda bisa meningkatkan ke:
• Collaborative Filtering — rekomendasi berdasarkan behavior user lain yang serupa.
• Content-Based — rekomendasi berdasarkan fitur item + preferensi user.
• Hybrid — kombinasi keduanya, biasanya menggunakan model ML terpisah di server.

5. Object Detection

Berbeda dengan image classification (mengenali satu objek dominan), object detection mendeteksi banyak objek sekaligus beserta posisinya (bounding box) dalam gambar.

  1. ML Kit Object Detection
    Deteksi objek generik (5 kategori: fashion, food, home, place, plant) dengan bounding box. Cocok untuk aplikasi e-commerce dan inventory.
  2. Custom Model
    Gunakan model TFLite custom (dari Teachable Machine atau AutoML) untuk mendeteksi objek spesifik domain Anda.
  3. Real-Time Tracking
    Object detection + camera stream = tracking objek real-time. Cocok untuk AR, game, dan assistive technology.
ML Kit vs TFLite Custom Model

ML Kit — gunakan jika task Anda cocok dengan API bawaan (OCR, face, barcode, pose). Tidak perlu training, langsung pakai.
TFLite Custom — gunakan jika Anda butuh model spesifik (klasifikasi jenis tanaman, deteksi penyakit kulit, dll). Perlu training via Teachable Machine atau tools lain.

Keduanya bisa digunakan bersamaan dalam satu aplikasi.

Arsitektur AI Features di Flutter

Berikut arsitektur yang direkomendasikan saat membangun fitur AI dalam aplikasi Flutter:

  1. Service Layer
    Setiap fitur AI dibungkus dalam class service terpisah (OcrService, FaceService, dll). Service bertanggung jawab atas inisialisasi, processing, dan cleanup resource ML.
  2. Controller/ViewModel
    Mengorkestrasikan flow antara UI dan service. Mengelola state (loading, result, error) menggunakan GetX, Provider, atau Riverpod.
  3. UI Layer
    Menampilkan hasil ML ke pengguna. Gunakan CustomPainter untuk overlay bounding box, landmark, atau teks di atas gambar.
  4. Dispose Pattern
    Selalu close() recognizer/detector saat tidak digunakan. ML Kit menggunakan native resource yang harus dilepas secara eksplisit.
Best Practices AI Features

• Loading state — selalu tampilkan indikator saat ML sedang memproses.
• Error handling — tangani kasus gambar kosong, format tidak didukung, atau model gagal load.
• Permission UX — jelaskan mengapa aplikasi butuh akses kamera/galeri sebelum meminta permission.
• Hasil tidak sempurna — ML tidak 100% akurat. Berikan opsi koreksi manual ke pengguna.
• Battery awareness — untuk fitur real-time, monitor penggunaan baterai dan beri opsi untuk mematikan.
