Modul 2: Google Teachable Machine

Estimasi: 60 Menit
Teori & Praktikum

Modul ini memperkenalkan Google Teachable Machine — platform berbasis web dari Google yang memungkinkan siapa saja melatih model Machine Learning tanpa menulis kode. Anda cukup menyediakan data, melatih model secara visual, lalu mengekspor hasilnya untuk digunakan di aplikasi Flutter.

Apa itu Teachable Machine?

Teachable Machine (teachablemachine.withgoogle.com) adalah eksperimen web dari Google yang memungkinkan Anda membuat model ML secara interaktif melalui browser. Tidak perlu instalasi, tidak perlu coding, dan hasilnya bisa langsung diekspor ke format TFLite untuk Flutter.

Jenis Project di Teachable Machine

Teachable Machine mendukung tiga jenis project klasifikasi. Masing-masing sudah dioptimalkan untuk use case tertentu:

  1. Image Project
    Klasifikasi gambar berdasarkan kelas yang Anda tentukan. Contoh: mengenali jenis buah (apel vs jeruk vs mangga), deteksi ekspresi wajah (senang vs sedih), atau klasifikasi sampah (organik vs anorganik).
  2. Audio Project
    Klasifikasi suara/bunyi secara real-time. Contoh: mengenali perintah suara ("lampu nyala", "lampu mati"), membedakan jenis instrumen musik, atau mendeteksi suara batuk.
  3. Pose Project
    Klasifikasi pose tubuh menggunakan kamera. Contoh: mengenali gerakan olahraga (squat vs push-up), deteksi postur duduk yang salah, atau gesture-based control.
Mana yang Paling Sering Digunakan?

Untuk pengembangan Flutter, Image Project adalah yang paling umum karena Flutter memiliki akses langsung ke kamera dan galeri. Audio dan Pose project membutuhkan plugin tambahan namun tetap bisa diintegrasikan.

Cara Kerja Teachable Machine

Di balik antarmuka yang sederhana, Teachable Machine menggunakan teknik Transfer Learning — sebuah pendekatan di mana model yang sudah dilatih sebelumnya (pre-trained model) di-fine-tune dengan data baru Anda.

  1. Base Model: MobileNet
    Teachable Machine menggunakan MobileNet sebagai backbone — arsitektur CNN ringan buatan Google yang sudah dilatih dengan jutaan gambar dari ImageNet. MobileNet sudah "paham" fitur visual dasar (garis, bentuk, tekstur).
  2. Transfer Learning
    Daripada melatih model dari nol (yang butuh ribuan gambar dan GPU powerful), Teachable Machine hanya melatih lapisan terakhir dari MobileNet menggunakan data Anda. Ini jauh lebih cepat dan hanya butuh puluhan sampel per kelas.
  3. Training di Browser
    Proses training dilakukan langsung di browser menggunakan TensorFlow.js dan WebGL. Data Anda tidak dikirim ke server Google — semua diproses secara lokal.
Berapa Data yang Dibutuhkan?

Tidak ada angka pasti, namun sebagai pedoman:
• Minimum: 20-30 sampel per kelas
• Rekomendasi: 50-100 sampel per kelas
• Tips: Variasikan sudut, pencahayaan, dan latar belakang agar model lebih robust. Semakin beragam data, semakin baik generalisasinya.

Langkah Praktikum: Melatih Model Pertama

Mari kita praktikkan langsung membuat model klasifikasi gambar sederhana — misalnya mengenali batu, gunting, kertas dari tangan.

Step 1: Buka Teachable Machine

Kunjungi teachablemachine.withgoogle.com → klik "Get Started" → pilih "Image Project" → pilih "Standard image model".

Step 2: Siapkan Kelas & Data

Buat 3 kelas dan isi masing-masing dengan minimal 30 sampel gambar:

  1. Kelas "Batu"
    Klik "Webcam" → tunjukkan tangan mengepal (batu) ke kamera → tahan tombol "Hold to Record" untuk mengambil banyak gambar sekaligus. Variasikan posisi dan sudut tangan.
  2. Kelas "Gunting"
    Ulangi langkah yang sama dengan pose tangan gunting (dua jari terbuka).
  3. Kelas "Kertas"
    Ulangi dengan telapak tangan terbuka (kertas).
Tips Pengambilan Data

• Gunakan latar belakang yang bervariasi — bukan hanya satu tempat
• Ubah jarak tangan ke kamera (dekat, sedang, jauh)
• Ambil dari berbagai sudut (kiri, kanan, atas, bawah)
• Pastikan pencahayaan cukup, hindari backlight yang kuat

Step 3: Training Model

Setelah data siap, klik tombol "Train Model". Proses training biasanya memakan waktu 30 detik hingga 2 menit tergantung jumlah data. Anda bisa melihat progress training secara real-time.

Pengaturan Training (Advanced)

Klik "Advanced" di bawah tombol Train untuk mengatur:
• Epochs — jumlah iterasi training (default: 50). Naikkan jika akurasi rendah.
• Batch Size — jumlah sampel per batch (default: 16). Biarkan default kecuali data sangat banyak.
• Learning Rate — kecepatan belajar (default: 0.001). Turunkan jika model overfitting.

Step 4: Testing Model

Setelah training selesai, panel "Preview" akan aktif secara otomatis. Arahkan tangan Anda ke webcam dan lihat apakah model bisa mengenali pose batu, gunting, atau kertas dengan benar. Perhatikan confidence score (persentase keyakinan) untuk setiap kelas.

Jika akurasi belum memuaskan:

  1. Tambah data
    Kelas dengan akurasi rendah biasanya kekurangan variasi data. Tambah lebih banyak sampel dengan sudut/pencahayaan berbeda.
  2. Cek class imbalance
    Pastikan jumlah sampel antar kelas tidak terlalu timpang (misalnya 100 vs 20). Idealnya seimbang.
  3. Naikkan epochs
    Buka Advanced settings dan naikkan epochs menjadi 100-200 untuk training lebih lama.

Step 5: Export Model

Ini adalah langkah paling krusial — mengekspor model agar bisa digunakan di Flutter.

  1. Klik "Export Model"
    Di panel Preview, klik tombol "Export Model".
  2. Pilih Tab "TensorFlow Lite"
    Dari tiga opsi (TF.js, TF, TFLite), pilih "TensorFlow Lite" karena format ini yang digunakan di Flutter.
  3. Pilih Quantization
    Pilih "Quantized" (bukan Floating point). Model quantized berukuran 4x lebih kecil dan lebih cepat di mobile, dengan penurunan akurasi minimal.
  4. Download File
    Klik "Download my model". Anda akan mendapat file ZIP berisi:
    • model_unquant.tflite — file model
    • labels.txt — daftar nama kelas
labels.txt 
# Isi file labels.txt yang dihasilkan
0 Batu
1 Gunting
2 Kertas

Menempatkan Model di Project Flutter

Setelah file model didownload, tempatkan ke dalam project Flutter Anda:

Struktur Folder 
my_flutter_app/
+-- assets/
|   +-- ml/
|       +-- model.tflite      # File model dari Teachable Machine
|       +-- labels.txt        # Daftar label kelas
+-- lib/
|   +-- main.dart
+-- pubspec.yaml              # Daftarkan assets di sini
pubspec.yaml 
# Tambahkan di bagian flutter:
flutter:
  assets:
    - assets/ml/model.tflite
    - assets/ml/labels.txt
Apa Selanjutnya?

Model sudah siap! Di modul berikutnya (Implementasi Model di Flutter), kita akan mengintegrasikan model TFLite ini ke dalam aplikasi Flutter menggunakan package tflite_flutter, memproses input gambar dari kamera, dan menampilkan hasil klasifikasi secara real-time.

Rangkuman Modul 2

Teachable Machine — platform no-code dari Google untuk training model ML di browser.

Transfer Learning — teknik memanfaatkan pre-trained model (MobileNet) agar training cepat dengan data sedikit.

3 Jenis Project — Image, Audio, dan Pose classification.

Export ke TFLite — pilih format Quantized untuk ukuran kecil dan performa optimal di mobile.

Data Privasi — semua data diproses lokal di browser, tidak dikirim ke server Google.
