Modul 5: Google Stitch

Estimasi: 60 Menit
Teori & Praktikum

Google Stitch adalah platform desain UI/UX berbasis AI dari Google Labs yang memungkinkan Anda merancang antarmuka aplikasi lengkap hanya dari deskripsi teks. Stitch menghasilkan layout multi-screen, design system, dan bahkan kode frontend — mempercepat workflow dari ide ke prototipe secara drastis.

Asal Usul Stitch

Google Stitch lahir dari akuisisi startup Galileo AI oleh Google. Platform ini diluncurkan melalui Google Labs pada Mei 2025, dan terus berkembang dengan fitur-fitur baru seperti infinite canvas, voice interaction, dan integrasi ke Google AI Studio.

Apa yang Bisa Dilakukan Stitch?

Stitch bukan sekadar generator gambar — ia memahami konteks UI/UX dan menghasilkan desain yang terstruktur:

  1. Prompt-to-UI Generation
    Deskripsikan antarmuka dalam bahasa natural, dan Stitch menghasilkan layout lengkap dengan komponen yang sesuai (button, card, form, navigation, dll).
  2. Multi-Screen Design
    Buat flow aplikasi lengkap: login → home → detail → checkout dalam satu sesi. Stitch menjaga konsistensi visual antar layar.
  3. Interactive Prototyping
    Hubungkan antar screen untuk membuat prototipe yang clickable. Simulasikan user flow tanpa menulis kode.
  4. Code Export
    Ekspor desain ke kode frontend (HTML/CSS, React, Tailwind). Kode yang dihasilkan bersih dan bisa langsung digunakan sebagai referensi.
  5. Iterasi via Natural Language
    “Perbesar tombol login”, “Ganti warna jadi biru navy” — cukup ketik instruksi dan Stitch memperbarui desain secara instan.
Stitch vs Figma

Figma — tool desain profesional untuk design system yang kompleks dan kolaborasi tim besar. Kurva belajar tinggi.
Stitch — tool AI untuk rapid prototyping dan eksplorasi ide. Cocok untuk tahap awal, developer solo, atau validasi konsep cepat.

Keduanya saling melengkapi: gunakan Stitch untuk menghasilkan konsep awal, lalu Figma untuk refinement profesional.

Teknologi di Balik Stitch

Stitch didukung oleh teknologi AI canggih dari Google:

  1. Google Gemini
    Model AI multimodal yang memahami teks, gambar, dan konteks UI. Gemini menangani layout structure, logic komponen, dan konsistensi visual.
  2. Infinite Canvas
    Workspace tak terbatas untuk menempatkan iterasi desain secara berdampingan. Bandingkan variasi dan pilih yang terbaik.
  3. Voice Interaction
    Bicara langsung ke Stitch untuk memodifikasi desain secara real-time — tanpa mengetik.
  4. MCP Integration
    Model Context Protocol memungkinkan Stitch terhubung ke Google AI Studio dan coding agents untuk pipeline desain-ke-kode yang seamless.

Praktikum: Mendesain UI dengan Stitch

Mari praktikkan membuat desain halaman login untuk aplikasi delivery food.

Step 1: Akses Google Stitch

Buka stitch.withgoogle.com → login dengan akun Google → klik “New Project”. Platform ini gratis dengan kuota bulanan.

Step 2: Tulis Prompt Pertama

Kunci dari Stitch adalah prompt yang deskriptif. Semakin detail prompt Anda, semakin akurat hasilnya:

Contoh Prompt 
// Contoh prompt untuk Google Stitch

"Buatkan halaman login untuk aplikasi delivery food.
Gunakan warna utama orange (#FF6B35) dan putih.
Ada logo di atas, form email dan password,
tombol Login besar, dan link Register di bawah.
Style modern dengan rounded corners dan shadow."
Tips Menulis Prompt yang Baik

• Sebutkan konteks aplikasi — “delivery food”, “fitness tracker”, dll.
• Definisikan warna — gunakan hex code untuk presisi.
• Deskripsikan komponen — sebutkan elemen UI spesifik (form, card, bottom nav).
• Tentukan style — modern, minimalis, glassmorphism, dll.

Step 3: Iterasi Desain

Setelah Stitch menghasilkan desain awal, Anda bisa menyempurnakannya dengan prompt tambahan. Tidak perlu mulai dari awal — cukup deskripsikan perubahan yang diinginkan:

Prompt Iterasi 
// Prompt iterasi setelah desain pertama

"Ubah tombol Login menjadi gradient orange ke kuning.
Tambahkan opsi login dengan Google dan Apple.
Perkecil logo 20% dan tambahkan tagline di bawahnya.
Buat form field lebih tinggi (48px) dengan icon di kiri."

Step 4: Export Kode

Setelah desain final, klik “Export” untuk mendapatkan kode frontend. Stitch menghasilkan HTML/CSS atau React yang terstruktur:

Output HTML/CSS dari Stitch 
// Contoh output HTML/CSS dari Google Stitch
<div class="login-container">
  <div class="logo-section">
    <img src="logo.svg" alt="Logo" />
    <p class="tagline">Fresh food, delivered fast</p>
  </div>
  <form class="login-form">
    <input type="email" placeholder="Email" />
    <input type="password" placeholder="Password" />
    <button class="btn-login">Login</button>
  </form>
</div>

Step 5: Konversi ke Flutter Widget

Stitch belum secara native menghasilkan kode Dart/Flutter, tapi output HTML/CSS-nya menjadi blueprint yang sangat akurat untuk dikonversi ke Flutter widget:

lib/pages/login_page.dart 
// Konversi manual dari output Stitch ke Flutter Widget
class LoginPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(child: Padding(
        padding: EdgeInsets.all(32),
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // Logo dari desain Stitch
            Image.asset('assets/logo.svg', height: 80),
            SizedBox(height: 8),
            Text('Fresh food, delivered fast',
                style: TextStyle(color: Colors.grey)),
            SizedBox(height: 40),

            // Form fields sesuai spesifikasi Stitch
            TextField(decoration: InputDecoration(
              hintText: 'Email',
              prefixIcon: Icon(Icons.email),
              border: OutlineInputBorder(
                  borderRadius: BorderRadius.circular(12)))),
            SizedBox(height: 16),
            TextField(obscureText: true,
              decoration: InputDecoration(
              hintText: 'Password',
              prefixIcon: Icon(Icons.lock),
              border: OutlineInputBorder(
                  borderRadius: BorderRadius.circular(12)))),
            SizedBox(height: 24),

            // Gradient button dari Stitch
            Container(
              width: double.infinity, height: 48,
              decoration: BoxDecoration(
                gradient: LinearGradient(
                  colors: [Color(0xFFFF6B35), Color(0xFFFFB347)]),
                borderRadius: BorderRadius.circular(12)),
              child: ElevatedButton(
                onPressed: () {},
                style: ElevatedButton.styleFrom(
                  backgroundColor: Colors.transparent,
                  shadowColor: Colors.transparent),
                child: Text('Login',
                    style: TextStyle(fontSize: 16,
                        fontWeight: FontWeight.bold)))),
    ])));
  }
}
Workflow Stitch → Flutter

Alur kerja yang direkomendasikan:
1. Desain di Stitch (prompt → iterate → finalize)
2. Export kode HTML/CSS sebagai referensi visual & struktural
3. Konversi manual ke Flutter widget (perhatikan spacing, warna, border radius)
4. Gunakan AI coding assistant untuk mempercepat konversi

Di masa depan, Stitch kemungkinan akan mendukung ekspor langsung ke Dart/Flutter.

Use Case Stitch untuk Flutter Developer

  1. Rapid Prototyping
    Validasi ide aplikasi dalam hitungan menit. Tunjukkan prototipe ke klien atau stakeholder sebelum menulis satu baris kode Flutter.
  2. Design Reference
    Gunakan output Stitch sebagai panduan visual saat membangun widget Flutter. Lebih cepat daripada mendesain dari nol di Figma.
  3. Eksplorasi Layout
    Generate 5-10 variasi layout dalam waktu singkat. Pilih yang paling cocok, lalu implementasikan di Flutter.
  4. Onboarding Klien
    Buat mockup aplikasi klien dalam meeting langsung. Stitch cukup cepat untuk demo real-time.
  5. Learning UI/UX
    Pelajari prinsip desain dengan melihat bagaimana Stitch menyusun layout, memilih spacing, dan mengatur hierarki visual.
Keterbatasan Stitch (Saat Ini)

• Belum ada export Dart/Flutter — konversi masih manual.
• Kuota terbatas — versi gratis memiliki batas generasi per bulan.
• Masih eksperimental — hasil bisa inkonsisten untuk desain yang sangat kompleks.
• Perlu koneksi internet — semua processing dilakukan di cloud Google.
• Bukan pengganti Figma — untuk design system enterprise, Figma tetap lebih mature.

Rangkuman Modul 5

Google Stitch — platform AI dari Google Labs untuk desain UI/UX via natural language prompt.
Powered by Gemini — menghasilkan layout terstruktur, bukan sekadar gambar.
Workflow — prompt → generate → iterate → export → konversi ke Flutter.
Export — HTML/CSS/React, digunakan sebagai blueprint untuk Flutter widget.
Best for — rapid prototyping, design reference, dan eksplorasi layout.
