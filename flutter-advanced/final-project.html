Modul 7: Final Project Advanced

Estimasi: 2-3 Minggu
Project

Selamat! Anda telah menyelesaikan seluruh materi teori dan praktikum di course Flutter Advanced. Sekarang saatnya menggabungkan semua yang telah dipelajari ke dalam satu proyek aplikasi lengkap — dari training model, integrasi AI, hingga UI yang polished dan siap dipresentasikan.

Tujuan Final Project

Membangun aplikasi Flutter yang mengintegrasikan minimal 2 fitur AI/ML (satu dari TFLite custom model + satu dari ML Kit), dengan arsitektur yang bersih, UI modern, dan pengalaman pengguna yang baik. Proyek ini menjadi portofolio Anda sebagai Flutter developer yang menguasai AI/ML.

Pilihan Tema Project

Pilih salah satu tema berikut, atau ajukan tema custom yang disetujui dosen:

  1. Smart Plant Identifier 🌱
    Aplikasi identifikasi jenis tanaman dari foto. Scan daun/bunga → klasifikasi species → tampilkan info perawatan. Bonus: OCR untuk membaca label tanaman.
  2. Food Calorie Scanner 🍲
    Ambil foto makanan → klasifikasi jenis makanan → estimasi kalori → catat ke food diary harian. Bonus: barcode scanner untuk makanan kemasan.
  3. Smart Attendance 👤
    Sistem absensi berbasis face detection. Deteksi wajah → verifikasi → catat kehadiran otomatis. Bonus: liveness detection (cek mata berkedip).
  4. Trash Classifier ♻
    Klasifikasi jenis sampah (organik/anorganik/B3/recycle) dari foto. Berikan panduan cara membuang yang benar. Bonus: gamifikasi dengan poin & leaderboard.
  5. Document Scanner Pro 📄
    Scan dokumen → OCR → konversi ke teks editable → simpan/share. Bonus: deteksi bahasa otomatis, translate via API.
  6. Tema Custom 💡
    Ajukan ide Anda sendiri! Syarat: wajib menggunakan minimal 1 model TFLite custom + 1 fitur ML Kit.
Rekomendasi untuk Pemula

Jika ini pertama kali Anda membangun proyek AI di Flutter, pilih Smart Plant Identifier atau Trash Classifier. Keduanya menggunakan image classification (sudah dipraktikkan di Modul 2-3) dan mudah mendapatkan dataset training dari internet.

Studi Kasus: Smart Plant Identifier

Sebagai panduan, berikut breakdown lengkap untuk tema Smart Plant Identifier. Adaptasikan struktur ini untuk tema yang Anda pilih.

Fase 1: Training Model (Minggu 1)

  1. Kumpulkan Dataset
    Siapkan minimal 5 kelas tanaman (misalnya: mawar, melati, anggrek, lidah buaya, kaktus). Kumpulkan 50-100 gambar per kelas dari berbagai sudut dan pencahayaan.
  2. Training di Teachable Machine
    Upload dataset → train model → test akurasi → iterate jika perlu. Target akurasi: >85% pada test set.
  3. Export ke TFLite
    Export sebagai Quantized TFLite. Rename file menjadi plant_model.tflite dan plant_labels.txt.
Sumber Dataset

• Kaggle — cari “plant species dataset” atau “flower dataset”
• Google Images — download manual, pastikan variasi cukup
• Foto sendiri — ambil foto langsung dari kebun/taman. Ini memberikan data paling realistis untuk use case lokal.

Fase 3: Implementasi Fitur (Minggu 2)

Implementasikan fitur-fitur berikut secara bertahap:

  1. ClassifierService (Hari 1-2)
    Implementasi dari Modul 3. Load model, preprocess gambar, jalankan inferensi. Test dengan gambar statis terlebih dahulu.
  2. Camera & Gallery Integration (Hari 2-3)
    Halaman scan dengan pilihan kamera dan galeri. Tampilkan preview gambar sebelum klasifikasi.
  3. Result Page (Hari 3-4)
    Tampilkan hasil klasifikasi dengan confidence bar yang visual. Tambahkan informasi tanaman (deskripsi, tips perawatan).
  4. Fitur ML Kit (Hari 4-5)
    Tambahkan OCR untuk membaca label tanaman, atau barcode scanner untuk scan QR code info tanaman.
  5. History & Persistence (Hari 5-6)
    Simpan hasil scan ke database lokal (sqflite). Tampilkan riwayat scan dengan thumbnail dan tanggal.
  6. Share Feature (Hari 6-7)
    Fitur share hasil scan ke media sosial menggunakan share_plus.

Fase 4: Polish & Deploy (Minggu 3)

  1. UI Polish
    Terapkan Material Design 3 dengan tema konsisten. Tambahkan animasi transisi antar halaman, loading skeleton, dan empty state yang menarik.
  2. Error Handling
    Tangani semua edge case: permission ditolak, kamera tidak tersedia, model gagal load, gambar terlalu gelap, dll.
  3. Performance Optimization
    Test inference time di device asli. Pertimbangkan GPU delegate jika diperlukan. Optimasi ukuran gambar sebelum processing.
  4. Testing
    Test dengan minimal 20 gambar dari masing-masing kelas. Catat akurasi dan kasus yang gagal. Dokumentasikan di laporan.
  5. Build APK/AAB
    Generate release build. Test di device fisik. Pastikan semua fitur berjalan di release mode (bukan hanya debug).

Format Pengumpulan

  1. Source Code
    Push ke repository GitHub/GitLab. Pastikan README.md berisi: deskripsi app, screenshot, cara setup, dan daftar fitur.
  2. APK Release
    Sertakan file APK release yang bisa langsung diinstall untuk demo.
  3. Laporan Singkat (2-3 halaman)
    Berisi: arsitektur aplikasi, proses training model (screenshot Teachable Machine + akurasi), tantangan yang dihadapi, dan solusinya.
  4. Presentasi (10 menit)
    Demo live aplikasi + penjelasan arsitektur + sesi tanya jawab.
Tips Sukses Final Project

• Mulai dari yang kecil — buat MVP (klasifikasi + 1 halaman) dulu, lalu iterasi.
• Test model duluan — pastikan model TFLite bekerja sebelum membangun UI yang kompleks.
• Commit sering — gunakan Git dengan commit message yang jelas. Ini juga bukti progres Anda.
• Jangan copy-paste buta — pahami setiap baris kode dari modul sebelumnya sebelum mengadaptasinya.
• Minta feedback — tunjukkan progres ke teman atau dosen di tengah pengerjaan, jangan tunggu deadline.
