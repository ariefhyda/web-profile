Teori & Praktikum

Di modul sebelumnya, kita berhasil mengintegrasikan model TFLite ke Flutter. Sekarang saatnya memahami lebih dalam tentang TensorFlow Lite — arsitekturnya, teknik optimasi model, penggunaan hardware accelerator (GPU/NNAPI), dan cara menjalankan inferensi real-time dari kamera.

Apa itu TensorFlow Lite?

TensorFlow Lite (TFLite) adalah framework open-source dari Google yang dirancang khusus untuk menjalankan model ML di perangkat edge — smartphone, IoT, embedded system — dengan footprint minimal. TFLite bukan untuk training, melainkan untuk inference (menjalankan prediksi).

Arsitektur TFLite di Mobile

Memahami arsitektur TFLite penting agar Anda bisa mengoptimasi pipeline ML di aplikasi Flutter:

  1. Model File (.tflite)
    File binary berisi graph operasi dan weight model yang sudah dioptimasi. Format FlatBuffer memungkinkan loading tanpa parsing — langsung memory-mapped.
  2. Interpreter
    Engine yang mengeksekusi model. Membaca input tensor, menjalankan setiap operasi secara berurutan, dan menghasilkan output tensor.
  3. Delegates
    Akselerator hardware opsional yang mendelegasikan operasi ke GPU, NNAPI (Android), atau Core ML (iOS) untuk performa lebih tinggi.
  4. Tensor
    Struktur data multi-dimensi (array) yang menjadi input dan output model. Setiap tensor punya shape, data type, dan ukuran tetap.
FlatBuffer vs Protobuf

Model TensorFlow biasa menggunakan Protobuf (ratusan MB, perlu parsing). TFLite menggunakan FlatBuffer — format serialisasi yang bisa diakses langsung dari memori tanpa deserialisasi. Hasilnya: loading model 10-100x lebih cepat.

Quantization: Memperkecil Model

Quantization adalah teknik mengkonversi weight model dari floating-point 32-bit (FP32) ke integer 8-bit (INT8) atau float 16-bit (FP16). Ini menghasilkan model yang jauh lebih kecil dan cepat.

  1. Dynamic Range Quantization
    Paling mudah diterapkan. Weight dikonversi ke INT8 saat penyimpanan, tapi komputasi tetap di FP32. Ukuran model berkurang ~4x dengan penurunan akurasi minimal.
  2. Full Integer Quantization
    Semua operasi dijalankan dalam INT8. Butuh representative dataset untuk kalibrasi. Paling optimal untuk hardware yang mendukung operasi integer (NNAPI, Edge TPU).
  3. Float16 Quantization
    Weight dikonversi ke FP16. Ukuran berkurang 2x, kompatibel dengan GPU delegate. Akurasi hampir identik dengan FP32.
quantize_model.py 
# Python: Konversi model ke TFLite dengan quantization
import tensorflow as tf

# Load model Keras/SavedModel
converter = tf.lite.TFLiteConverter.from_saved_model('saved_model/')

# === Dynamic Range Quantization ===
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()

with open('model_quantized.tflite', 'wb') as f:
    f.write(tflite_model)
print('Model quantized!')
Kapan Menggunakan Quantization?

Selalu! Untuk deployment mobile, quantization hampir selalu menguntungkan. Panduan singkat:
• Dynamic Range — default terbaik, cocok untuk hampir semua kasus.
• Full INT8 — jika target perangkat mendukung NNAPI/Edge TPU.
• FP16 — jika butuh akurasi tinggi + GPU delegate.

Hardware Delegates

Secara default, TFLite menjalankan operasi di CPU. Dengan delegates, operasi tertentu bisa didelegasikan ke hardware khusus untuk percepatan signifikan:

  1. GPU Delegate
    Memanfaatkan GPU mobile (Adreno, Mali, PowerVR) via OpenGL ES / OpenCL. Cocok untuk model dengan banyak operasi matrix (convolution, dense). Speedup: 2-7x vs CPU.
  2. NNAPI Delegate (Android)
    Android Neural Networks API — mendelegasikan ke akselerator terbaik yang tersedia (GPU, DSP, atau NPU). Performa sangat device-dependent. Terbaik di Pixel dan Samsung flagship.
  3. Core ML Delegate (iOS)
    Memanfaatkan Apple Neural Engine di chip A11+. Sangat efisien di perangkat Apple modern. Diakses via platform channel di Flutter.
lib/services/classifier_with_delegate.dart 
import 'package:tflite_flutter/tflite_flutter.dart';

enum DelegateType { cpu, gpu, nnapi }

class ClassifierWithDelegate {
  Future<Interpreter> _createInterpreter(DelegateType delegate) async {
    final options = InterpreterOptions();

    switch (delegate) {
      case DelegateType.gpu:
        options.addDelegate(GpuDelegateV2());
        break;
      case DelegateType.nnapi:
        options.useNnApiForAndroid = true;
        break;
      case DelegateType.cpu:
        options.threads = 4;
        break;
    }

    return await Interpreter.fromAsset(
      'assets/ml/model.tflite',
      options: options,
    );
  }
}
Kapan GPU Delegate Tidak Membantu?

GPU delegate tidak selalu lebih cepat. Beberapa kasus di mana CPU bisa lebih optimal:
• Model sangat kecil (overhead transfer data ke GPU > waktu komputasi).
• Model menggunakan operasi yang tidak didukung GPU (custom ops).
• Perangkat low-end dengan GPU lemah.

Real-Time Inference dari Kamera

Salah satu use case paling powerful dari on-device ML adalah klasifikasi real-time dari stream kamera. Berikut pattern-nya menggunakan package camera:

lib/pages/realtime_page.dart (pattern) 
import 'package:camera/camera.dart';
import '../services/classifier_service.dart';

class _RealtimeState extends State<RealtimePage> {
  CameraController? _camCtrl;
  final _classifier = ClassifierService();
  bool _isProcessing = false;
  Map<String, double>? _results;

  @override
  void initState() {
    super.initState();
    _classifier.loadModel();
    _initCamera();
  }

  Future<void> _initCamera() async {
    final cameras = await availableCameras();
    _camCtrl = CameraController(
      cameras[0],
      ResolutionPreset.medium,
      enableAudio: false,
      imageFormatGroup: ImageFormatGroup.yuv420,
    );
    await _camCtrl!.initialize();

    // Stream setiap frame untuk klasifikasi
    _camCtrl!.startImageStream((CameraImage image) {
      if (!_isProcessing && _classifier.isReady) {
        _isProcessing = true;
        _classifyFrame(image);
      }
    });
    setState(() {});
  }

  void _classifyFrame(CameraImage camImg) {
    // Konversi YUV420 ke RGB (img.Image), lalu ke Uint8List
    final results = _classifier.classifyFromCameraImage(camImg);
    if (mounted) {
      setState(() {
        _results = results;
        _isProcessing = false;
      });
    }
  }
}
Tips Performa Real-Time

• Konversi ImageFormat — Kamera Android mengirim format YUV420. Perlu dikonversi ke RGB sebelum masuk ke model.
• Skip frame — jangan proses setiap frame. Proses 1 dari setiap 3-5 frame sudah cukup.
• Resolusi rendah — gunakan ResolutionPreset.medium atau low.
• Isolate — jalankan inferensi di background isolate agar UI tidak lag.
• Dispose — selalu dispose controller dan delegate saat halaman ditutup.

Best Practices On-Device ML

• Pilih model yang tepat — MobileNet, EfficientNet-Lite dirancang khusus untuk mobile.
• Quantize selalu — Dynamic range quantization adalah default yang aman.
• Benchmark di device asli — emulator tidak representatif untuk performa ML.
• Fallback strategy — jika GPU delegate gagal, fallback ke CPU secara graceful.
• Batasi frekuensi inferensi — untuk real-time, 10-15 FPS sudah cukup smooth.
• Monitor battery — inferensi terus-menerus bisa menguras baterai.
