Modul 2.1: Image Project

Estimasi: 45 Menit
Praktikum

Image Project adalah jenis project yang paling populer di Teachable Machine. Modul ini akan membimbing Anda mulai dari pembuatan model hingga strategi implementasi di Flutter baik melalui galeri maupun kamera real-time.

Langkah Praktikum: Membuat Model (Batu, Gunting, Kertas)

Kita akan membuat model yang dapat membedakan pose tangan Batu, Gunting, dan Kertas. Ikuti tahapan berikut secara berurutan agar Anda memahami konsep di baliknya.

Step 1: Persiapan Project

Buka Teachable Machine, klik Get Started, lalu pilih Image Project. Pilih Standard image model. Model standar memiliki kompatibilitas yang sangat luas untuk diimplementasikan ke dalam aplikasi Flutter.

Step 2: Pengumpulan Data (Dataset Gathering)

Kualitas model Anda sangat bergantung pada data yang Anda berikan (Garbage In, Garbage Out). Pada antarmuka Teachable Machine, ikuti panduan berikut saat mengambil gambar melalui webcam:

Definisi Kelas

Buat 3 kelas utama: Batu, Gunting, dan Kertas.

Jumlah Sampel

Gunakan webcam untuk mengambil minimal 100 gambar per kelas.

Variasi Data

Variasikan jarak tangan, sudut, dan latar belakang agar model lebih robust.

Step 3: Proses Training

Setelah data terkumpul, klik tombol Train Model. Pada tahap ini, Teachable Machine sedang menyesuaikan (fine-tuning) lapisan terakhir dari model MobileNet agar dapat mengenali pola spesifik dari tangan Anda.

Penting Saat Training

Jangan berpindah tab browser selama proses kalkulasi berlangsung. Browser Anda menggunakan resource secara lokal (CPU/GPU) sehingga berpindah tab dapat menyebabkan proses terhenti.

Step 4: Evaluasi & Testing

Setelah training selesai, panel Preview akan otomatis aktif. Uji coba dengan mengarahkan pose batu, gunting, atau kertas ke webcam dan perhatikan hasil persentase (confidence score).

Troubleshooting Akurasi Rendah
  • Data Tidak Seimbang (Class Imbalance): Pastikan jumlah data per kelas hampir sama (misal: 100 vs 105 vs 98).
  • Kurang Variasi Sudut: Model mungkin gagal menebak dari samping jika Anda hanya melatihnya dengan foto dari depan.
  • Latar Belakang Bias: Jika Anda memfoto "Batu" di kamar dan "Kertas" di luar, model mungkin salah belajar membedakan lokasi, bukan tangannya.

Step 5: Export Model ke TFLite

Ini adalah tahap krusial untuk menghubungkan Teachable Machine dengan Flutter. Ikuti konfigurasi export ini dengan teliti:

  1. Klik tombol Export Model pada panel Preview.
  2. Pilih tab TensorFlow Lite. (TFLite adalah format yang dioptimalkan untuk perangkat mobile seperti Android/iOS).
  3. Pilih jenis conversion Quantized. Model quantized mengorbankan sedikit sekali akurasi namun menghasilkan ukuran file yang ~4x lebih kecil dan komputasi yang jauh lebih cepat di memori HP.
  4. Klik Download my model. File ZIP yang terunduh akan berisi dua file: model_unquant.tflite dan daftar label di dalam labels.txt.

Langkah Praktikum 2: Implementasi di Flutter (Gambar Statis)

Setelah model diekspor, saatnya mengintegrasikan model_unquant.tflite dan labels.txt ke dalam project Flutter Anda. Kita akan membangun aplikasi yang dapat mengambil gambar dari kamera/galeri lalu mengklasifikasikannya.

Step 1: Tambahkan Dependencies

Kita membutuhkan tiga package utama untuk menjalankan model ML dan memproses gambar:

Terminal 
# Jalankan di terminal project Flutter Anda
flutter pub add tflite_flutter
flutter pub add image_picker
flutter pub add image
  • tflite_flutter: Eksekutor model TFLite di Flutter.
  • image_picker: Akses galeri dan kamera bawaan HP.
  • image: Manipulasi piksel gambar (resize, crop) untuk preprocessing.
Catatan Platform

Android: Pastikan Anda menggunakan minSdkVersion 21 di android/app/build.gradle karena tflite membutuhkan versi Android yang cukup baru.
iOS: Tambahkan key NSCameraUsageDescription dan NSPhotoLibraryUsageDescription di Info.plist untuk izin akses kamera dan galeri.

Step 2: Siapkan Assets Model

Buat folder assets/ml/ di dalam direktori utama project Anda. Masukkan model_unquant.tflite dan labels.txt ke dalamnya, lalu daftarkan di pubspec.yaml:

pubspec.yaml 
flutter:
  assets:
    - assets/ml/model.tflite
    - assets/ml/labels.txt

Step 3: Buat Classifier Service

Buat file lib/services/classifier_service.dart. Class ini memisahkan logika Machine Learning (Separation of Concerns) dari antarmuka pengguna (UI).

lib/services/classifier_service.dart 
import 'dart:io';
import 'dart:typed_data';
import 'package:flutter/services.dart';
import 'package:tflite_flutter/tflite_flutter.dart';
import 'package:image/image.dart' as img;

class ClassifierService {
  Interpreter? _interpreter;
  List<String>? _labels;
  static const int inputSize = 224;
  bool _isReady = false;

  bool get isReady => _isReady;

  /// Load model dan labels dari assets
  Future<void> loadModel() async {
    try {
      _interpreter = await Interpreter.fromAsset('assets/ml/model.tflite');
      final labelData = await rootBundle.loadString('assets/ml/labels.txt');
      _labels = labelData.split('\n')
          .where((l) => l.trim().isNotEmpty)
          .map((l) => l.replaceAll(RegExp(r'^\d+\s*'), ''))
          .toList();
      _isReady = true;
    } catch (e) {
      _isReady = false;
      rethrow;
    }
  }

  /// Preprocess gambar — model quantized memerlukan input uint8 (0-255)
  Uint8List _preprocessImage(File imageFile) {
    final raw = img.decodeImage(imageFile.readAsBytesSync())!;
    final resized = img.copyResize(raw, width: inputSize, height: inputSize);
    var buf = Uint8List(1 * inputSize * inputSize * 3);
    int idx = 0;
    for (int y = 0; y < inputSize; y++) {
      for (int x = 0; x < inputSize; x++) {
        final p = resized.getPixel(x, y);
        buf[idx++] = p.r.toInt();
        buf[idx++] = p.g.toInt();
        buf[idx++] = p.b.toInt();
      }
    }
    return buf;
  }

  /// Jalankan inferensi secara async agar tidak membekukan (freeze) UI
  Future<Map<String, double>> classify(File imageFile) async {
    if (!_isReady || _interpreter == null || _labels == null) {
      throw Exception('Model belum dimuat. Panggil loadModel() dahulu.');
    }

    // Isolasi komputasi berat di microtask
    final input = await Future(() => _preprocessImage(imageFile).reshape([1, inputSize, inputSize, 3]));
    var output = Uint8List(_labels!.length).reshape([1, _labels!.length]);
    
    _interpreter!.run(input, output);

    final results = <String, double>{};
    for (int i = 0; i < _labels!.length; i++) {
      // Hasil output model quantized berupa angka 0-255, jadikan persentase 0.0-1.0
      results[_labels![i]] = (output[0][i] as int) / 255.0;
    }

    // Urutkan probabilitas terbesar di atas
    return Map.fromEntries(results.entries.toList()..sort((a, b) => b.value.compareTo(a.value)));
  }

  void dispose() => _interpreter?.close();
}
Mengapa Preprocessing Penting?

Model MobileNet secara baku hanya menerima gambar berukuran 224x224 piksel. Selain itu, karena kita menggunakan model Quantized, data piksel RGB tidak perlu dinormalisasi ke angka desimal (float), melainkan cukup dikirim mentah-mentah sebagai bilangan bulat Uint8 (0-255). Jika tidak di-preprocess dengan benar, aplikasi akan crash.

Step 4: Buat UI Klasifikasi

Buat file lib/pages/classify_page.dart. UI ini menyediakan tombol kamera/galeri, preview gambar, dan progress bar untuk persentase kecocokan (confidence score).

lib/pages/classify_page.dart 
import 'dart:io';
import 'package:flutter/material.dart';
import 'package:image_picker/image_picker.dart';
import '../services/classifier_service.dart';

class ClassifyPage extends StatefulWidget {
  @override
  State<ClassifyPage> createState() => _ClassifyPageState();
}

class _ClassifyPageState extends State<ClassifyPage> {
  final _classifier = ClassifierService();
  final _picker = ImagePicker();
  File? _selectedImage;
  Map<String, double>? _results;
  bool _isLoading = false;
  bool _modelLoading = true;
  String? _errorMessage;

  @override
  void initState() {
    super.initState();
    _initModel();
  }

  Future<void> _initModel() async {
    try {
      await _classifier.loadModel();
      setState(() => _modelLoading = false);
    } catch (e) {
      setState(() {
        _modelLoading = false;
        _errorMessage = 'Gagal memuat model: $e';
      });
    }
  }

  Future<void> _pickAndClassify(ImageSource src) async {
    final picked = await _picker.pickImage(source: src);
    if (picked == null) return;

    setState(() {
      _selectedImage = File(picked.path);
      _isLoading = true;
      _results = null;
      _errorMessage = null;
    });

    try {
      final res = await _classifier.classify(_selectedImage!);
      setState(() {
        _results = res;
        _isLoading = false;
      });
    } catch (e) {
      setState(() {
        _isLoading = false;
        _errorMessage = 'Error klasifikasi: $e';
      });
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Image Classifier')),
      body: _modelLoading
          ? Center(child: CircularProgressIndicator())
          : SingleChildScrollView(
              padding: EdgeInsets.all(20),
              child: Column(children: [
                if (_errorMessage != null)
                  Text(_errorMessage!, style: TextStyle(color: Colors.red)),
                if (_selectedImage != null)
                  ClipRRect(
                    borderRadius: BorderRadius.circular(16),
                    child: Image.file(_selectedImage!, height: 250, fit: BoxFit.cover)),
                SizedBox(height: 20),
                Row(mainAxisAlignment: MainAxisAlignment.center, children: [
                  ElevatedButton.icon(
                    onPressed: _isLoading ? null : () => _pickAndClassify(ImageSource.camera),
                    icon: Icon(Icons.camera_alt), label: Text('Kamera')),
                  SizedBox(width: 12),
                  OutlinedButton.icon(
                    onPressed: _isLoading ? null : () => _pickAndClassify(ImageSource.gallery),
                    icon: Icon(Icons.photo_library), label: Text('Galeri')),
                ]),
                SizedBox(height: 24),
                if (_isLoading) CircularProgressIndicator(),
                if (_results != null)
                  ..._results!.entries.map((e) => Padding(
                    padding: EdgeInsets.symmetric(vertical: 6),
                    child: Row(children: [
                      Expanded(flex: 3, child: Text(e.key, style: TextStyle(fontWeight: FontWeight.w600))),
                      Expanded(flex: 7, child: LinearProgressIndicator(value: e.value, minHeight: 14, borderRadius: BorderRadius.circular(6))),
                      SizedBox(width: 12),
                      Text('${(e.value * 100).toStringAsFixed(1)}%'),
                    ]))),
              ]),
            ),
    );
  }

  @override
  void dispose() {
    _classifier.dispose();
    super.dispose();
  }
}

Langkah Praktikum 3: Implementasi Realtime (Kamera Stream)

Selain gambar statis, salah satu use case paling powerful dari on-device ML adalah klasifikasi secara real-time langsung dari stream kamera. Implementasi ini membutuhkan penanganan performa ekstra karena model harus memproses frame demi frame dengan sangat cepat.

Step 1: Tambahkan Package Kamera

Tambahkan package camera resmi dari Flutter ke dalam project Anda untuk mendapatkan akses stream langsung dari lensa kamera perangkat genggam Anda.

Terminal 
flutter pub add camera

Step 2: Bangun Logika Realtime Inference

Buat file baru lib/pages/realtime_page.dart. Di sini kita menggunakan fungsi startImageStream bawaan CameraController. Kunci utamanya adalah jangan memproses frame baru jika frame sebelumnya masih dikalkulasi oleh model.

lib/pages/realtime_page.dart 
import 'package:camera/camera.dart';
import '../services/classifier_service.dart';

class _RealtimeState extends State<RealtimePage> {
  CameraController? _camCtrl;
  final _classifier = ClassifierService();
  bool _isProcessing = false;
  Map<String, double>? _results;

  @override
  void initState() {
    super.initState();
    _classifier.loadModel();
    _initCamera();
  }

  Future<void> _initCamera() async {
    final cameras = await availableCameras();
    _camCtrl = CameraController(
      cameras[0], // Gunakan kamera belakang
      ResolutionPreset.medium, // Hemat RAM dengan resolusi medium
      enableAudio: false,
      imageFormatGroup: ImageFormatGroup.yuv420, // Format standar Android
    );
    await _camCtrl!.initialize();

    // Mulai stream setiap frame (sekitar 30 frame per detik)
    _camCtrl!.startImageStream((CameraImage image) {
      // Flag _isProcessing mencegah aplikasi hang akibat menumpuknya frame
      if (!_isProcessing && _classifier.isReady) {
        _isProcessing = true;
        _classifyFrame(image);
      }
    });
    setState(() {});
  }

  void _classifyFrame(CameraImage camImg) {
    // Di skenario aslinya, Anda perlu menambahkan fungsi classifyFromCameraImage
    // pada ClassifierService untuk mengonversi frame YUV420 ke RGB.
    final results = _classifier.classifyFromCameraImage(camImg);
    if (mounted) {
      setState(() {
        _results = results;
        _isProcessing = false; // Buka gerbang untuk memproses frame selanjutnya
      });
    }
  }

  @override
  void dispose() {
    _camCtrl?.dispose();
    _classifier.dispose();
    super.dispose();
  }
}
Tips Performa Real-Time

• Konversi ImageFormat: Kamera Android mengirim format YUV420. Data ini perlu diolah menjadi array RGB mentah sebelum masuk ke TFLite.
• Resolusi Rendah: Selalu gunakan ResolutionPreset.medium atau low. Model MobileNet kita toh hanya menerima format 224x224 piksel.
• Skip Frame: Membaca 30 frame per detik terlalu memberatkan CPU. Memproses 1 dari 5 frame sudah sangat cukup untuk real-time.
• Clean Up: Selalu panggil dispose() pada kamera controller ketika user menutup halaman (kembali ke *home*) agar kamera tidak stuck.

Langkah Praktikum 4: Benchmarking Performa (Opsional)

Sangat penting untuk selalu mengukur kecepatan (waktu inferensi) model Anda secara aktual di perangkat target. Proses ini disebut dengan Benchmarking. Hal ini untuk memastikan bahwa model yang Anda buat cukup ringan untuk diakses oleh HP pengguna.

Menghitung Waktu Eksekusi 
// Mulai stopwatch sebelum fungsi run dijalankan
final sw = Stopwatch()..start();

// Eksekusi klasifikasi model
_interpreter.run(input, output);

// Hentikan stopwatch
sw.stop();
print('Waktu inferensi (ms): ${sw.elapsedMilliseconds} ms');
Target Performa MobileNet (Image Project)
Jenis Model Ukuran File Perkiraan Waktu
Standar (FP32) ~14 MB 80-100 ms / frame
Quantized (INT8) ~3.4 MB 10-45 ms / frame

*Pengujian di Flutter menunjukkan bahwa memilih model Quantized pada tahap export dari Teachable Machine adalah kunci utama untuk mendapatkan performa realtime yang optimal (di bawah 50ms) tanpa membebani memori HP.
