7 Modul • Advanced Level
Teori & Praktikum

Selamat datang di course Flutter Advanced. Course ini membahas integrasi Artificial Intelligence dan Machine Learning ke dalam aplikasi Flutter, termasuk Google Teachable Machine, TensorFlow Lite, dan Google Stitch.

Prasyarat

Course ini membutuhkan pemahaman kuat dari Flutter Fundamentals dan Flutter Intermediate. Anda harus sudah familiar dengan arsitektur GetX, REST API, dan konsep async programming di Dart.

Mengapa On-Device Machine Learning?

Secara tradisional, aplikasi mobile mengirim data ke cloud server untuk diproses oleh model ML, lalu menerima hasilnya kembali. Pendekatan ini memiliki banyak keterbatasan. On-Device ML memindahkan proses inferensi langsung ke perangkat pengguna — sehingga model berjalan di HP, bukan di server.

  1. Privasi Data
    Data pengguna (foto, suara, teks) tidak perlu dikirim ke server. Semua diproses secara lokal di perangkat.
  2. Latensi Rendah (Real-Time)
    Tanpa round-trip ke server, inferensi bisa dilakukan dalam hitungan milidetik.
  3. Bekerja Offline
    Aplikasi tetap berfungsi tanpa koneksi internet.
  4. Hemat Biaya Server
    Tidak perlu menyewa GPU cloud yang mahal untuk serving model.
  5. Skalabilitas Otomatis
    Setiap perangkat menjalankan modelnya sendiri tanpa bottleneck server.
Cloud ML vs On-Device ML

Cloud ML — cocok untuk model besar (LLM, generative AI) yang membutuhkan GPU powerful. Contoh: ChatGPT, DALL·E.

On-Device ML — cocok untuk model ringan yang butuh respons cepat dan privasi tinggi. Contoh: face detection, image classification, pose estimation.

Dalam course ini, kita fokus pada On-Device ML karena paling relevan untuk pengembangan aplikasi mobile.

Ekosistem Format Model di Mobile

Sebelum model ML bisa berjalan di perangkat mobile, model harus dikonversi ke format yang dioptimalkan untuk hardware terbatas (CPU/GPU mobile). Berikut adalah format-format utama yang perlu Anda ketahui:

  1. TensorFlow Lite (.tflite)
    Format dari Google, paling populer untuk Android dan Flutter. Mendukung quantization untuk memperkecil ukuran model hingga 4x. Inilah format utama yang akan kita gunakan di course ini.
  2. Core ML (.mlmodel)
    Format native Apple untuk iOS/macOS. Dioptimalkan untuk Apple Neural Engine. Flutter bisa mengaksesnya melalui platform channel.
  3. ONNX (.onnx)
    Format open-source yang bersifat framework-agnostic. Model dari PyTorch, Keras, atau scikit-learn bisa dikonversi ke ONNX, lalu ke TFLite untuk Flutter.
  4. MediaPipe (Google)
    Framework siap pakai dari Google untuk task ML umum (face mesh, hand tracking, pose detection) tanpa perlu training model sendiri.
Workflow Umum di Course Ini

Train (Teachable Machine / Python) → Export (.tflite) → Integrate (Flutter + tflite_flutter) → Deploy (APK/AAB)

Anda tidak perlu menguasai TensorFlow atau Python secara mendalam. Teachable Machine menyediakan antarmuka visual untuk training, dan kita akan fokus pada sisi integrasi di Flutter.
