Pengenalan YOLO (Object Detection)

Estimasi: 30 Menit
Teori & Konsep

YOLO (You Only Look Once) adalah salah satu algoritma Machine Learning paling populer untuk Object Detection (Deteksi Objek) secara real-time. Tidak seperti klasifikasi gambar biasa yang hanya menebak "apa" yang ada di gambar, Object Detection menebak "apa" dan "di mana" lokasinya menggunakan kotak pembatas (Bounding Box).

1. Mengapa YOLO Sangat Populer?

Sebelum YOLO, algoritma deteksi objek menggunakan metode "Two-Stage Detector" (seperti R-CNN), di mana sistem pertama-tama mencari region (area) yang mungkin berisi objek, lalu menjalankan klasifikasi pada setiap area tersebut. Pendekatan ini akurat namun sangat lambat.

Sesuai namanya (You Only Look Once), YOLO memandang seluruh gambar hanya satu kali (satu tahap/one-stage) menggunakan arsitektur jaringan saraf tunggal (Single Neural Network). Ini membuat YOLO sangat cepat dan efisien, sehingga ideal untuk deteksi objek realtime pada video atau kamera ponsel.

Real-Time Speed

YOLO mampu memproses frame video dengan sangat cepat (bisa mencapai 45-155 FPS), memungkinkan deteksi tanpa lag pada kamera.

Global Context

Karena memproses seluruh gambar sekaligus, YOLO memahami konteks global dan jarang membuat kesalahan (false positive) pada background.

Single Network

Satu Neural Network memprediksi Bounding Box dan probabilitas kelas secara bersamaan secara end-to-end.

2. Konsep Kunci dalam YOLO

A. Bounding Box & Confidence Score

YOLO membagi gambar menjadi sebuah grid (misalnya 13x13). Setiap sel grid bertanggung jawab untuk mendeteksi objek yang titik tengahnya berada di dalam sel tersebut. Sel akan memprediksi Bounding Box (x, y, lebar, tinggi) dan nilai Confidence Score (seberapa yakin kotak itu berisi objek tertentu).

B. Intersection over Union (IoU)

YOLO sering menghasilkan banyak prediksi kotak untuk satu objek. Untuk membersihkannya, digunakan IoU (Irisan dibagi Gabungan) untuk mengevaluasi seberapa tumpang tindih prediksi kotak dengan area objek sebenarnya, dan algoritma Non-Maximum Suppression (NMS) digunakan untuk hanya menyimpan kotak dengan probabilitas tertinggi dan membuang duplikatnya.

3. Perkembangan Versi YOLO

  • YOLOv1 hingga v3: Mengukuhkan dominasi YOLO di bidang deteksi objek real-time.
  • YOLOv4 & v5: Mulai dioptimalkan untuk performa komputasi yang lebih ringan, mempopulerkan implementasi di mobile (iOS/Android).
  • YOLOv8 & v10 (Ultralytics): Versi terbaru yang sangat canggih, tidak hanya untuk deteksi objek (Object Detection), tetapi juga Instance Segmentation dan Pose Estimation dengan akurasi dan efisiensi memori yang jauh lebih baik.
Membawa YOLO ke Flutter

Untuk menjalankan model YOLO (seperti YOLOv8) di aplikasi Flutter, biasanya model asli (misal format PyTorch .pt) harus dieksport ke format TensorFlow Lite (.tflite). Di Flutter, package tflite_flutter digunakan untuk membaca model tersebut, lalu kita memerlukan script tambahan di sisi Dart untuk mengolah output tensor menjadi Bounding Box yang bisa digambar menggunakan CustomPaint di atas layar kamera (Realtime).
