Modul 8: Convolutional Neural Network (CNN)

Estimasi: 90 Menit
Deep Learning & Praktikum

Convolutional Neural Network (CNN) adalah arsitektur deep learning yang dirancang khusus untuk memproses data berbentuk grid — terutama citra. CNN merevolusi bidang Computer Vision sejak kemenangan AlexNet di kompetisi ImageNet 2012, dan kini menjadi backbone hampir seluruh sistem vision modern.

Mengapa CNN untuk Citra?

Neural network biasa (Fully Connected) tidak efisien untuk memproses citra karena jumlah parameter yang sangat besar. Citra 224×224×3 saja memiliki 150,528 input neuron. CNN mengatasi ini dengan tiga prinsip kunci:

  1. Local Connectivity
    Setiap neuron hanya terhubung ke wilayah kecil (receptive field) dari input, bukan seluruh input. Ini memanfaatkan fakta bahwa pixel yang berdekatan saling berkorelasi tinggi.
  2. Parameter Sharing
    Filter/kernel yang sama diterapkan ke seluruh area citra. Satu filter 3×3×3 hanya memiliki 27 parameter — dibanding jutaan pada fully connected layer.
  3. Translation Invariance
    Karena filter yang sama digeser ke seluruh posisi, CNN dapat mendeteksi fitur yang sama di lokasi manapun pada citra.

Arsitektur CNN: Layer by Layer

1. Convolutional Layer

Layer utama CNN. Sejumlah filter (kernel) berukuran kecil (misal 3×3, 5×5) digeser (convolved) sepanjang citra input. Setiap filter menghasilkan satu feature map yang mendeteksi fitur spesifik (tepi, tekstur, pola).

2. Activation Function (ReLU)

ReLU (Rectified Linear Unit): f(x) = max(0, x). Menambahkan non-linearitas agar network dapat mempelajari fungsi kompleks. Menggantikan sigmoid/tanh karena menghindari masalah vanishing gradient.

3. Pooling Layer

Melakukan down-sampling untuk mengurangi dimensi spasial feature map. Max Pooling 2×2 (paling umum) mengambil nilai maksimum dari setiap window 2×2, mengurangi ukuran feature map menjadi setengah.

4. Fully Connected Layer

Di akhir network, feature map di-flatten menjadi vektor 1D dan dihubungkan ke layer fully connected untuk melakukan klasifikasi akhir menggunakan softmax.

Praktikum: Image Classification dengan PyTorch

cnn_classifier.py 
import torch
import torch.nn as nn
import torchvision
import torchvision.transforms as transforms

# 1. Definisi arsitektur CNN sederhana
class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128 * 4 * 4, 256),
            nn.ReLU(),
            nn.Dropout(0.5),
            nn.Linear(256, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

# 2. Dataset CIFAR-10
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5,0.5,0.5), (0.5,0.5,0.5))
])
trainset = torchvision.datasets.CIFAR10(
    root='./data', train=True,
    download=True, transform=transform
)
trainloader = torch.utils.data.DataLoader(
    trainset, batch_size=64, shuffle=True
)

# 3. Training loop
model = SimpleCNN()
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(10):
    running_loss = 0.0
    for images, labels in trainloader:
        optimizer.zero_grad()
        outputs = model(images)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
        running_loss += loss.item()
    print(f"Epoch {epoch+1}, Loss: {running_loss/len(trainloader):.4f}")
Transfer Learning

Dalam praktiknya, jarang kita melatih CNN dari nol. Transfer Learning menggunakan model pre-trained (ResNet, EfficientNet) yang sudah belajar fitur dari jutaan gambar ImageNet, lalu hanya melatih ulang layer terakhir untuk tugas spesifik kita. Ini menghemat waktu dan data secara drastis.
