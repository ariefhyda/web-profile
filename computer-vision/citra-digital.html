Modul 2: Representasi Citra Digital

Estimasi: 60 Menit
Teori & Praktikum

Sebelum kita melakukan pengolahan atau analisis citra, kita harus memahami terlebih dahulu bagaimana sebuah citra direpresentasikan secara digital di dalam komputer. Pemahaman ini menjadi fondasi untuk seluruh materi Computer Vision selanjutnya.

Apa itu Citra Digital?

Citra digital adalah representasi dua dimensi dari sebuah scene visual yang disimpan dalam format numerik. Secara teknis, sebuah citra digital tersusun atas kumpulan elemen terkecil yang disebut pixel (picture element). Setiap pixel menyimpan nilai intensitas cahaya pada posisi (x, y) tertentu.

Jika sebuah citra memiliki ukuran W × H pixel, maka citra tersebut dapat direpresentasikan sebagai sebuah matrix atau array 2D berukuran H baris dan W kolom. Pada citra berwarna (RGB), setiap pixel memiliki 3 nilai — sehingga representasinya menjadi array 3D berukuran H × W × 3.

Analogi Sederhana

Bayangkan sebuah citra sebagai spreadsheet Excel raksasa — setiap sel menyimpan angka yang mewakili kecerahan (grayscale) atau tiga angka untuk warna merah, hijau, biru (RGB). Semakin banyak sel (pixel), semakin detail citra tersebut.

Model Warna (Color Space)

Sebuah pixel dapat direpresentasikan dalam berbagai color space (ruang warna). Pemilihan color space yang tepat sangat penting karena memengaruhi efektivitas algoritma CV yang digunakan.

  1. Grayscale (Skala Abu-abu)
    Setiap pixel memiliki satu nilai intensitas antara 0 (hitam) hingga 255 (putih). Cocok untuk analisis bentuk dan tepi karena menghilangkan informasi warna yang tidak relevan. Ukuran data = H × W.
  2. RGB (Red, Green, Blue)
    Model warna paling umum. Setiap pixel memiliki 3 channel: R, G, B masing-masing bernilai 0-255. Total kemungkinan warna: 256³ = 16.7 juta warna. Ukuran data = H × W × 3.
  3. HSV (Hue, Saturation, Value)
    Memisahkan informasi warna (Hue), kepekatan warna (Saturation), dan kecerahan (Value). Sangat berguna untuk deteksi objek berdasarkan warna karena lebih robust terhadap perubahan pencahayaan dibanding RGB.
  4. BGR
    Format default yang digunakan oleh OpenCV. Urutannya dibalik dari RGB: Blue → Green → Red. Penting untuk diperhatikan saat konversi antar library (OpenCV vs PIL vs Matplotlib).

Praktikum: Membaca & Menampilkan Citra dengan OpenCV

01_read_image.py 
import cv2
import numpy as np
import matplotlib.pyplot as plt

# 1. Membaca citra dari file
img_bgr = cv2.imread('sample.jpg')  # Format: BGR (default OpenCV)

# 2. Cek properti citra
print(f"Shape   : {img_bgr.shape}")    # (H, W, C) misal (480, 640, 3)
print(f"Dtype   : {img_bgr.dtype}")    # uint8 (0-255)
print(f"Min/Max : {img_bgr.min()} / {img_bgr.max()}")

# 3. Konversi BGR → RGB (untuk Matplotlib)
img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)

# 4. Konversi ke Grayscale
img_gray = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2GRAY)

# 5. Konversi ke HSV
img_hsv = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2HSV)

# 6. Visualisasi
fig, axes = plt.subplots(1, 3, figsize=(15, 5))
axes[0].imshow(img_rgb)
axes[0].set_title('RGB')
axes[1].imshow(img_gray, cmap='gray')
axes[1].set_title('Grayscale')
axes[2].imshow(img_hsv)
axes[2].set_title('HSV')
for ax in axes: ax.axis('off')
plt.tight_layout()
plt.show()

Resolusi dan Bit Depth

Dua parameter penting yang menentukan kualitas dan ukuran citra digital:

  1. Resolusi (Spatial Resolution)
    Jumlah pixel per dimensi. Resolusi 1920×1080 berarti citra memiliki ~2 juta pixel. Semakin tinggi resolusi, semakin detail citra namun semakin besar ukuran file dan memori yang dibutuhkan.
  2. Bit Depth (Kedalaman Bit)
    Jumlah bit yang digunakan untuk merepresentasikan nilai setiap pixel. 8-bit = 256 level (0–255), 16-bit = 65536 level. Citra medis dan astronomi sering menggunakan 16-bit atau 32-bit float untuk presisi yang lebih tinggi.

Praktikum: Memanipulasi Pixel Secara Langsung

02_pixel_manipulation.py 
import cv2
import numpy as np

# Membuat citra kosong (hitam) berukuran 300×400, 3 channel
canvas = np.zeros((300, 400, 3), dtype=np.uint8)

# Mengakses dan mengubah pixel individual
canvas[100, 200] = [0, 255, 0]  # Pixel hijau di (y=100, x=200)

# Menggambar region/ROI (Region of Interest)
canvas[50:150, 100:300] = [255, 0, 0]   # Kotak biru (BGR)
canvas[160:260, 50:200] = [0, 0, 255]   # Kotak merah (BGR)

# Split channel RGB
b, g, r = cv2.split(canvas)
print(f"Blue channel shape: {b.shape}")  # (300, 400)

# Menampilkan hasil
cv2.imshow('Canvas', canvas)
cv2.waitKey(0)
cv2.destroyAllWindows()
Penting: Konvensi Axis

Dalam OpenCV dan NumPy, akses pixel menggunakan format [y, x] atau [row, col] — bukan [x, y]! Ini adalah salah satu sumber bug paling umum bagi pemula. Baris (y) dihitung dari atas ke bawah, kolom (x) dari kiri ke kanan.
